# 0. 다형성

## 1. 다형성이란?

- 한 타입의 참조변수를 통해 여러 타입의 객체를 참조할 수 있도록 하는 것.
	- 즉 하나의 자리에 여러가지 객체가 올 수 있다.
- <font color="#ff0000">상위 클래스가 동일한 메시지로 하위 클래스가 서로 다른 동작이 가능</font>

>즉 역할과 구현의 분리가 가능하게 하는 성질
>-  하나의 인터페이스를 구현한 다양한 구현체 클래스가 있다고 하자.
>-  각각의 클래스는 하나의 인터페이스를 구현했으므로 하나의 인터페이스 타입으로 관리가 가능하다
>- 해당 인터페이스를 구현한 어떠한 클래스가 와도 상관이 없다.
>- 즉 역할(인터페이스)만 알고 있다면 구현(구현 클래스)는 몰라도 된다.

### 1.1 다형성의 장점

- 유리보수의 편의성 : 여러 객체를 하나의 타입으로 관리할 수 있어 유지보수가 용이하다
- 재사용성 : 객체의 재사용이 쉬워 재사용성이 높아진다.
- 느슨한 결합 : 클래스 간의 의존성을 줄여 확장성을 높아지고, 결합도는 낮아진다.

### 1.2 다형성의 조건

- 상위 클래스와 하위 클래스는 상속관계여야 한다.
- 다형성이 보장되기 위해 오버라이딩(하위 클래스의 메서드 재정의)가 반드시 필요하다.
- 자식 클래스 객체가 부모 클래스의 타입으로 업캐스팅해야 한다.

### 1.3 JAVA에서 다형성을 실현하는 방법

- 오버로딩, 오버라이팅
- 업캐스팅, 다운 캐스팅
- 인터페이스
- 추상클래스, 추상 메소드

# 1. 상속

 - 상속이란? - 부모 클래스의 클래스 <font color="#ff0000">변수와 메서드를 재사용, 확장</font>하는 것

### 1. 1 클래스 상속

```java title:"클래스 상속받는 방법 - extends"
public Class Parent{ .... }; // 부모 클래스
public Class Child extends Parent { .... }; // 자식 클래스
```
- 위와 같이 extends 키워드를 붙이고, 상속받을 부모 클래스를 적는다.
	- <font color="#ffff00"><u>자바는 다중상속은 불가능!</u></font>
- 자식 객체를 생성하면 부모 객체가 먼저 생성된 다음에 자식 객체가 생성된다.(그렇기 때문에 부모의 생성자가 호출된다.)
	- <span style="background:#fff88f">어떻게 보면 당연하다. extends를 해주는 부모 클래스가 생성이 되야지, 자식 클래스를 만들 수 있기 때문이다.</span>

```java title:"클래스 상속 예시 - 부모 클래스"
public class ParentBook{
  String name; //필드
  int price;

public void Print(){ // 메소드
  System.out.println("책의 이름과 가격 : "+name+" "+price);
}
```

```java title:"클래스 상속 예시 - 자식 클래스", hl:3-4
public class ChildBook extends ParentBook{
  ChildBook (String name, int price){ // 생성자
    this.name = name;  //  ChildBook이 ParetBook의 필드를 상속받아서 가능한 선언
    this.price = price;
  }
}
```

```java title:"클래스 상속 예시 - 실행코드", hl:4
public static void main (String[] args){
  ChildBook Child = new ChildBook("나의 라임오렌지 나무", 10000);
  System.out.print("[구현 결과 1] ");
  Child.Print();
}

==========<출력결과>==========

[구현 결과 1] 책의 이름과 가격 : 나의 라임오렌지 나무 10000
```

- 위와 같이 ChildBook은 name과 price, 즉 필드와 Print() 메소드를 선언하지 않았음에도, ParentBook을 상속받았기 때문에,
	-  this.name 선언 가능
	- Child.Print() 메소드 사용가능

> **😀 부모 생성자 호출 - super()**
>
> super() : 자식 생성자 안에서 사용되면 부모 클래스의 생성자를 의미한다 
   -> 즉 ChildBook 클래의 생성자에서 super()를 하면 부모 클래스 Book의 생성자가 호출된다.
>
>무조건 생성자의 첫번째 줄에서만 사용가능 하다.



## 1.2 상속의 장점

### 1.2.1 코드의 재사용

- 부모 클래스의 필드와 메서드를 상속받은 자식 클래스가 다시 사용
- 적은 양의 코드로 새로운 클래스 작성 가능
- 생산성이 올라감

### 1.2.2 유지 보수의 편의성

- 코드를 공통적으로 관리하기 때문에, 여러 코드의 추가 및 변경에 용이

# 2. 인터페이스

- 인터페이스란? - 클래스들이 필수로 구현해야하는 추상 자료형

```java title:"인터페이스 사용 예제 - Animal 인터페이스"
package Example;

public interface Animal {
    public static final String name = "동물";
    
    public abstract void move();
    public abstract void eat();
    public abstract void bark();
}
```
- 인터페이스는 <font color="#ff0000">static, final로 생성할 수 있는 상수</font>와 <font color="#ff0000">abstract로 생성할 수 있는 추상 메서드</font>만 가짐

```java title:"인터페이스 사용 예제 - Dog 클래스"
package Example;

public class Dog implements Animal{
    
    @Override
    public void move() {
        System.out.println("슥슥~~");
    }
    
    @Override
    public void bark() {
        System.out.println("멍멍!");
    }
}

```
- 위의 Animal 인터페이스를 implemnet 키워드로 구현
- 인터페이스의 <font color="#ff0000">추상 메서드들을 모두 재정의 함.(Override)</font>

___
# 3. 상속 vs 인터페이스

### 차이점
- 상속은 단일 클래스에 대해서만 가능
- 인터페이스는 다중 구현 가능

- 상속은 자식 클래스에서 부모 클래스의 메서드를 전부 Override할 필요는 없음
- 인터페이스는 반드시 구현해야함.

### 공통점
- 상속과 인터페이스 모두 자바에서 <span style="background:#fff88f">다형성을 실현하는 방법임!</span>
