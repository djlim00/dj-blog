6. 다음 중 예외(Exception) 에 대한 설명 중 올바르지 않은 것은?
	 예외 : 잘못된 사용 또는 코딩으로 인한 오류
	- 에러와 달리 예외 처리를 통해 계속 실행상태 유지 가능
		- 일반 예외(exception) : 컴파일러가 예외처리 코드 여부를 검사하는 예외
		- 실행 예외(run-time exception) : 컴파일러가 예외 처리 코드 여부를 검사하지 않은 예외
	
		**try - catch - finally 블록으로 구성**
	
	- try : 안에 있는 코드의 예외를 처리하겠다는 범위
	- catch : 어떤 종류의 예외를 어떻게 처리할지를 결정함
	- finally : 최종적으로 수행되어야하는 작업을 지정
	
	- catch문은 자세한 클래스(하위 클래스) -> 큰 클래스(상위 클래스) 순서로 적어야한다.
