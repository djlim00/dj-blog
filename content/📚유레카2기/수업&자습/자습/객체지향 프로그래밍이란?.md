객체지향프로그래밍(Object-Oriented-Programming/OOP)이란 프로그래밍에서 필요한 <font color="#ffff00">데이터를 추상화</font>시켜 <font color="#ffff00">상태와 행위를 가진 객체</font>로 만들고, 객체들간의 <font color="#ffff00">상호작용 통해 로직을 구성</font>하는 프로그래밍 방법이다.

객체지향 프로그래밍이란, 프로그램에서 필요한 데이터를 추상화([[모델링#개념 모델(비즈니스 모델)]])하여, 상태와 행위를 가진 객체로 만들고, 객체들의 상호작용을 통해서 로직을 구성하는 프로그래밍 방법이다.

##### 객체
그럼 여기서 '객체'란 무엇일까?
객체는 상태와 행위를 가진다. 이때 프로그래밍에서 상태란 '변수'를 뜻하고, 행위는 '메서드'를 의미한다.
즉 객체는 어떠한 데이터를 표현하지 위한 변수와 메서드를 모아서 만든 것이다.
ex) 자동차
- 상태 -> 변수 : 속력, 방향, 차 색상, 차종...etc
- 행위 -> 메서드 : 전진, 후진, 우회전, 좌회전, 브레이크, 창문열기...etc

##### 객체지향프로그래밍 지원언어
객체지향프로그래밍을 구현하기 위해서는 위의 내용을 표현하고, 구현하는데 필요한 기능이 있어야한다.
아래의 언어들은 그러한 기능을 제공한다.
: C++ , C# , Java , Python , JavaScript , Ruby , Swift

##### 객체지향프로그래밍 특징

1. **추상화** 
	- 객체에서 공통된 속성(상태)과 행위를 추출하는 것
	- 내가 만드려는 프로그램의 목적에 맞는, 필요한 변수와 메서드를 추출

2. **캡슐화**
	- 변수와 메서드를 묶어서 데이터로 저장하는 것
	- 즉 그냥 <font color="#ffff00">클래스로</font> 만드는 것.
3. **상속**
	- 클래스의 속성과 행위를 하위 클래스에 물려주거나, 하위 클래스가 상위 클래스의 속성과 행위를 물려 받는 것
	- 즉 이미 있는 클래스를 재사용할 수 있는가?
	- 장점
		1. 코드 재사용 가능
		2. 범용적으로 클래스 사용 가능
		3. 데이터와 메서드의 자유로운 사용 및 추가 가능
	- 단점
		1. 상위 클래스를 바꾸기 어려움
		2. 클래스의 과잉
		3. 잘못된 상속 사용
4. **다형성**
	- 하나의 변수명, 함수명이 다양한 의미로 해석
	- <font color="#ffff00">역할과 구현의 분리</font>
	- 오버로딩 - 하나의 함수명이 파라미터를 다르게 해서 재사용
	- 오버라이딩 - 상위 클래스의 메서드를 재정의해서 사용


##### OOP의 장단점

1. 장점
	- 업무 분담 편리 -> 대규모 프로젝트
	- 유지보수
	- 확장성
2. 단점
	- 처리속도 느림
	- 객체의 수가 많음 -> 용량 커짐
	- 설계에 어려움
