# 자바 컬렉션 프레임워크

- 널리 알려진 <font color="#ffff00">자료구조</font>를 바탕으로 객체들을 효율적으로 추가, 삭제, 검색할 수 있도록 인터페이스와 클래스들을 포함시켜 놓은 java.util 패키지 -> 즉 자바의 자료구조 프레임워크
	- 주요 인터페이스 : List, Set, Map

### 1. ArrayList

- 중간의 값을 삭제하면, 전부 앞, 뒤로 한칸씩 당겨지거나 밀리 때문에 -> 빈번한 추가와 삭제가 있다면 좋지 않은 자료구조
- index로 value를 구분
> List 인터페이스
> - 객체로 생성을 할려면, List 인터페이스를 구현한 ArrayList, LinkedList, Stack, Vector를 이용해서 객체를 만든다.

### 1.1 LinkedList

- 각각의 데이터 객체들의 앞 뒤의 노드에 대한 정보를 가지고 있음
- 추가, 삭제가 빈번한 경우 사용하면 좋음
- 조회가 잦은 경우는 조회를 위해서는 0번부터 N번까지 전부 조회를 해야하므로 불리하다.
### 2. Map

- key-value 형식으로 저장 -> key값으로 value를 구분(key값은 중복되지 않는다)
### 3. Set

- 중복을 허용하지 않는 자료구조, 순서가 존재하지 않는다.
#### 3.1 TreeSet
- 이진트리 기반, 부모보다 작은건 왼쪽 큰건 오른쪽
- 정렬이 되는 Set
- lower(), higher() : 지금보다 하나 크거나 작은 객체를 반환


> **Iterator** : 지시자(커서)를 달아주는 개념 
> hasNext() : 다음 데이터가 있는지 여부 반환
> next() : 다음 데이터 읽어옴

# Input/OutputStream

- Input  : 프로그램으로 데이터가 들어온다 -> 프로그램이 파일을 읽는 것
- Output : 프로그램이 파일을 쓰는 것

1. byteStream - 단위 8비트(알파벳) : ~inputStream, ~outputStream
2. charStream - 단위 16비트(한국어) : ~reader, ~writer

- nodeStream - 데이터 소스 or destination를 연결하는 기능
- filterStream - 데이터 이동 중 가공처리

- FileInputStream = node(8비트)
- InputStreamReader = filter(8비트를 16비트로 바꿈) / out도 마찬가지

- BufferReader / BufferedWriter는 직접적으로 입출력이 불가능 하기 때문
	-> InputStreamReader/OutputStreanWriter가 필요힘
