# 스트림
___
: 배열의 요소를 반복 처리하기 위해 스트림 사용
: 요소를 하나씩 흘러가면서 처리된다는 의미

- List컬렉션의 stream()매소드로 tream객체를 얻고, forEach()를 통해서 각각 어떻게 할지를 정함
- for문, while문 - 외부반복자 : 외부에서 값을 처리
- 스트림을 이용한 람다식 - 컬렉션 내부에서 처리

### 일반적인 forEach()사용
```java
Iteratort<String> iterator = set.iterator();

Stream<String> stream = set.stream(); // 스트림 객체 뽑기
stream.forEach( name -> System.out.println(name) )
```

### 쓰레드 병렬처리
```java hl:15-18
package Stream;  
  
import java.util.ArrayList;  
import java.util.List;  
import java.util.stream.Stream;  
  
public class ParallelStreamExample {  
    public static void main(String[] args) {  
        List<String> list = new ArrayList<>();  
        list.add("333");  
        list.add("444");  
        list.add("555");  
        list.add("666");  
  
        Stream<String> parallelStream = list.parallelStream();  
        parallelStream.forEach(name -> {  
            System.out.println(name + ": " + Thread.currentThread().getName());  
        });  
    }  
}

______________________________________________________________________

444: ForkJoinPool.commonPool-worker-1
333: ForkJoinPool.commonPool-worker-3
666: ForkJoinPool.commonPool-worker-2
555: main
```
- List에 parallelStream()메서드를 이용해서 <span style="background:#fff88f">ForkJoinPool</span>을 사용하는 스트림을 만들어준다.
- 결과를 보면 쓰레드는 항상 경쟁상태이기 떄문에 list의 순서와는 다르게 그때그때 다른 순서로 나온다.

### 포크조인 프레임워크

- 포크 단계 : 전체 요소들을 서브 요소셋으로 분할하고, 각각의 서브 요소셋을 멀티 코어에서 병렬로 처리
- 조인 단계 : 서브 결과를 결합해서 최종 결과를 만들어냄.

# 자료구조
___
- **배열의 인덱스를 이용한 접근은 빠르다** -> 계속 호출되야하는 값을 배열로 만들어 저장해보자
- List의 경우
	- Linked-List : 값의 추가, 삭제가 빠르다, 탐색시간 증가(순차적으로 접근해야하므로O(n)만큼 걸림), 포인터 메모리 먹음
	- ArrayList : 값을 추가하는 연산이 시간복잡도가 크다(전부 당기거나 밀어야함)

- HashMap은 탐색 시간이 O(1)로 빠르다 -> 탐색, 추가, 삭제가 빠름 
	- Hash Key를 이용한 인덱스를 가지고 있기 때문에

- TreeMap, TreeSet은 레드-블랙 트리로 이루어진 정렬이된 Map과 Set이다! -> 종종 사용하자
