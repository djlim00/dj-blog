
# 데이터 타입 분류

- 기본 타입(Primitive Type) : 실제 값을 가지고 있음

- 참조 타입(Reference Type) : 네임스페이스에 변수의 이름 -> 값이 있는 주소 공간에 대한 주소 -> 실제 값이 있는 주소에 대한 데이터 -> 실제 값 접근
	- 즉 바로 데이터를 넣는 것이 아니라, 데이터가 들어가 있는 곳을 저장함.
		- ex) String[]이면 값 자체가아니 값이 들어있는 주소값을 가지고 있음.

# 메모리 사용영역

### 1. 메소드 영역
: 바이트코드 파일을 읽은 내용이 저장되는 영역
- 모든 쓰레드가 공유하는 영역
- 상수, 정적필드, 메소드 코드, 생성자 코드 등이 저장됨.


### 2. 힙 영역
: 객체가 생성되는 영역이다. 객체의 번지는 메소드 영역과 스택 영역의 상수와 변수에서 참조할 수 있다.
- 모든 쓰레드가 공유

### 3. 스택 영역
: 메서드가 호출될 떄 마다 생성되는 프레임이 저장되는 영역. 메서드가 호출 종료되면 프레임을 자동 제거된다.
- 메서드를 위한 공간은 모든 쓰레드가 공유하면 안된다 -> 너무 많으니까


메모리 관리를 따로 하지 않아도 되지만 -> 메모리 번지를 찾아볼 수가 없다!

## null과 NullPointerExeption(NPE)

heap 영역에 어쩌구 저쩌구 이런식으로 나옴

- null -> 객체가 아무것도 가르키고 있지 않은 상태
- 힙 영역의 객체를 가르키고 있지 않은 상태(null)인데 가르키려고 할 때 -> NPE


```java title:"객체 참조의 과정"
String s4 = new String("여행");  
s4 = null;  
// 변수 s4를 만들고 -> "여행"객체를 만들고 -> =에 의해서 생성된 "여행"의 주소값을 s4에 대입  
// 이제 new String("여행")은 가비지 컬렉터의 수거 대상이 된다.  
  
String s5 = new String("취미");  
String s6 = s5; // pass by reference  
// namespace의 s5가 생기고 -> 새로운 객체 "취미" 생기고 -> "취미"객체의 저장공간 주소를 s5를 가져옴  
// namespace에 s6가 생기고 -> s5가 가지고 있는 주솟값을 s6에 대입
```
