# Static 정적멤버 사용

- 클래스가 메모리로 로딩되면 정적멤버를 바로 사용할 수 있음 -> 이미 메모리에 등록이 '한번만' 됨.
	- 즉 클래스에 static으로 선언을 하면 new 키워드로 객체를 선언하지 않아도 직접 사용이 가능함.
	- static으로 선언된 부분은 공유되는 영역(프로그램 종료 전까지 메모리에 남아있음)
		- 따라서 너무 많이 쓰면 메모리를 많이 먹고 있어서 성능이 저하됨.


## 정적블록

```java title:"static블럭 사용"
package ch05;  
  
public class StaticNonClass {  
  
    static int staticBlockTest;  
  
    static {  
        staticBlockTest = 123456;  
    }  
  
    int nonStaticInt;  
}
```


- static영역에서 non-static 영역은 걸드릴수가 없다!
- 바이트 코드가 JVM이 읽어들이는 시점에 static 영역은 이미 메모리에 로딩이 되어 이음
	- 그러나 non-static들은 new 키워드가 없으면 메모리에 로딩이 될 수 없음

# 메소드 @Override

- 부모의 메서드를 재정의 하는 것 -> 부모 메소드는 숨겨지고, 자식 메소드가 우선적으로 사용
- 선언부는 동일해야함 -> return 타입, 매개변수

#### 🙉 작년 시험문제
Q. 다음 중 메소드 재정의 관련 올바르지 않은 것은?
1. 상속 받은 메소드의 접근 제한자를 더 제한할 수 없다(O)
2. 리턴 타입이 Primitive인 경우 동일해야한다. (O) -> Primitive든 아니든 그냥 동일해야함
3. <font color="#ff0000">상속받은 메소드의 발생 예외는 동일해야한다 (X)</font> -> 더 넓은 예외처리는 가능하다(더 제한은 X)
4. @Override를 사용하는게 코드 유지 보수에 좋다. (O)


- 자식 클래스의 메서드에서 Override한 메서드 안에서 super.Method()와 같은 형태로 부모의 메서드를 실행시킬 수 있다. 

- final은 상속해서 Override가 불가능함!

# 상속 타입 캐스팅 -> 다형성으로의 길..

- is a 관계 
	- 부모 : 투수
	- 류현진, 박찬호
	- ex)  투수 a = new 박찬호(); -> <span style="background:#fff88f">자동으로 박찬호 타입이 투수 타입으로 형변환 된다.</span>
	- ex2) 투수 b = new 류현진();
	- <font color="#ff0000">부모타입에 선언된 필드멤버와 메서드만 사용가능하다.. -> 강제 타입 변환을 해야지 자식 클래스의 메서드를 사용할 수 있음</font>

```java title:"상속 타입 캐스팅"
package ch7;  
  
public class CastTest {  
  
    public static void main(String[] args) {  
        CastSuper cs = new CastChild(); // 자동형변환, 다형성의 기초  
        cs.printSuper(); // 이거 하나만 가능함  
  
        CastChild ch = (CastChild) cs;  
        ch.printChile();  
    } 
}
```

- 부모타입 하나로 여러가지 자식타입을 담을 수 있다는 점에서 '다형성'을 구현할 수 있음
