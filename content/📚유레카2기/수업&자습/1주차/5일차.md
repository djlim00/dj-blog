```java title:"Date함수를 이용한 파일 이름 업데이트"
package ch12;  
  
import java.text.SimpleDateFormat;  
import java.util.Date;  
  
public class DataformatTest {  
    public static void main(String[] args) {  
  
        Date date = new Date();  
  
        SimpleDateFormat simpleDateFormat = new SimpleDateFormat("yyyy-MM-dd");  
        String dateStr = simpleDateFormat.format(date);  
        System.out.println(dateStr);  
  
        SimpleDateFormat simpleDateFormat1 = new SimpleDateFormat("_HHmmss");  
        String timeStr = simpleDateFormat1.format(date);  
        System.out.println(timeStr);  
  
        String filename = "uploadfile";  
        String saveFilename = filename + dateStr + timeStr;  
        System.out.println(saveFilename);  
  
    }  
}
```

# 1. 어노테이션

- 어노테이션 정의 시 @AnnotationName을 언제까지 유지할지 지정
- 어노테이션 유지 정책은 RetentionPolicy 열거 상수로 정의
	- 메모리에 로딩되는 시점에 필요한 정보를 어노테이션 -> 컴파일 후에 사라짐 -> 메모리에 로딩되면서 제거됨
	- 실행되는 와중에도 적용되는 어노테이션 -> 실행 후에도 유지

- 개발자가 직접 어노테이션을 만들면, 해당 어노테이션이 어디서 사용되는 어노테이션인지 지정

# 2. 제네릭(Generic)

### 제네릭이란?
: <font color="#ffff00">결정되지 않은 타입을 파라미터로 처리</font>하고 실제 사용할 때 파라미터에 구체적인 타입으로 대체시키는 기능
- < T >는 T가 타입 파라미터임을 뜻하는 기호. 타입이 필요한 자리에 T를 사용할 수 있음을 알려줌

```java
public class Box <T>{
	public T content;
}
```
- T에 따라서 박스의 내용물(content)가 달라짐
ex) List< String> list;
여기서 String 자리는 T로 String, Integer, Character, Box, Coffee...etc 등등 여러가지 타입이 올 수 있음.

- 여기서 이제 T는 데이터 타입 -> 즉 클래스다.
