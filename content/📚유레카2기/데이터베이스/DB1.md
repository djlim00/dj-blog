
# MySQL : 오픈소스였던 것..
___
- **오픈소스** : 개발에 누구나 참여가 가능하다, 무료사용 가능(MySQL)
- **프리웨어** : 개발에 참여X,  무료사용 가능(JAVA)

- MySQL은 호주에 중심을 둔 오픈소스였다.
- 그러다보니 같은 기능을 가진 함수의 이름이 굉장히 많다.(많은 사람들이 참여하면서 생긴 현상)
- 이걸 Oracle이 사서 더 이상 오픈소스가 아니다.
- <span style="background:#fff88f">MySQL은 5.xx 버전은 그냥 Oracle과 상관없는 무료버전임!</span>
- 웹 포스팅을 하던가 하면 MySQL 5.xx 버전을 무료로 제공한다. -> 8.xx은 Community버전은 학생들 대상으로 무료다!
- MySQL 5.xx 버전으로 부터 새롭게 만든 DB -> Maria DB
- MySQL은 전반적으로 좀 느리다,,(오라클이 훨씬 빠르다 근데 SAP가 그거보다도 더 빠르다!)


# DBMS의 역사
___
- 1960년대 부터 만들기 시작했음 -> 만드는 회사마다 SQL이 다른 문제가 생김(명령어가 다름)
	- 이를 해결하기 위해서 ANSI SQL이 나옴(America National Standard Institute) -> ISO SQL
	- DBMS 명령어 통합의 시초
- <span style="background:#fff88f">DBMS에 특화된 펑션을 쓰지 마라 -> 다른 DB에도 쓸 수 있게</span>




# 데이터 독립성
___
**논리적 독립성** : 외부 단계와 개념 단게 사이의 독립성
- 개념 스키마가 변경되어도 외부 스키마에는 여향을 미치지 않도록 지원
- 논리적 구조가 변경되어도 응용 프로그램에는 영향이 없도록 하는 개념
ex) Exel의 테이블 구조는 항상 유지되어야 한다!
	- <span style="background:#fff88f">Column이 하나 추가 된다고 외부에서 읽어오는 방식이 달라져서는 안된다</span>

**물리적 데이터 독립성** : 개념 단계와 내부 단계 사이의 독립성
- 저장 장치 구조 변경과 같이 내부 스키마가 변경되어도 개념 스키마에 영향을 미치지 않도록 지원함
ex) <span style="background:#fff88f">FAT 방식의 HDD에서 저장되어있음 -> NTFS 방식의 HDD로 복구 -> 데이터가 깨지면 안된다</span>
