# 무결성 제약조건
___
### Key
- <mark style="background: #FFF3A3A6;">무엇인가를 유일하게 식별한다는 의미</mark>
- <span style="background:#fff88f">관계 데이터베이스에서 키는 릴레이션에서 특정 튜플을 식별할 때 사용하는 속성 혹은 속성의 집합</span>

#### Forein key(FK)
-> 해당 테이블이 아닌 바깎 테이블을 참조하는 키
- null값, 중복값 허용
#### Super key
-> 튜플을 유일하게 식별할 수 있는 하나의 속성 또는 속성의 집합
ex) id, 고객번호, 핸드폰 번호, 주민번호,{주민번호, 이름, id}, {id, 이름}...etc
-> 너무 완벽한 튜플은 PK로 쓰면 안된다 -> 그래서 이제는 {생년월일+주민번호첫자리}만 받는다.(근데 이러면 너무 많아서 PK로 못쓴다)


#### Candidate Key
-> 튜플을 유일하게 식별할 수 있는 하나의 속성 또는 속성의  <span style="background:#fff88f">최소의 집합</span>


#### Primary Key
-> Candidate key 중 하나
- 변경이 있으면 안됨 -> 전화번호는 안됨!
- NOT NULL
- 최대한 적은 수의 속성
- 향후 키를 사용하는데 문제 발생 소지가 없어야함

<font color="#d83931">즉 Super key 중 크기가 최소인 Candidate Key 중 하나를 고른 Primary Key</font>




### 데이터 무결성
- 데이터베ㅐ이스에 저장된 데이터의 일관성과 정확성을 지키는 것
- 일관성과 정확성을 바탕으로 구축된 데이터베이스가 계속해서 무결성을 유지하려면 튜플의 CRID시 데이터의 제약조건을 준수해야함!
