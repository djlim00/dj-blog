
- 계속해서 사용자의 질문 query와 vectorstore에서 유사도 비교를 해도 아무결과가 나오지 않았음..
- 그 이유가 뭘까?

🙉 LangChain의 PGVector는 항상  **langchain_pg_collection** 과 **langchain_pg_embedding** 테이블만을 참조해서 검색하기 때문이다..!

- 두 테이블은 우리가 langchain에서 PGVector를 사용하는 순간 자동으로 생성된다.


## 유사도 검색과정
1. 우리가 collection_name="plan_vector"라고 지정을 하면 langchain은 langchain_pg_collection 테이블에서 "plan_vector"를 찾아서 collection_id(uuid)를 가져와서
2. langchain_pg_embedding 테이블에서 해당 collection_id(uuid)를 갖는 문서들을 임베딩 비교대상으로 삼는다.

# 수습을 하기

- 그럼 우리가 plan_vectort에 잔뜩 넣어놓은 데이터가 사실은 langchain_pg_embedding 테이블에 있어야 한다.
- 기존의 데이터를 langchain_pg_embedding에 형식에 맞춰서 마이그레이션만 하면된다.

```sql title:"기존의 데이터가 다 옮기기"
// pgcrypto가 활성화 되어있는지 확인하기
CREATE EXTENSION IF NOT EXISTS pgcrypto; 


// 그 이후에 plan_vector의 정보를 옮기는 sql
INSERT INTO langchain_pg_embedding (uuid, collection_id, embedding, document, cmetadata)  
SELECT  
    gen_random_uuid(),  -- 각 행마다 새로운 UUID    'f31bb763-c90a-4b78-9775-3e0a5da2f7d1'::uuid,  -- collection_id  
    embedding::vector,  -- plan_vector 테이블의 embedding    description,        -- 요금제 설명 텍스트  
    jsonb_build_object('plan_id', plan_id)  -- 메타데이터: plan_id만 넣음  
FROM  
    plan_vector;  

// custom_id에 plan-{id}값으로 넣는 sql
UPDATE langchain_pg_embedding  
SET custom_id = 'plan-' || (cmetadata::json ->> 'plan_id')  
WHERE custom_id IS NULL;
```
