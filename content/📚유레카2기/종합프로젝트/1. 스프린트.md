아주 좋은 마인드셋이에요. 작고 짧은 프로젝트라도 **애자일(Agile)** 방식으로 운영하면 팀워크와 생산성이 훨씬 올라갑니다. 아래에 팀장으로서 꼭 알아야 할 **애자일 기본 개념**, **스크럼 역할 및 진행 방식**, 그리고 **실제 적용 예시**를 단계별로 정리해드릴게요.

---

## ✅ 1. 애자일이란?

- **변화에 유연하게 대응**하면서도,
    
- **짧은 주기(Iteration)** 안에 **작은 결과물**을 반복적으로 내놓는 방식
    
- 핵심: "계획 → 실행 → 피드백"을 짧고 빠르게 반복

---

## ✅ 2. 스프린트(Sprint)란?

- 일정 기간(보통 **1~2주**) 동안 **정해진 목표를 가지고 작업**하는 단위

- 예: "이번 주 스프린트 목표는 회원가입 + 로그인 완성"

| 요소        | 설명                         |
| --------- | -------------------------- |
| **기간**    | 1~~2주 (짧은 프로젝트면 3~~5일도 가능) |
| **산출물**   | 실행 가능한 결과물 (MVP 수준도 OK)    |
| **평가 방식** | 매 스프린트 후 회고(회고 회의) 진행      |

---

## ✅ 3. 스크럼(Scrum) 핵심 구성 요소

|구성 요소|역할 및 설명|
|---|---|
|**스크럼 마스터**|프로세스 가이드. 팀장이 겸할 수 있음|
|**스프린트 플래닝**|"이번 주에 무엇을 할까?" 정하는 회의|
|**데일리 스크럼**|매일 10~15분 회의. 진행 상황 공유|
|**스프린트 리뷰**|스프린트 결과물을 팀이 함께 리뷰|
|**스프린트 회고**|무엇을 개선할 수 있을까? 피드백 나누기|

---

## ✅ 4. 실전 적용 가이드 (팀장 입장)

### ⏱ 기획 페이즈 (Day 1~2)

- **1차 아이디어 수집**: 팀원들 자유 발산 (Notion 등 활용)
    
- **기능 MVP 선정**: 이번 프로젝트에서 꼭 필요한 핵심 기능만!
    
- **기능 리스트 작성** (예: 회원가입, 로그인, 게시글 등록)
    

### 🏁 구현 페이즈 (Day 3~n)

- **스프린트 1 (3~5일)**: 핵심 기능 구현 → 예: 로그인, 회원가입, DB 설계
    
- **스프린트 2 (3~5일)**: 나머지 기능 + UI 정리 + 테스트
    
- 매일 **데일리 스크럼 (10분)**: 각자 "어제 한 일 / 오늘 할 일 / 문제" 공유
    
- 스프린트 종료 후 **회고 + 리뷰**: "뭐가 잘 됐고, 뭐가 아쉬웠는지"
    

---

## ✅ 팀장으로서 해야 할 일 요약

|시점|해야 할 일|
|---|---|
|**기획 전**|애자일/스크럼 개념 공유 + 스프린트 단위 설정|
|**스프린트 시작**|작업 분배 + 스프린트 목표 명확화|
|**매일**|데일리 스크럼 주도 (시간 엄수!)|
|**중간중간**|진행률 확인 + 블로커(장애물) 제거 지원|
|**스프린트 종료**|리뷰 회의, 회고 회의 주도|

---

## ✅ 보너스: Notion 스프린트 관리 꿀팁

- `스프린트 1` 페이지 만들기 → 그 안에 `To Do`, `Doing`, `Done` 칸반 보드
    
- 각 태스크에 담당자, 기한, 우선순위 태그 달기
    
- 매일 회의 시 이 페이지 열고 진행 상황 확인하면 됩니다

## ✅ 업무 분배 실전 팁

### 🧠 1. 누구에게 어떤 Task를?

- 과거 경험/강점 기반으로 할당
    
    - 예: A는 DB 설계 잘함 → DB 구조 초안 작성
        
    - B는 REST API 잘 짬 → 로그인, 회원가입
        
- 혹은 각자 1~2가지 "해보고 싶은 것" 받아서 반영도 OK
    

### 📋 2. Notion에서 업무 분배 방법

- 칸반 보드의 각 카드(태스크)에서
    
    - `담당자` 속성 추가 → 멤버 선택
        
- 태스크 제목, 마감일, 스프린트, 우선순위도 같이 설정
    

### 🔄 3. 이후에는 자율 분배 가능

- 1~2회 스프린트 이후, 팀원들이 서로 익숙해지면
    
- 이후에는 **"담당자 자율 선택 + 팀장이 검토"** 방식으로 전환 가능
