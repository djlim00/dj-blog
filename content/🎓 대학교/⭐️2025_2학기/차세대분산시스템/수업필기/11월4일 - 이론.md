![[Pasted image 20251104092952.png]]

# 3. PaaS
![[Pasted image 20251104101724.png]]
- 개발환경 서비스를 제공한다.
	- low-level에 대한 고민 없이 서비스 개발에만 집중할 수 있도록
	- 프로그래밍 모델, 언어, 프레임워크를 지원
	- in-memory캐시에, 데이터 저장

![[Pasted image 20251108215611.png]]
- 어떤 특정한 문제를 효율적으로 해결하기
	- 거대한 데이터셋 프로세싱
	- 웹 서비스 저장
	- 워크 플로우 오케스레이션

![[Pasted image 20251108215947.png]]
- 다양한 프로그래밍 언어
- 소프트웨어 프레임웤,
- 지속적인 옵션

![[Pasted image 20251108220008.png]]
- 그냥 소프트웨어 애플리케이션이라고 생각하면 됨.

![[Pasted image 20251108220031.png]]
- 설정과 커스텀 가능
- HTTP, REST에 따른 통합적인 프로토콜을 따른다.
![[Pasted image 20251108220529.png]]
- 사실상 인터넷 서비스의 특성이라고 볼 수 있다.  -> 업데이트가 실시간으로 개발자에 의해 행해짐

ex) Gmail, google docs, MS Sharepoint



![[Pasted image 20251108220709.png]]
- 위의 내가지 방법론이 클라우드 컴퓨팅의 배포 모델들임


![[Pasted image 20251108220731.png]]
- public cloud : 대중에게 오픈된 클라우드 인프라
	- 학술적, 사업적, 정부 등등의 목적
- private cloud : 하나의 조직을 위한 클라우드 인프라(주로 비즈니스)
- community cloud : 특정한 커뮤니티의 클라우드
	- mission, 보안 사항, 정책..
- hybrid cloud : 두개 이상의 특정한 클라우드 인프라


![[Pasted image 20251108220930.png]]
