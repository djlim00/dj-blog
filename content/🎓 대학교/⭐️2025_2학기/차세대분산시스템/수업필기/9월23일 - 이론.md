# 📚 Virtualization History & Evolution

## 1960s ~ 1970s (Begin)

- 💰 **컴퓨팅 비용이 매우 비쌌음** → 자원 공유가 큰 이슈
- **가상화 시작 (1960s)**
    - 시스템 자원을 논리적으로 나누어 여러 애플리케이션에서 사용
    - **Mainframe Computers** 기반
- 🖥️ **IBM VM/370**
    - IBM 메인프레임용 VMM

## 1980s ~ 1990s (Lost Attention)

- 🖥️ **PC 시대 도래 (1980s)**    
- 저렴한 HW 보급 → 자원 공유 필요성 감소
- PC 하드웨어는 가상화 지원 부족
- 👉 가상화 연구 관심 축소    

## Late 1990s (Re-Attention)

- 🔥 **서버 증가 문제 (Server Proliferation Problem)**    
    - → **서버 통합(Server Consolidation)** 필요
    - 비용 효율적인 대규모 컴퓨팅으로 전환
- 🎓 **Stanford "Disco" 프로젝트 → VMware 설립 (1998)**    

## 1998 ~ 2003: VMware & Xen

### 🔹 1998~2002: VMware
- **Full-virtualization**
    - 수정되지 않은 OS 실행 가능
    - SW 기반 Binary Translation 활용
- 산업 및 학계에서 큰 성공

### 🔹 2003: Xen

- Cambridge Univ. 프로젝트 (Open Source)
- **Para-virtualization**
    - OS 수정 필요 → Near-native 성능
- 2003~ Xen 기반 연구 활성화
- 2007년 Citrix 인수

## 2005 ~ Now: Mainstream Virtualization

### 🔹 2005~2006: HW-assisted Virtualization

- x86 가상화 기능 도입
    - Intel VT-x, AMD-V
- 수정되지 않은 OS 실행 가능 (Near-native 성능)
- 현재는 **기본 기능화**
### 🔹 2006 ~ 현재

- **Cloud Computing (IaaS)**
- **Virtual Desktop Infrastructure (VDI)**
- **Mobile Virtualization**

# 📌 핵심 정리

1. **1960s**: 메인프레임 기반 가상화 시작 (IBM VM/370)    
2. **1980s~1990s**: PC 시대 → 가상화 관심 축소
3. **Late 1990s**: 서버 폭증 문제 해결 위해 VMware 등장
4. **2003**: Xen → 오픈소스, Paravirtualization
5. **2005~**: HW 지원 가상화 (Intel VT-x, AMD-V) → 클라우드, VDI, 모바일까지 확장

# 가상화의 장점

1. **여러개의 OS를 동시에 돌릴 수 있다**
	- <u>보안관련 모니터링과 테스팅에 좋음</u>
	- <u>유연한 테스팅과 디버깅</u>
2. **다양한 요구사항을 가지고 있는 소프트웨어 개발에 좋음**
3. **Fast and Easy Provisioning for Elastic Business**
	- Amazon을 생각하면 됨
4. **Live VM migration**
	- 서비스 도중에 VM을 이동할 수 있음.(스냅샷)
	- 로드 밸런싱, 고가용성, 업데이트 최적화, 재난 복구
