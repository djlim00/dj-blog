- 우선 JAR(Java ARchive)와 WAR(Web ARchive)에 대해서 알아보자

## 1. JAR 파일
### 장점
- **독립적인 JAVA 애플리케이션을 패키징**할 수 있고, 실행 가능한 JAR 파일로 만들어서 개발자가 쉽게 애플리케이션을 실행할 수 있음.
- 압축된 형식으로 클래스, 리소스, 라이브러리 등을 포함하기 때문에 배포와 전달이 용이
- Java 애플리케이션의 진입점인 main 메서드를 포함하고 있어서 **JVM에서 직접 실행**하기 때문에 **웹 컨테이너나 서버가 필요하지 않다**

### 단점
- 웹 애플리케이션을 개발에 있어서는 JSP나 서블릿 컨테이너에 대한 표준 기능을 사용하기 어려움
- 웹 애플리케이션에서 동적인 웹 컨텐츠 생성 및 관리가 제한적
#### 웹 애플리케이션보다는 독립적으로 실행 가능한 애플리케이션을 패키징하는 용도!

____
## 2. WAR 파일
### 장점
- **Java 웹 애플리케이션을 패키징하는 데 최적화**가 되어 있고, 웹 구성 요소인 JSP, 서블릿, 필터, 리스너 등과 웹 애플리케이션을 실행하기 위한 서블릿 컨테이너(웹 컨테이너)에서 필요한 설정 파일, 라이브러리, 리소스 등을 포함하여 **한번에 배포 및 실행**할 수 있음
- 웹 컨테이너(서블릿 컨테이너)를 통해 애플리케이션을 실행하므로 다양한 **웹 기능과 서버 환경**을 활용할 수 있다.

### 단점
- **웹 컨테이너나 서버에 종속적이기 때문에 특정한 웹 컨테이너 환경이 필요하다.**
- 배포 및 전달에 있어서 JAR 파일에 비해 크기가 크고 번거로울 수 있음.

## 5-1. 우선 Nginx란 무엇인가?

- 높은 성능과 안정성으로 널리 사용되고 있는 <span style="background:#fff88f">웹서버</span>

1. WAS의 부담을 줄여줌(동적인 작업이 많기 때문에 정적 자원 처리는 웹 서버에 맡김)
2. 보안 기능 제공. 웹 페이지에 대한 접근 제어가 가능하다.

	- 이벤트 기반, 멀티 프로세싱, 스레드 풀..etc

### 5-3. 프록시(Proxy)란?
프록시는 "대리"라는 의미를 갖고 있으며, 인터넷 접속 시 보안 문제 등으로 직접 통신이 어려운 경우 **중간에서 대신 통신을 수행하는 기능**을 말한다. 이런 역할을 하는 것을 **프록시 서버**라고 하며, 프록시는 크게 **포워드 프록시**와 **리버스 프록시**로 나뉜다.
### 5-4. 포워드 프록시 (Forward Proxy)
- <u>클라이언트와 인터넷(서버) 사이에 위치</u>
- 클라이언트가 요청한 정보를 **프록시가 대신 받아 서버에 전달**
- 서버의 응답도 **프록시가 받아 클라이언트에게 전달**
#### 포워드 프록시의 장점:
- 클라이언트의 IP 주소가 서버에 노출되지 않아 **익명성** 확보
- **접근 통제 가능** (특정 IP/도메인 차단 등)
- **보안 강화**, **위치 은폐**
- **캐싱**을 통해 미디어 스트리밍 등 **속도 개선**
![[Pasted image 20250629100743.png]]


### 5-5. 리버스 프록시(Reverse Proxy)
- <u>인터넷(서버)와 백엔드 사이에 위치</u>
- WAS로 사용되고 있는 서버가 여러대 있을 때, 사용자의 요청을 서버들에게 분배(로드 밸런싱)
	
>즉, 클라이언트는 “**서버한테 직접 가는 게 아니라, 리버스 프록시 서버한테 요청 → 프록시가 내부 서버에게 전달 → 응답도 대신 받아서 전달”**

![[Pasted image 20250629101117.png]]

- 이걸 통해서 :8080을 붙이지 않게 할 수 있음!!!

# 6. 실제로 Nginx 설치 및 리버스 프록시 설정하기

```text title:"EC2에 Nginx 설치"
sudo apt update
sudo apt install nginx
```

- 그럼 이제 나의 EC2 컴퓨터에서는 Nginx가 설치되었다

```text title:"리버스 프록시 설정을 위해 nano 편집기를 이용해서 설정 파일 열기"
sudo nano /etc/nginx/sites-available/default
```

- 그럼 아래와 같은 화면이 나올 것이다.

![[Pasted image 20250629102221.png]]
- 이건 nano 편집기의 화면인데, 여기서 `server{...}` 부분을 우리의 목적에 맞게 수정하면 된다.

```python title:"수정코드"
server {
    listen 80 default_server;
    listen [::]:80 default_server;

    server_name _;

    location / {
        proxy_pass http://localhost:8080;  # 8080에서 돌고 있는 서버로 프록시
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```
- 보면 알겠다시피 80포트(HTTTP 포트)로 들어오는 요청을 listen 하면
	- <span style="background:#fff88f">localhost:8080(EC2에서 JAR 파일로 돌아가고 있는 Spring 서버)로 리다이렉트 하게 한다.</span>
