## 2. 카카오 로그인 과정 - 로그인 자체
- 위의 세팅을 마쳤다면 카카오 로그인 과정을 보자.

![[Pasted image 20250502103132.png]]
그림을 보면 살짝 이해가 잘 안간다.

1. 카카오 로그인 - 인가 코드 발급 요청
	- 이 단계에서 아래의 url에 위에서 언급한 <span style="background:#b1ffff">{REST_API_KEY}</span>와 <span style="background:#fdbfff">{Redirect_URI}</span>가 쓰인다
	- 우리가 `https://kauth.kakao.com/oauth/authorize?response_type=code&client_id={REST_API_KEY}&redirect_uri={Redirect_URI}` 로 요청을 보내면..
	- `http://localhost:8080/auth/kakao/callback?code=RnMRuk6pa0YP-LpC_QmD_o59VcqRGiXeel5xWl1bJRZD1ZgjFEb9xAAAAAQKFxJVAAABlo6_U2ltZc76WqiBKA` 여기 보면 `?code={인가코드}` 가 온다.


2.  그럼 우리가 받아온 <span style="background:#d3f8b6">{인가코드}</span>로 다시 카카오 서버에 요청을 해야한다
	- <mark style="background: #FFF3A3A6;">왜? -> 진짜 카카오 로그인을 하기 위한 엑세스 토큰(Access Token)을 받아오기 위해서</mark>


3. 그럼 이제 카카오 서버에 엑세스 토큰을 요청하자 - 엑세스 토큰이 발급됐다는 것은 카카오 소셜 로그인이 성공했다는 것![[Pasted image 20250502111207.png]]
	- 저기 보면 Body에서 code에 우리가 위에서 받은 <span style="background:#affad1">{인가코드}</span>를 넣어서 요청하면 된다.
	- 그럼 아래에 보면 <span style="background:#ff4d4f">{access_token}</span> 이 발급되는 것을 볼 수 있다.


#### 여기까지 했으면 일단 로그인자체는 성공적으로 된 것이다 😎

## 정리
- 이 모든 과정에서 각자의 프로젝트에 맞게 코드의 흐름을 수정할 필요가 있다.
- 이 글에서의 과정은 일반적인 과정을 언급한 것이며 크게 수정될 일은 없을 것 같다.

- 웬만하면 이렇게 카카오 로그인을 성공하면 카카오 엑세스 토큰을 사용하지 않고, 각각의 서비스에서 독자적으로 JWT를 사용해서 Access Token과 Refresh토큰을 사용해서 관리한다.
	- 즉 카카오 로그인에서 발급받은 Access Token을 계속 사용하지는 않는다.
- JWT관련한 로직 설명은 나중에 글을 작성하겠다..
