## 안드로이드 앱 방식(앱)

- 카카오 엑세스 토큰을 백엔드에서 받을 필요가 없다.
- 클라이언트에서 카카오 엑세스 토큰을 받아서 우리한테 쏴준다.
- 그걸 우리는 그냥 받아와서, 클라로 부터 받은 카카오 엑세스 토큰을 카카오 서버에 쏴서 회원가입에 필요한 사용자 정보를 받아온다.

### 1. 카카오 인가코드를 받는  URL
https://kauth.kakao.com/oauth/authorize?response_type=code&client_id=682ac9c72ed183acc3c2b6bdcc25e1d4&redirect_uri=http://localhost:8080/auth/kakao/callback

- 로그인이 완료된 후, 브라우저 주소창에 나오는 인가코드를 복사
```kotlin
http:// ~~~~ ?code=xxxxx 
-> code값 복사
```

### 2. Postman : 카카오 엑세스 토큰 획득
```kotlin
1. Body의 code 부분에 위에서 복사한 값 넣기
2. 응답으로 오는 access_token 복사
```

### 3.  Postman : 로그인
```kotlin
1. Header에 Authorization의 Bearer 뒷 부분에 위에서 복사한 access_token 붙여넣기
```
- 기존 회원 : userId, accessToken, refreshToken, <mark style="background: #FFF3A3A6;">isRegister: true</mark>
- 신규 회원 : userId, accessToken, refreshToken, <span style="background:#b1ffff">isRegister: false</span>

## 로그아웃 구현

1. 현재 우리는 카카오에서 받아오는 <u>카카오 access token</u>은 소셜 로그인을 하는 시점에 일회성으로만 사용한다.
2. 카카오 로그인에 성공한 이후에는 모든 요청에 대한 access token은 자체적으로 발급한 jwt access token을 사용함.
3. 사용자가 로그아웃을 하는 경우,  새로운 카카오 로그인을 하도록 유도해야함.

![[Pasted image 20250423151704.png]]
4.  로그아웃을 하게 되면
	- 프론트에서 초기 로그인 화면으로 리다이렉트
	- 프론트에서 로컬 저장소에 가지고 있던 jwt access token, refresh token을 만료시킴.
