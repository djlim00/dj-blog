### 3.2 기능
- 검색 -> **엘라스틱 서치**
- **채팅 만들기**
	- 단일 서버 인메모리 방식 -> **오토 스케일링**이 되면 서버가 달라지면서 채팅 못함
	- **Redis Pub/Sub 기능**을 활용해서 통신 불가능 문제 해결
	- 채팅 지연율 증가 -> 채팅 서버와 비즈니스 서버 분리
- <mark style="background: #BBFABBA6;">MCP를 사용해서 필요한 기능을 찾아서 사용하는 REASONING LLM</mark> 
	- 이거 좀 히트인 듯?
- 입찰 비교 로직 - **비관적 락 vs Redis Lua 스크립트**
	- 비관적 락 : 너무 오래 걸림
	- Redis Lua 스크립트 - 락 없이 한번에 원자적으로 스크립트를 실행시킴
		- Redis뿐만 아니라 RDB에서 실패했을 경우 처리 로직도 있음.
- **배치 처리 성능 병목(시세 통계, 쿠폰 사용여부)**
	- 조회 속도 개선 -> 복합 인덱스 추가, JPA기반 Reader가 느려서 JDBC로 바꿈
	- 쓰기 성능 개선 -> JPA기반 Writer -> JDBC기반 Writer
- **다중 인스턴스로 인한 알림 안오는 문제**
	- 단일 서버 X
	- Redis Pub/Sub -> Redis 이벤트를 구독하고 거기서 알림을 받아옴.
	- Kafka는 메시지를 영속적으로 보관해서 Redis Pub/SUb를 통해서 모든 인스턴스에 메시지를 보냄 -> 해당 메시지에 맞는 인스턴스만 응답을 보냄
- 단순 SSE쓰려다가 -> 오토 스케일링 때문에 Redis Stream 적용
- 실시간 사용자 로그 수집
	-  Redis Stream를 통해서 행동에 따라서 action_log 수집(비동기) -> Airflow로 분석/요약 -> 테이블에 저장
	- 소비 패턴 분석해서 RAG에 넣음. -> 패턴 파악해서 벡터로 만들어서 코사인 유사도 높은 것들 거리 기반으로 추천해줌
- <mark style="background: #BBFABBA6;">AirFlow는 뭐여?</mark>
