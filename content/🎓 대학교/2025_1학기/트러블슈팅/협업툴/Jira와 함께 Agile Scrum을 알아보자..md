

# 주요 용어 정리

| ==용어==     | ==설명==                                                                     |
| ---------- | -------------------------------------------------------------------------- |
| 애자일(Agile) | '신속 개발 방법론'들을 통칭하는 단어. 대표적으로 스크럼, 칸반, XP, 린 등이 있음                          |
| 스크럼        | 애자일 방법론 중의 하나.<br>팀원과 조직, 개인이 문제를 해결하기 위해 취사 선택하는 행동방식을 이용해 만들어짐.          |
| 프로덕트       | 프로젝트에서 구현하고자 하는 목표 대상을 **프로덕트**로 정의                                        |
| **백로그**    | 프로덕트와 관련된 이해관계자의 요구사항, **수행이 필요한 작업**                                      |
| 스프린트       | 일정기간 내에 정해진 백로그 아이템을 구현해내기 위해 수행되는 스크럼 팀 이벤트                               |
| 데일리 스크럼    | 스크럼 팀이 매일 15분 정도 진행하는, 동기화를 위한 회의                                          |
| 이슈 유형      | 백로그 아이템을 기술한 것들로, 이슈 유형에 따라 다음과 같이 구분<br>- Epic, Story, Task, Subtask, Bug |

___
# 스크럼 수행 프로세스

> 1. **프로젝트** 생성
> 2. **로드맵** 생성
> 3. **백로그** 등록
> 4. **스프린트** 계획 수립
> 5. **스프린트** 수행
> (4-5의 과정을 목표 달성까지 반복)
#### 즉 스크럼 수행 프로세스는 "어떤 프로젝트를 완료"하기 위해서 "로드맵을 그리고", 이를 위해 필요한 "백로그를 산출"한 다음 실제 작업 진행을 위한 "스프린트를 수행"하는 개념.
