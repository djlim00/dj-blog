1. 다음의 <font color="#ff0000">UML Diagram</font> 중, 시스템과 외부 환경 간의 Interactions을 시간의 순서대로 보여줄 수 있는 것을 모두 고르세요.
Choose all of the following UML Diagram that can show the interactions between the system
and the external environment in chronological order.
(2.5점, 복수선택 가능)
1) <span style="background:#d4b106">Activity Diagram</span>
2) Class Diagram
3) State Diagram
4) Use Case Diagram
5) <span style="background:#d4b106">Sequence Diagram</span>

- UML이란?
  : 소프트웨어를 개발하는 과정에서 산출물의 명세화, 시각화, 문서화 할 때 사용하는 모델링 언어로서 하나의 시스템을 표현하기 위한 표준적인 방법을 제공하기 위해 만들어짐.\
이 답변은 업로드한 PDF 파일을 기반으로 하고 있으며, UML 다이어그램에 대한 설명은 **4장**에서 다뤄지고 있습니다 (Software Engineering, 4장, 155~156 페이지)​(Software Engineering (2…).

- **Class Diagram**: 시스템의 클래스, 속성, 연관 관계를 표현하며, 객체 지향 설계의 기초를 형성합니다.
- **Use Case Diagram**: 시스템 사용자가 목표를 달성하기 위해 수행할 수 있는 상호작용을 정의하여, 사용자와 시스템 간의 상호작용을 설명합니다.
- <span style="background:#d4b106">Sequence Diagram</span>: 객체 간의 상호작용을 시간 순서대로 보여주며, 특정 시나리오에서 메시지가 어떻게 주고받는지를 설명합니다.
- **Activity Diagram**: 시스템 또는 객체의 활동 흐름을 설명하며, 조건부 흐름이나 분기와 같은 로직을 포함합니다.
- **State Diagram**: 객체의 상태 변화와 상태 간 전이를 설명하며, 객체의 생명 주기를 시각화합니다.
- **Component Diagram**: 시스템의 물리적 컴포넌트와 그들 간의 관계를 설명하여, 시스템의 구조적 구성을 나타냅니다.
- **Deployment Diagram**: 소프트웨어가 물리적 하드웨어에 배치되는 방식을 설명하며, 하드웨어 노드 간의 상호작용을 보여줍니다.

4. 다음의 <font color="#ff0000">SDLC Models 중, 유지보수</font>(Maintenance) 측면에서 <font color="#ff0000">가장 취약한 것</font>은 무엇인가요?
Which of the following SDLC models is the most vulnerable (week) in terms of maintenance?
(2점, 복수선택 가능)
1) Incremental
<span style="background:#d4b106">2)</span> <span style="background:#d4b106">Waterfall</span>
3) CBD
4) Evolutionary
5) Agile/Iterative

- Incremental : 점진적으로 개발하여 기능을 추가하는 방식, 각각의 증분이 된 이후에도 수정 및 유지보수가 가능
- Waterfall : <span style="background:#d4b106">각 개발과정이 완료가 되면 다음 단계로 넘어감 -> 다음 단계로 가면 전에 완료한 단계에 대한 유지보수가 어려움</span>
- CBD(Component-Based Development) : "재사용 가능한 컴포넌트"를 기반으로 시스템을 개발하기 때문에 유지보수를 하게 되면 각각의 컴포넌트에 대한 수정만 하면 되서 유지보수에 유리함
- Evolutionary : 소프트웨어가 점진적으로 진화하는 방식으로, 시스템이 계속해서 변화하는 요구사항에 맞춰 **유연하게 발전**함. 유지보수도 이 과정에서 자연스럽게 가능함.
- Agile/Iterative : **반복적**으로 개발되며, 고객의 요구사항에 따라 지속적으로 수정하고 개선할 수 있어 **유지보수에 매우 유리**합니다. 특히 <font color="#ffff00">Agile 방식에서는 빠르게 요구사항을 반영</font>하고, 지속적인 피드백을 통해 시스템을 개선하므로, 유지보수 문제가 적습니다.
