### 1. 위키화 (핵심)

#### 1.1 파일 태깅 시스템

- **필수 태그**: 연도, 회사, 업무 유형(회계감사/세무조정/가치평가/용역/컨설팅)
- **선택 태그**: 세부 업무(법인세/소득세/상속세), 수임처, 관련 인물
- ingest 시 LLM이 파일 내용을 분석해 **태그 자동 제안**, 사용자 확인 후 확정
- 한 파일에 다중 태그 허용 (A사 회계감사+용역 동시 진행 케이스 대응)

#### 1.2 통합검색

- **태그 필터**: 다중 태그 AND/OR 조합
- **자연어 검색**: 위키 페이지 본문 + 원본 파일 본문까지 검색 (Ctrl+F 한계 극복)
- **검색 결과 표시**: 위키 페이지 / 원본 파일 / 회사 대시보드 구분 표시
- 파일명에 의존하지 않는 검색 (인터뷰에서 명시된 페인포인트)

#### 1.3 출처 명시 (필수)

- 위키의 모든 답변·요약·추론에 **원본 파일 링크와 해당 페이지/시트** 표기
- "AI가 잘못된 정보 가져온다"는 인터뷰 우려에 대한 직접 대응
- 출처 없는 LLM 추론은 별도 표시 ("LLM 추정" 라벨)

---

### 2. 회사별 대시보드 (영구조서)

#### 2.1 정적 정보 영역

- 회사 기본정보: 업종, 결산월, 사업자번호, 대표자
- 주요 주주 현황 (현재 시점)

#### 2.2 시계열 추세 영역

- **재무 추세 차트**: 매출, 영업이익, 차입금, 자본금의 연도별 변화
- **주주 분포 변화 이력**: 연도별 지분율 변동 테이블
- 자동 추출 또는 수동 입력 모두 지원

#### 2.3 업무 이력 타임라인

- 해당 회사에 수행한 모든 업무를 시간순 표시 (감사/세무/가치평가/용역)
- 각 업무 항목 클릭 시 관련 위키 페이지·원본 파일로 이동
- "A사 폴더 vs 용역 폴더" 분류 문제 우회

#### 2.4 특이사항 누적

- 매년 ingest 시 LLM이 자동 추출 (예: "2025년 차입금 30% 감소", "주요주주 변경")
- 사용자가 수동 메모 추가 가능
- 영구조서의 "히스토리 메모" 디지털 버전

#### 2.5 빠른 진입점

- 관련 위키 페이지 링크
- 원본 파일 링크
- 다음 예정 업무 (결산일/신고기한)

---

### 3. 위키 관리 대시보드

#### 3.1 ingest 현황

- 최근 추가된 원본 파일
- 최근 업데이트된 위키 페이지
- 처리 대기 중인 파일

#### 3.2 위키 건강 상태 (Lint)

- 미연결 페이지(orphan) 목록
- 출처 누락 페이지
- 오래된(stale) 페이지 — 마지막 업데이트 시점 기준
- 태그 불일치·중복 탐지

#### 3.3 통계

- 회사 수, 업무 유형별 건수, 연도별 자료 분포

---

### 4. 위키 활용 기능

#### 4.1 유사 케이스 검색·추천

- 새 업무 시작 시 위키에서 **유사 과거 사례 자동 추천**
- 추천 기준: 업종, 업무 유형, 회사 규모, 평가 방식 등
- 인터뷰의 "A기업 보고서 토대로 B기업 보고서 작성" 워크플로우 시스템화
- 추천 결과에서 바로 템플릿으로 활용 가능

#### 4.2 회사별 히스토리 자동 누적

- 매년 ingest 시 LLM이 회사별 변화 포인트 자동 추출
- 회사 대시보드의 "특이사항 누적" 영역과 연동
- 누적된 히스토리는 다음 감사·세무조정 시 자동 참조

---

### 5. 부가 기능

#### 5.1 한글(hwp) 파일 처리

- ingest 파이프라인에 hwp → 텍스트/마크다운 변환 모듈 포함
- 관공서 자료 처리를 위한 필수 기능

#### 5.2 문서 호환성 보조

- 위키 답변·표·차트를 한글/워드에 깨지지 않게 export
- 엑셀 표를 한글에 붙일 때의 호환 문제 보조
- 인터뷰의 ★ 표시 페인포인트 직접 대응

#### 5.3 법인세 세무조정 체크리스트

- 회사별 세무조정 작업 시점에 **적용 가능한 개정/공제 항목을 체크리스트로 제시**
- 외부 세법 데이터(국세청·국회 의안정보시스템) 연동
- 단순 알림이 아니라 **작업 시점 기반 체크리스트** 형태
- 회사 업종·규모·과거 적용 이력을 고려한 맞춤 항목 제시
