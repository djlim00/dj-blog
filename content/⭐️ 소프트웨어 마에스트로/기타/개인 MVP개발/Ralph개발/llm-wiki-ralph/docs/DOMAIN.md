# DOMAIN.md - 회계 도메인 지식

이 문서는 LLM이 코드를 짜거나 위키를 작성할 때 참조하는 회계 실무
배경지식입니다. 인터뷰 자료(`docs/INTERVIEW.md`)를 요약·재구성한
것입니다.

---

## 사용자

회계법인·세무사 사무실에서 일하는 회계사. 주로 1인 또는 소규모
사무실 환경. 큰 회계법인(삼일·삼정·한영·안진)은 자체 AI 툴이 있지만,
개인 사무실은 상용화된 도구가 없다.

## 주요 업무

### 1. 회계감사
회사의 재무제표가 회계기준에 맞게 작성되었는지 검증. 매년 반복.
- 입력: 회사가 제공하는 회계 자료(더존·위하고에서 export한 엑셀,
  자체 ERP 자료).
- 산출: 감사보고서 + 조서(작업 근거).
- 핵심: 매년 동일 회사를 반복 감사하므로 **연도별 누적 히스토리**가
  중요.

### 2. 세무조정 (법인세 신고)
회계상 이익을 세무상 이익으로 변환해 법인세를 계산.
- 이슈: 세법이 매년 개정됨. 세액공제·감면 항목 적용 여부가 해마다
  달라질 수 있음.
- 한시적 혜택(예: 3년간 세액공제)을 놓치면 손해.
- 더존이 알림을 주지만 누락·미반영 가능성 있음.

### 3. 가치평가 (주식가치평가, 컨설팅)
- 입력: 회사 재무 + 내부 사정 + 시장 상황.
- 작업 흐름: 비슷한 과거 케이스(A기업 보고서) → 변수만 바꿔 새 케이스
  (B기업) 작성. **유사 케이스 검색**이 핵심 페인포인트.

### 4. 용역
범위가 다양함. 회계감사 받는 회사에 추가 용역도 흔하다 → 폴더 분류
충돌의 주범.

## 자료 종류

- **재무제표**: 재무상태표, 손익계산서, 자본변동표, 현금흐름표.
- **장부**: 더존·위하고에서 export. 큰 회사는 자체 ERP.
- **조서**: 감사·평가 작업의 근거 문서. 워드·엑셀 혼합.
- **영구조서**: 회사별 누적 현황 기록(업종, 주주, 차입 추세 등).
  매년 갱신. **이 시스템의 회사 대시보드가 영구조서의 디지털 버전**.
- **보고서**: 워드 또는 PPT로 출력. 클라이언트 제출용.
- **한글(hwp)**: 관공서 자료. 호환성 문제로 페인포인트 1순위.

## 핵심 용어

| 용어 | 설명 |
|------|------|
| 조서 | 회계 작업의 근거를 정리한 내부 문서 |
| 영구조서 | 회사별로 매년 갱신하는 누적 현황 메모 |
| 세무조정 | 회계상 이익 → 세무상 이익 변환 작업 |
| 외감 | 외부감사 대상 회사. 재무제표·감사보고서가 금감원 전자공시에 공개됨 |
| 수임처 | 회계사가 업무를 맡은 고객사 |
| 결산월 | 회사의 회계연도 종료월 (대부분 12월, 일부는 3월·6월 등) |
| 차입 | 회사가 빌린 돈. 추세 추적이 중요 |
| 주주 분포 | 누가 얼마나 지분을 가졌는지. 매년 변동 추적 |

## 페인포인트 (우선순위)

1. **자료 정리 혼란**: 연도별로 둘지 회사별로 둘지 케이스마다 달라
   오래되면 못 찾음. → **태그 기반 다차원 분류**로 해결.
2. **파일명 검색의 한계**: Ctrl+F로 파일명만 검색 가능, 내용 검색
   불가. → **본문 통합 검색**.
3. **영구조서 갱신의 번거로움**: 회사별 히스토리를 매년 수동 누적.
   → **자동 추출 + 수동 보완**.
4. **유사 케이스 찾기**: 새 업무 시작 시 비슷한 과거 케이스 수배가
   사람 네트워크에 의존. → **유사 케이스 자동 추천** (P1).
5. **한글 호환**: 관공서 자료. → **hwp 변환** (P2).
6. **세법 개정 누락**: 적용 가능한 공제·감면 놓침. → **체크리스트** (P2).
7. **AI 부정확성**: 일반인이 AI에 물어 틀린 정보 가져옴. → **출처
   필수 명시**, LLM 추정과 사실을 구분.

## 자료 보안

- 외감기업 재무제표·감사보고서: 금감원 전자공시(공개).
- 그 외 내부 거래·내부 자료: 외부 노출 금지.
- → 본 시스템은 **로컬 파일 시스템**에서만 동작. 외부 서버 전송 없음.

## 워크플로우 시사점

- **반복성**: 매년 같은 회사를 다룬다 → 회사별 누적 뷰가 핵심.
- **참조성**: 과거 케이스를 템플릿 삼아 새 케이스 작성 → 유사 검색
  추천 기능이 가치 있음.
- **신뢰성**: 회계 자료는 출처가 곧 신뢰 → 모든 추론에 출처 표기.
