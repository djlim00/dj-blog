# SPECS.md - Acceptance Criteria

이 문서는 "무엇이 완성인가"를 정의합니다. Ralph 루프는 이 기준을
만족할 때까지 반복합니다. 각 항목은 **검증 가능한 형태**로 작성되어
있습니다.

우선순위: P0 = MVP 필수 / P1 = 다음 단계 / P2 = 나중에

---

## P0-1. 마크다운 위키 읽기 인프라

**완료 조건**:
- [ ] `lib/wiki.ts`에 `wiki/` 디렉토리를 읽어 페이지 목록을 반환하는
  함수가 있다.
- [ ] 페이지 frontmatter는 zod 스키마로 검증된다.
- [ ] frontmatter가 잘못된 페이지는 콘솔 경고와 함께 스킵된다 (앱이
  죽지 않는다).
- [ ] `npm run typecheck`, `npm run lint` 통과.

---

## P0-2. 회사 페이지 frontmatter 스키마

**완료 조건**:
- [ ] `docs/SCHEMA.md`에 정의된 회사 페이지 스키마가 `lib/schema.ts`에
  zod로 구현되어 있다.
- [ ] 샘플 회사 페이지 1개가 `wiki/companies/`에 존재하며 스키마를
  통과한다.
- [ ] 스키마 검증 단위 테스트가 있고 통과한다.

---

## P0-3. 메인 대시보드 (홈 화면)

URL: `/`

**완료 조건**:
- [ ] 회사 목록이 카드 또는 테이블로 표시된다.
- [ ] 각 회사 카드 클릭 시 회사 대시보드로 이동한다.
- [ ] 최근 업데이트된 위키 페이지 5개가 표시된다.
- [ ] 전체 통계(회사 수, 업무 유형별 건수)가 표시된다.
- [ ] 빈 상태(위키에 회사가 0개일 때) UI가 깨지지 않는다.

---

## P0-4. 회사 대시보드

URL: `/companies/[slug]`

**완료 조건**:
- [ ] 상단에 회사 기본정보(업종, 결산월, 사업자번호, 대표자) 표시.
- [ ] 재무 추세 라인 차트(매출, 영업이익, 차입금, 자본금, 연도별)가
  Recharts로 렌더링된다.
- [ ] 주주 분포 변화 테이블이 연도별로 표시된다.
- [ ] 업무 이력 타임라인(감사·세무·가치평가·용역)이 시간순으로
  표시된다.
- [ ] 특이사항 누적 영역에 frontmatter의 `notable_events` 항목이
  렌더링된다.
- [ ] 관련 원본 파일 링크 목록이 표시된다 (file:// 또는 상대 경로).
- [ ] 데이터 누락 시 "데이터 없음" 등 안전한 fallback이 표시된다.

---

## P0-5. 위키 페이지 뷰어

URL: `/wiki/[...path]`

**완료 조건**:
- [ ] 좌측 사이드바에 `wiki/` 디렉토리 트리가 표시된다.
- [ ] 중앙에 마크다운이 렌더링된다.
- [ ] 위키링크 `[[페이지명]]`이 클릭 가능한 링크로 변환된다.
- [ ] 우측 사이드바에 백링크(이 페이지를 가리키는 페이지들)가 표시된다.
- [ ] 우측 사이드바에 출처 파일 링크가 표시된다.

---

## P0-6. 통합 검색

URL: `/search?q=...`

**완료 조건**:
- [ ] 검색 입력창에서 키워드 입력 시 결과가 표시된다.
- [ ] 결과는 세 영역으로 구분: 위키 페이지 / 원본 파일 / 회사.
- [ ] 태그 필터(연도, 회사, 업무 유형)가 동작한다.
- [ ] 결과 클릭 시 해당 페이지로 이동한다.

## 백프레셔 명령

매 iteration이 통과해야 하는 명령:

```bash
npm run typecheck   # tsc --noEmit
npm run lint        # next lint
npm run build       # 프로덕션 빌드 성공
npm test            # vitest 또는 jest, 단위 테스트 0 fail
```

이 중 하나라도 실패하면 commit하지 않습니다.
