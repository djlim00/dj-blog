# 4. TEXT vs BLOB

#### 1. TEXT
- <mark class="hltr-yellow">TEXT는 행 내부에 저장하지 않음! (포인터)</mark>
	- 행에 있는 데이터의 크기는 상관없음
	- Disk에 있는 데이터의 포인터를 저장

#### 2. 이진데이터 : BLOB
- <mark class="hltr-yellow">이미지, 오디오, 비디오를 직적 저장</mark>

- 근데 보통 S3에 저장함 -> URL만 VARCHAR로 저장
### 왜 BLOB을 많이 안쓸까?

1. **성능문제** : 데이터베이스에 큰 이진 파일(ex. 이미지)를 저장하면 DB 성능 저하, 관리에 어려움, 백업 및 복구 시간 증가
2. **보안문제** : DB에 이미지를 저장하면, 이 이미지에 대한 접근을 제어하고 관리하는 것이 복잡함. 보안 정책을 적용하고 유지하기 어려움.
	- S3에서는 버튼 클릭 하나씩 설정하면 그냥 단순함

# 5. ENUM vs SET (문자열 열거)

#### 1. ENUM
- <mark class="hltr-yellow">열에서 단일 선택만 할 수 있음</mark>
	- ENUM이 {red, green, blue}있으면 셋 중에 하나만 선택할 수 있음.
	- 1,2,3 처럼 숫자로 저장됨

#### 2. SET
- 한번에 여러개 선택가능
- 하나의 열에 여러개 저장 가능
- 비트연산 가능

- 슈퍼키 -> <mark class="hltr-yellow">유일성</mark>

- NULL일 수 잇음
