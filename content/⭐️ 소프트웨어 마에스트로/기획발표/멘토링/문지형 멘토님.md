# 어떻게 하면 뾰족하게 만들 수 있을까?

#### 1. 책 구절
: 왜 사람들에게 읽었던 구절을 다시 찾아봐야하는지? 
-> 문제 자체는 맞는데, 구체화를 어떻게 할 수 있을까?라는 생각이 들었음.
- 책을 통해서 얻고 싶은게 사람마다 다르다 : 1.유흥, 2.감정적인 교류, 3.해결책, 4.지식확장
	- ex) 스타트업을 하게 되면서, 회사 내의 가치충돌을 해결하기/기본적인 것들을 상기하기 위해서 많이 읽게 되었음/책에서 힌트를 얻기 -> <u>해결책을 얻기 위해서 읽었음</u>
	- **즉, 소설 뿐만 아니라 사용자가 어떤 문제상황을 말했을 때, 책에서 그런 부분을 찾아주는 기능을 만들어보자.**
		- 책에서 답을 얻고자 하는 사람들의 인터뷰를 얻어보자.
- 다른 방식들 : 기록(노트)앱과 경쟁을 해야함.

- **하나의 기능에 집중해서 선택하는 것이 좋다 => 유저가 느끼는 문제의 강도에 따라서 달라짐!**
- **수익화 관점에서는 다른 기능이 있는 것이 좋다.** 
	- 수익모델 가능한 두가지 : 광고모델, 커머스(책 판매)
