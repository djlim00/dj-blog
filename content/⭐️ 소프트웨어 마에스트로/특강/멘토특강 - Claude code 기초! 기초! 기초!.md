# CLAUDE.md

- 자동으로 읽어서 컨텍스트 앞부분에 삽입
	- 규칙을 따르도록 유도
	- 굳이 모든 코드를 읽지 않아도 됨(파일 위치를 알려준다던가)
- 프로젝트를 생성을 하고 -> /init으로 만들면 됨(루트에서)
#### 시스템 프롬프트 -> CLAUDE.md -> 유저 대화 내용

- 모든 프로젝트 -> 해당 프로젝트 -> 해당 폴더 (스코프가 작은 claude.md일 수록 우선순위가 올라감)

> 1. 프로젝트명-설명
> 2. 기술스택
> 3. 도메인 용어
> 4. 주요 명령어(실행/테스트/빌드/검증)
> 5. 규칙-컨벤션
##### CLAUDE.mc 작성원칙
- 200자 이내
- 구체적으로 작성(검증 가능하면 더 좋음)
- 처음부터 완벽하게 쓰려하지 않기
- 초안 후 /init 또는 명시적 요청으로 업데이트

#### 팀 공유 - Compound Engineering
- 팀의 경험이 claude.md를 지속적으로 축적
	- PR리뷰

##### 유용한 단축키
- /clear : 작업간의 콘텍스트 삭제
- /context : 현재 내가 쓰고 있는 컨텍스트(토큰 사용량 확인 - 50%를 넘기지 않게 하는 것이 좋음)
	- 기본 시스템 프롬프트
	- 기본tools+연결된 MCP tools
- /resume : 이전의 대화 내역(session)으로 들어가서 할 수 있음.
- /export : 컨텍스트를 내보낼 수 있음.(gemini나 gpt한테 다시 물어볼 수 있음)

# Skill

- SKILL.md 파일에 정의 - 에어전트가 호출할 수 있는 구체적인 도구(함수)

#### 자동화 / 품질일정 / 공유 / 컨텍스트 관리

- 스킬은 claude.md 처럼 무조건 들어가는 것은 아니고

1. 자동 매칭 트리거 : 클로드가 상황과 description을 매칭해 자동 판단하여 호출

# ⭐️ 컨텍스트 윈도우 관리 팁

- 한 세션에는 한 피처만 - 작성 세분화 추천
	- 하나 끝나면 commit하고 /clear
- 이전 대화에 집착하지 않기 (필요시 /clear)
	- 컨텍스트가 오염 -> 말을 안들음
- 사용하지 않는 MCP는 disabled 설정
	- 자주 쓰는 tools만 커스텀 MCP로 묶기
- 무거운 작업은 스크립트로 처리하고 결과만 받아 사용(오프로드)
	- ex) 테스트 같은 것들은 이제 스크립트를 만들고 돌려달라고만 하기
