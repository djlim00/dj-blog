**자바에서 Object 또는 Data를 다른 컴퓨터의 자바 시스템에서도 사용할 수 있도록 바이트 스트림 형태로 연속적인 데이터로 변환하는 포맷 변환 기술**

## 직렬화 vs JSON

- 요즘 세상에 데이터를 내보낼 때 JSON형태의 파일로 보내면 되는데 굳이 바이트 스트림으로 바꾸어서 보내야 하나?

### JSON의 장점

1. 웹 뿐만 아니라, 여러가지 상황에서 설정 파일로 쓰이거나 데이터를 교환할 때 사용된다
2. 직렬화는 자바 프로그램에서만 사용 가능 | JSON은 파이썬, 자바스크립 등등 어디서나 사용 가능


### 직렬화의 장점

1. 자바 고유 기술인 만큼 자바 시스템에서 개발에 최적화되어 있다.
2. 광활한 레퍼런스 타입에 대해 제약 없이 외부(자바프로그램 한정)로 내보낼 수 있다.
자바에서만 쓰이는 온갖 컬렉션, 클래스, 인터페이스 타입들을 다른 언어로 바꾸려면 이걸 또 파싱하는 과정이 필요하다. 
반면 직렬화를 사용하면, 바이트 스트림으로 바뀐 데이터가 바로 다른 자바 프로그램에서 역직렬화되서 사용 가능하다. -> 그렇기에 직렬화된(바이트 스트림) 데이터를 데이터 베이스에 저장하기도 한다.
