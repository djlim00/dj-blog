🚀 페스티 초이스 — 실제 축제 현장 투표 시스템 함께 만들 개발자 모집 (BE 1 / FE 1)


🎯 프로젝트 소개

페스티 초이스는
실제 지역 축제 현장에서 참가자가 직접 참여할 수 있는 온라인 투표 시스템을 구축하는 프로젝트입니다.

기획+배포 단계에서 끝나는 단순 프로젝트가 아닙니다.

약 4만 명 이상 방문하는 실제 지역 축제 테스트베드 확보
실사용자 기반 테스트 예정동시 접속 및 트래픽 환경 경험 가능
현재 기획 및 UI 디자인 완료 상태이며,
3월 한 달 동안 MVP를 완성하여 4월 4일(토) 실제 축제 현장에서 테스트를 진행합니다.

이 프로젝트를 통해

✔️ 동시성 문제 대응
✔️ 트래픽 분산 세팅 경험
✔️ 실사용자 기반 장애 대응 경험
✔️ 운영 환경에서의 구조 설계 경험

을 직접 경험할 수 있습니다.


📅 프로젝트 일정

- 개발 시작: 3월 초
- 1차 목표: 3월 말 MVP 제작 및 배포 완료
- 유저 테스트: 4월 4일(토)


🔵 백엔드 주요 업무

1. 투표 생성 및 집계 API 설계/개발
2. 실시간 집계 반영 구조 설계트래픽 대응 세팅 (동시성 고려)
3. 페스티 통합회원 인증/권한 동기화
4. AWS EC2 배포 환경 세팅

#### 🛠️ Backend Stack

- Spring Boot (Java 기반)
- AWS EC2
- Slack / Notion
※ 배포 및 인프라 관련 기술 지원 가능

#### 🤝 협업 방식

• 주 2회 회의
① 비대면: 회고 및 주간 개발 성과 공유
② 대면: 주간 개발 킥오프 및 구조 논의
• Slack / Notion 기반 협업
• 실제 축제 적용 전 집중 점검 진행

#### 🙋 이런 분을 찾습니다

- 단순 구현이 아닌 구조를 고민해보고 싶은 분실제 사용자 트래픽을 경험해보고 싶은 분책임감 있게 MVP 완성까지 함께할 수 있는 분열린 소통이 가능한 분
