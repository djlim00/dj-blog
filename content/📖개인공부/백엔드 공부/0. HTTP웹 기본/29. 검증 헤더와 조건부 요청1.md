
- 캐시 유효 시간이 초과해서 서버에 다시 요청하면 둘 중 하나의 케이스다

1. 서버에서 기존 데이터를 변경함 -> 이건 그냥 속절없이 다시 다운로드
2. <u>서버에서 기존 데이터를 변경하지 않음</u>

___
# 1. 캐시 시간 초과

- 캐시 만료 후에도 서버에서 데이터를 변경하지 않음
- 생각해보면 그냥 <font color="#ffff00">저장해 놓은 캐시를 재사용!</font>
- 단 <font color="#ffff00">클라이언트의 데이터와 서버의 데이터가 같다는 사실을 확인할 수 있는 방법 필요</font>
