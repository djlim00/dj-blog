# 1. POST - 신규 자원 등록 (Collection)

![[Pasted image 20250102150500.png]]
![[Pasted image 20250102150334.png]]
- <span style="background:#d4b106">클라이언트는 등록될 리소스의 URI를 모른다!</span>
- 서버가 새로운 리소스 URI를 생성함 -> **/100**
	-  HTTP/1.1 201 Created
		Location: /members/100 
- <font color="#ffff00">컬렉션(Collection)</font>
	- <u>서버가 관리하는 리소스 디렉토리</u>
	- 서버가 리소스의 URI를 생성하고 관리
	- 여기서 컬렉션은 **/members**

# 파일 관리 시스템 예시 - PUT 기반 등록

![[Pasted image 20250102150557.png]]
- 파일은 새로 생성하거나, 기존을 덮어쓰는 경우이므로 -> 등록을 PUT으로 하면 된다!

# 2. PUT - 신규 자원 등록 (Store)
![[Pasted image 20250102150923.png]]
- <span style="background:#d4b106">클라이언트가 리소스 URI를 알고 있어야한다!</span>
- 클라이언트가 직접 리소스의 URI를 지정한다.

> POST : Body에 데이터를 넣고, /members 라고만 한다.
> PUT : /files/star.jpg(클라이언트가 알고 있음) 라고 해야한다.

- 스토어(Store)
	- <u>클라이언트가 관리하는 리소스 저장소</u>
	- 클라이언트가 리소스의 URI를 알고 관리
	- 여기서 스토어는 **/files**
