# 1. 4xx (클라이언트 오류)

- 클라이언트의 요청이 잘못된 문법 등으로 <u>서버가 요청을 수행할 수가 없음</u>
- <font color="#ffff00">오류의 원인이 클라이언트에 있음</font>
- <span style="background:#d4b106">중요! 클라이언트가 이미 잘못된 요청 데이터를 보내고 있기 떄문에, 똑같은 재시도가 실패함(복구 불가)</span>
	- 5xx 오류의 경우는 서버에서 오류가 발생한 것이므로 클라이언트에서 보낸 요청 데이터는 멀쩡하기 때문에 재시도하면 성공할 가능성이 있음(복구가능)

___
# 2. 400 Bad Request - 잘못된 요청

- 클라이언트가 잘못된 요청을 해서 서버가 요청을 처리할 수 없음
- 클라이언트에서 요청 내용을 <font color="#ffff00">다시 검토하고 보내야함</font>
	-  요청 파라미터가 잘못됨, API 스펙이 맞지 않을 때
	- 서버 개발자는 철저한 validation을 해서, 빠르게 튕겨내줘야함

___
# 3. 401 Unauthorized - 인증 안됨

- <font color="#ffff00">인증(Authentication)</font> 되지 않음
- 401 오류 발생 시 응답에 WWW-Authenticate 헤더와 함께 인증 방법을 설명
- 참고
	- <font color="#ffff00">인증</font>(Authentication) : 본인이 누구인지 확인(로그인)
	- <font color="SKYBLUE">인가</font>(Authorization) : 권한부여(ADMIN 권한 처럼 특정 리소스에 접근할 수 있는 권한, 인증이 있어야 인가가 있음)
	- 오류 메시지 Unauthorized이지만 <font color="#ffff00">인증</font>되지 않음(인가에 관련된 것 같은 이름이라서 이름이 아쉬움) -> Unauthenticated가 맞는 말인 것 같은데!!!

___
# 4. 403 Forbidden - 요청확인/승인거부

- 주로 <font color="#ffff00">인증 자격 증명은 있지만, 접근 권한이 불충분한 경우</font>
	- ADMIN 등급이 아닌 사용자가 로그인은 했지만, ADMIN 등급의 리소스에 접근하는 경우
- 즉 인증 자격은 있지만 인가가 안된 경우

___
# 5. 404 Not Found - 요청 리소스를 찾을 수 없음

- 요청 리소스가 서버에 없음
- 클라이언트가 권한이 부족한 리소스에 접근할 때 해당 리소스를 숨기고 싶을 때

___
____
# 6. 5xx (서버 오류)

- 서버 문제로 오류 발생
- 서버에 문제가 있기 떄문에 재시도 하면 성공할 수도 있음(복구가 되거나 동등)
ex) NPE, DB에러

#### <font color="#ffff00">정말로 서버에 문제가 생겼을 때 500이 되도록 하는게 맞다</font>
- 고객의 잔고가 부족해서 안되는 경우 -> 500 에러로 하면 안된다
- 고객의 나이가 부족해서 안된다 -> 500 에러로 하면 안된다
- 비즈니스 로직 상으로 예외 케이스의 경우지 500대 에러를 사용하면 안된다.

___
# 7. 500 Internal Server Error - 공용 서버 에러

- 서버 내무 문제 -> 애매한 경우

___
# 8.  503 Service Unavailable - 서비스 이용 불가

- 서버가 일시적인 과부하 또는 예정된 작업으로 잠시 요청을 처리하지 못함
- Retry-After 헤더 필드로 얼마뒤에 복구되는지 보낼 수도 있음

-> 근데 서버의 문제는 보통 예상치 못함 -> 그냥 500번으로 떨어진다
