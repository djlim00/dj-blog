# 1. 안전(Safe Methods)

- 호출해도 리소스를 변경하지 않는다. ex) GET
- 리소스를 바꾸는 메서드는 안전하지 않다. ex) POST,PUT,DELETE...etc
	- 로그가 쌓여서 장애가 생기고, 이런건 고려하지 않는다.

___
# 2. 멱등(Idempotent)

- f(f(x)) = f(x)
- 몇번을 호출해도 결과가 똑같다.
- 멱등 메서드
	1. GET : 몇번을 조회해도, 같은 결과가 조회된다.
	2. PUT : 똑같은 파일을 몇번을 PUT해도 결과가 같다.
	3. DELETE : 삭제는 계속 해도 삭제된다.
	4. <font color="#ffff00">POST는 멱등하지 않은 메서드다!!</font>
- 활용
	- 자동 복구 매커니즘
	- DELETE를 호출했는데 뭔가 서버에서 응답이 없음 -> 그냥 클라이언트가 한번 더 시도를 해도 괜찮음(멱등하기 때문에)

# 3. 캐시가능(Cacheable)

- 응답 결과를 캐시에서 사용해도 되는가?
- GET,POST,PATCH,HEAD는 가능
- 실제로는 GET, HEAD 정도만 캐시로 사용
	- POST, PATCH는 본문의 내용(Body)까지 캐시 키로 고려해야하는 데, 구현이 쉽지 않음
	- <font color="#ffff00">GET은 그냥 URL만 캐시 키로 고려하면 됨.</font>
