# 1. if-Modified-Since 사용

![[Pasted image 20250103112905.png]]
1. <font color="#ffff00">캐시랑 서버랑 똑같으면</font> -> 304 Not Modified(Body 안보냄) -> 3xx니까 나의 캐시로 리다이렉션

2. <font color="#ffff00">진짜로 수정이 되었다면</font>
	200 OK -> 모든 데이터 다시 전송(Body 포함)

# 단점
![[Pasted image 20250103113038.png]]
- 수정을 2번하게 되면 날짜가 바뀌어서 알 수가 없음 -> 실제 컨텐츠는 알 수 없음
- 서버에서 별도의 캐시 로직을 관리하고 싶은 경우
	- 큰 변경이 없으면 그냥 캐시 유지하고 싶으면

# 2. ETag (Entitiy Tag)


- <font color="#ffff00">캐시용 데이터의 임의의 고유한 버전 이름을 달아둠</font>
	- 예) ETag: "v1.0", ETag: "a2jiodwjekjl3"
- 데이터가 변경되면 이 이름을 바꾸어서 변경(Hash 갱신)
	- 예) ETag: "aaaaa" -> ETag: "bbbbb"
- <font color="#ffff00">단순하게 ETag만 보내서 같으면 유지, 다르면 다시 받기</font>
