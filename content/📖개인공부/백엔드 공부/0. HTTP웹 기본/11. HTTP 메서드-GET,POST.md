# 1. GET - 리소스 조회

![[Pasted image 20241231225849.png]]
- 서버에 전달하고 싶은 <font color="#ffff00">데이터는 query(쿼리 스트링)을 통해서 전달</font>
- 메시지 바디를 이용해서 데이터 전달 가능 BUT
	- 지원하는 곳이 많지 않아서 권장하지 않음

![[Pasted image 20241231230014.png]]
- 클라이언트 -> GET 메서드를 통해서 자원을 요청
- 서버 -> 데이터 베이스에서 가져와서 넘겨줄 JSON파일(다른 파일도 가능)을 생성

![[Pasted image 20241231230020.png]]
- 서버에서 HTTP 응답 데이터를 만들어서 전송
	- JSON파일은 body에 들어감
	- 시작 라인 : 상태코드, HTTP 버전 
	- 헤더에 :Content-Type, Content-Length

# 2. POST - 요청 데이터 처리

![[Pasted image 20241231230401.png]]
- <font color="#ffff00">클라이언트에서 서버에 요청을 보낼 때, 클라이언트가 정보를 줄거니까, 서버에서 이걸 처리해줘</font>
- 주로 신규 리소스 등록, 프로세스 처리 등에 사용

![[Pasted image 20241231230459.png]]
- 클라이언트가 데이터와 POST요청을 보낸다
- 이미 서버에는 클라이언트가 특정 데이터와 POST요청을 보냈을 때 어떤 처리를 할지는 약속을 해둠

![[Pasted image 20241231230543.png]]
- 서버에서 데이터를 받아서, 신규 리소스 식별자를 생성 -> 여기서는 100번

![[Pasted image 20241231230608.png]]
- 자원이 생성된 PATH(URI)와 등록된 자원의 정보를 보냄
- 상태코드 201 : 신규데이터 등록되었음


## 2.1 POST 메서드 - 요청 데이터를 어떻게 처리하는 것인가?

![[Pasted image 20241231230708.png]]
- ex1) HTML FORM에 입력된 정보를 이용해서 <font color="#ffff00">데이터 프로세스 제공</font>
	- 회원가입, 주문, 게시글 업로드
- ex2) 서버가 아직 식별하지 않은 <font color="#ffff00">새로운 리소스 생성</font>
	- 신규주문 생성
- ex3) 기존 자원에 <font color="#ffff00">데이터 추가</font>
	- 문서 끝에 내용 추가


## 2.2 POST 메서드 정리

![[Pasted image 20241231230925.png]]
- <font color="#ffff00">데이터 프로세스를 처리하는 과정(단순한 값 변화가 아닌)</font>
	- 서버에서 큰 변화가 일어나는 경우
- <font color="#ffff00"><u>이름을 지을 때, 리소스만으로 해결이 안되는 경우</u></font>
	-  POST /orders/{orderId}/<font color="#92d050">start-delivery</font> (<font color="#92d050">Control URI</font>)
- <font color="#ffff00">조회의 경우에도 Body에 넘겨야하는 경우</font>
