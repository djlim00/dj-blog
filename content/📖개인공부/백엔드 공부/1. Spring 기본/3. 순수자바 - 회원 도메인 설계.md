- 회원 저장소는 지금 , 메모리, DB, 외부 시스템 연동 회원 저장소 셋 중에서 하나를 선택한다.
	- 일단 지금은 정말 간단한 메모리 기반의 저장소로 만들고 -> 나중에 DB로 구현을 바꾸기만 하면 됨.

- 객체 다이러그램은 객체 간의 메모리 참고를 보여주는 다이어그램.
	- 위에서 회원 서비스는 MemberServiceImple(구현)을 참조하는 것을 추가해야함.

1. 도메인 협력 관계 다이어그램 : 기획자도 이해할 수 있음
2. 클래스 다이어그램 : 개발자가 만듦 (static)
3. 객체 다이어그램 : 클래스 다이어그램만으로 판단하기 어려운 경우 실제 인스턴스끼리의 참조를 보여줌 (dynamic)
