
- 의존성 주입을 할 때, 생성자 주입이 많이 쓰이지만 코드가 살짝 길어진다.
- 대부분 다 불변이고, final 키워드를 붙인다.
- 그런데 생성자도 만들고, 주입 받는 값을 대입하는 코드도 만들고.. 그냥 필드 주입 처럼 간단하게 사용할 수는 없는걸까?

### Lombok이란?

- 자동으로 Getter, Setter 등을 만들어주는 어노테이션 관련 라이브러리

<설치 방법>
1. 인텔리제이 플러그인에 Lombok 설치
2. Settings > Annotation Processor에서 활성화 > build.gradle 수정


- 그럼 이제 롬복의 @Constructer를 이용해서 생성자 주입을 편하게 해보자.
- <font color="#ffff00">@RequiredArgsConstructer</font>를 사용하면 --- Required라는 것 즉, 무조건 필요하다 --- 즉 final이 붙은 애들의 생성자를 만들어준다.

이제 아래의 코드를 비교해보면서 얼마나 간단해지는지 보자.


```java title:"@RequiredArgsConstructer가 없는 기본 생성자 코드", hl:21-22
package hello.example.core.order;  
  
import hello.example.core.discount.DiscountPolicy;  
import hello.example.core.discount.FixDiscountPolicy;  
import hello.example.core.discount.RateDiscountPolicy;  
import hello.example.core.member.Member;  
import hello.example.core.member.MemberRepository;  
import hello.example.core.member.MemoryMemberRepository;  
import lombok.RequiredArgsConstructor;  
import org.springframework.beans.factory.annotation.Autowired;  
import org.springframework.stereotype.Component;  
  
@Component  
public class OrderServiceImpl implements OrderService {  
  
    private final MemberRepository memberRepository;  
    private final DiscountPolicy discountPolicy;  
  
    @Autowired // - 생성자가 하나면 안써도 괜찮음  
    public OrderServiceImpl(MemberRepository memberRepository, DiscountPolicy discountPolicy) {  
        this.memberRepository = memberRepository;  
        this.discountPolicy = discountPolicy;  
    }  
  
    @Override  
    public Order createOrder(Long memberId, String itemName, int itemPrice) {  
        Member member = memberRepository.findById(memberId);  
        int discountPrice = discountPolicy.discount(member, itemPrice);  
  
        return new Order(memberId, itemName, itemPrice, discountPrice);  
    }  
  
    public MemberRepository getMemberRepository() {  
        return memberRepository;  
    }  
}
```
- <font color="#ffff00">memberRepository와 discountPolicy에 대한 생성자가 필요함</font>

```java title:"@RequiredArgsConstructor가 있는 코드", hl:14
package hello.example.core.order;

import lombok.RequiredArgsConstructor;  
import org.springframework.beans.factory.annotation.Autowired;

    private final DiscountPolicy discountPolicy;  
  
    @Override

```

- <font color="#ffff00">이제 final이 붙은 애들은 그냥 선언만 해두고 @RequiredArgsConstructor가 자동으로 생성자를 만들어준다</font>
