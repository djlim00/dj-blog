# 0. 생성자 주입

- 지금까지 우리가 하던 방식
- 스프링 빈에 등록되면서 생성자를 호출하는 와중에 @Autowired를 통해서 주입한다.
- <font color="#ffff00">생성자를 호출하는 시점에 딱 한번만 호출되는 것이 보장됨.</font> 
  => <span style="background:#d4b106">"불변, 필수" 의존관계에서 사용</span>

> 불변 : 프로젝트에 대한 의존관계를 절대 바꿀일이 없을 떄, <u>Setter메서드를 만들지 않기.</u>
> 필수 : 생성자에 의해서 무조건 값이 있어야 할 때.

><font color="#ffff00"> 생성자가 <u>딱 한개</u>만 있으면 @Autowired를 생략해도 괜찮음.</font>

____
# 1. Setter 주입
 
``` java title:"수정자 주입 예시" , hl:7-10,12-15 
@Component  
public class OrderServiceImpl implements OrderService {  
  
    private MemberRepository memberRepository;  
    private DiscountPolicy discountPolicy;  
  
    @Autowired  
    public void setDiscountPolicy(DiscountPolicy discountPolicy) {  
        this.discountPolicy = discountPolicy;  
    }  
  
    @Autowired  
    public void setMemberRepository(MemberRepository memberRepository) {  
        this.memberRepository = memberRepository;  
    }  
  
    //@Autowired - 생성자가 하나면 안써도 괜찮음  
    public OrderServiceImpl(MemberRepository memberRepository, DiscountPolicy discountPolicy) {  
        System.out.println("memberRepository = " + memberRepository);  
        System.out.println("discountPolicy = " + discountPolicy);  
  
        this.memberRepository = memberRepository;  
        this.discountPolicy = discountPolicy;  
    }  

    @Override  
    public Order createOrder(Long memberId, String itemName, int itemPrice) {  
        Member member = memberRepository.findById(memberId);  
        int discountPrice = discountPolicy.discount(member, itemPrice);  
  
        return new Order(memberId, itemName, itemPrice, discountPrice);  
    }  
  
    public MemberRepository getMemberRepository() {  
        return memberRepository;  
    }  
}
```

- 컨테이너를 만들고 빈을 등록한다. -> 그러고 다음에 의존관계를 주입한다.

- 생성자 주입은 위의 예시로 보자면 <span style="background:#d4b106">어쩔 수 없이 OrderServiceImpl을 생성을 해야함</span>
  당연하다.. 생성자에서 의존관계를 주입하니까. 
  그래서 그냥 빈을 등록하면서 의존관계 주입도 동시에 일어난다.


> 선택 : MemberRepository가 스프링에 생성이 안되어 있어도, 즉 의존성이 반드시 필요하지 않은 경우에느 (required = false / 이거 안하면 오류 발생함)
> 변경 : 변경 가능성이 있는 인스턴스

# 3. 일반 메서드 등록 - 거의 안씀

그냥 아무 메서드에 @Autowired하기


``` java ,title:"아무 메서드 init()를 등록함"
  
@Component  
public class OrderServiceImpl implements OrderService {  
  
    private MemberRepository memberRepository;  
    private DiscountPolicy discountPolicy;  
  
//    @Autowired(required = false_  
//    public void setDiscountPolicy(DiscountPolicy discountPolicy) {  
//        System.out.println("discountPolicy = " + discountPolicy);  
//        this.discountPolicy = discountPolicy;  
//    }  
//  
//    @Autowired  
//    public void setMemberRepository(MemberRepository memberRepository) {  
//        System.out.println("memberRepository = " + memberRepository);  
//        this.memberRepository = memberRepository;  
//    }  
  
    //@Autowired // - 생성자가 하나면 안써도 괜찮음  
    public OrderServiceImpl(MemberRepository memberRepository, DiscountPolicy discountPolicy) {  
        System.out.println("memberRepository = " + memberRepository);  
        System.out.println("discountPolicy = " + discountPolicy);  
  
        this.memberRepository = memberRepository;  
        this.discountPolicy = discountPolicy;  
    }  
  
    //private final DiscountPolicy discountPolicy = new RateDiscountPolicy();  
    //private DiscountPolicy discountPolicy; 이렇게만 쓰면 NPE당연하게 뜸  
  
  
    @Autowired  
    public void init(MemberRepository memberRepository,DiscountPolicy discountPolicy){  
        this.memberRepository = memberRepository;  
        this.discountPolicy = discountPolicy;  
    }  
  
  
    @Override  
    public Order createOrder(Long memberId, String itemName, int itemPrice) {  
        Member member = memberRepository.findById(memberId);  
        int discountPrice = discountPolicy.discount(member, itemPrice);  
  
        return new Order(memberId, itemName, itemPrice, discountPrice);  
    }  
  
    public MemberRepository getMemberRepository() {  
        return memberRepository;  
    }  
}
```

- 일반 메서드를 통해서 주입 가능, 한번에 여러 필드를 주입 받을 수 있음
- 거의 사용하지 않음
- 당연히 스프링 빈이여야지 동작한다. 일반 클래스에서는 @Autowired를 하면 아무 의미가 없음.
