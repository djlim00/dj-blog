
``` java title:"Spring과 아무 상관없는 코드 - 테스트"
package hello.example.core.autowired;  
  
import hello.example.core.member.Member;  
import io.micrometer.common.lang.Nullable;  
import org.junit.jupiter.api.Test;  
import org.springframework.beans.factory.annotation.Autowired;  
import org.springframework.context.annotation.AnnotationConfigApplicationContext;  
import org.springframework.context.annotation.Configuration;  
  
import java.util.Optional;  
  
public class AutowiredTest {  
  
    @Test  
    void AutowiredOption(){  
        AnnotationConfigApplicationContext ac = new AnnotationConfigApplicationContext(TestBean.class);  
    }  
  
  
    static class TestBean{  
        @Autowired(required = false)  
        public void setNoBean1(Member noBean1){  
            System.out.println("noBean1 = " + noBean1);  
        }  
  
        @Autowired  
        public void setNoBean2(@Nullable Member noBean2){  
            System.out.println("noBean2 = " + noBean2);  
        }  
  
        @Autowired  
        public void setNoBean3(Optional<Member> noBean3){  
            System.out.println("noBean3 = " + noBean3);  
        }  
    }  
}
```

- 위의 코드를 보면 TestBean은 스프링에서 관리하는 클래스조차 아니다.
- 이떄 required = false, @Nullable, @Optional<>을 사용하게 되면
	1. required = false : 자동 주입할 대상이 없기 떄문에 메서드 자체가 호출이 안된다,
	2. @Nullabel : 자동 주입할 대상이 없기 때문에 null이 뜬다
	3. Optional<> : 아무것도 없기 때문에 "Optional.empty"가 뜬다.

이렇게 1번은 호출조차 안되고, 나머지는 null에 대응되는 값이 나온다.



> 여기서 근데 궁금한 점, 왜 자동으로 주입될 것이 없는가?
> 그건 바로 지금 위의 테스트 코드에서 Member에 대해서 주입을 하고 있는데, Member는 스프링 빈에 등록되지 않았기 때문이다.
> 아래를 보자

```java title:"스프링 빈이랑은 아무 상관없는 Member클래스"
package hello.example.core.member;  
  
import org.springframework.stereotype.Component;  
  
public class Member {  
    private Long id;  
    private String name;  
    private Grade grade;  
  
    public Member(Long id, String name, Grade grade) {  
        this.id = id;  
        this.name = name;  
        this.grade = grade;  
    }  
  
    public Long getId() {  
        return id;  
    }  
  
    public void setId(Long id) {  
        this.id = id;  
    }  
  
    public String getName() {  
        return name;  
    }  
  
    public void setName(String name) {  
        this.name = name;  
    }  
  
    public Grade getGrade() {  
        return grade;  
    }  
  
    public void setGrade(Grade grade) {  
        this.grade = grade;  
    }  
}
```

- 보면 뭐 @Component도 없고, 현재 테스트 코드에서 AnnotationConfigApplication에서 TestBean.class를 가져오고 있는데, 거기에서는 Member에 대한 빈이 없다.
- 그렇기 때문에 1에서는 자동으로 주입할 대상이 없는거고
- 2에서는 없을 때 null처리를 해주는 거고
- 3에서도 없을 때 Optonal.empty로 null처리를 해주는거다.
