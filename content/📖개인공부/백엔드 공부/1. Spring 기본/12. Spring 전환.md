#### ApplicationContext를 이용한 스프링 빈 등록 및 Bean호출
![[Pasted image 20241122233530.png]]
``` JAVA
ApplicationContext applicationContext = new AnnotationConfigApplicationContext(AppConfig.class);  

MemberService memberService = applicationContext.getBean("memberService", MemberService.class);  

OrderService orderService = applicationContext.getBean("orderService", OrderService.class);
```

___
### 1. 스프링 컨테이너

- <span style="background:#d4b106">ApplicationContext</span>를 스프링 컨테이너 라고 한다.
- 기존에는 개발자가 AppConfig 객체를 생성해서 DI를 직접 헀지만, 이제부터는 스프링 컨테이너를 통해서 한다.
- 스프링 컨테이너는 <font color="#ffff00">@Configuraion</font>이 붙은 AppConfig를 설정(구성) 정보로 사용.
- <font color="#ffff00">@Bean</font>이라 적힌 모든 메서드를 호출해서 반환된 객체를 스프링 컨테이너에 등록함.
	- 이렇게 등록된 객체를 <font color="#ffff00">스프링 빈</font>이라고 한다.
	- Ex) memberService, orderService
	- 기본적으로 등록된 메서드명을 스프링 빈의 이름으로 사용 / @Bean(name="ABC")로 이름을 바꿀 수도 있음.
- 전에는 개발자가 필요한 객체를 AppConfig를 이용해서 직접 조회했지만, 이제부터는 스프링 컨테이너를 통해서 필요한 스프링 빈을 찾아서 써야함.
	- 이러한 스프링 빈은 <font color="#ffff00">applicationContext.getBean()</font>을 통해서 가져올 수 있음.
- 이제는 스프링 컨테이너에 스프링 빈으로 등록하고, 스프링 컨테이너에 스프링 빈을 찾아서 사용할 수 있도록 한다.
