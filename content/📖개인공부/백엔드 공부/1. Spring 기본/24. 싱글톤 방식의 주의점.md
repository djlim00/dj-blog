
- <font color="#ffff00">여러 클라이언트가 하나의 객체를 공유하기 때문에, 객체는 상태를 유지(staeful)하게 설계하면 안된다.</font>
- 필드 대신에 자바에서 공유되지 않는, 지역변수, 파라미터, ThreadLocal 등을 사용해야한다.

```java title:"싱글톤 방식에서 문제가 생기는 코드", hl:6,3
public class StatefulService {  
  
    private int price; // 상태를 유지하는 필드 10000 -> 20000  
    public void order(String name, int price) {  
        System.out.println("name = " + name + " price = " + price);  
        this.price = price; // 여기가 문제  
    }  
  
    public int getPrice() {  
        return price;  
    }  
}
```

```java title:"값이 바뀌게 되는 문제 - 테스트 코드", hl:10,13,20
class StetefulServiceTest {  
  
    @Test  
    void statefulServiceSingleton() {  
        AnnotationConfigApplicationContext ac = new AnnotationConfigApplicationContext(TestConfig.class);  
        StatefulService statefulService1 = ac.getBean(StatefulService.class);  
        StatefulService statefulService2 = ac.getBean(StatefulService.class);  
  
        // ThreadA: A사용자가 10000원 주문  
        statefulService1.order("userA", 10000);  
  
        // ThreadB: B사용자가 10000원 주문  
        statefulService1.order("userB", 20000);  
  
        // A가 주문하고 조회하는 그 사이에 B가 주문을 해버림 -> A가 조회했는데 200만원이 나옴..!  
        // ThreadA: 사용자A가 주문 금액 조회  
        int price = statefulService1.getPrice();  
        System.out.println("price = " + price);  
  
        Assertions.assertThat(statefulService1.getPrice()).isEqualTo(20000);  
    }  
  
    static class TestConfig {  
  
        @Bean  
        public StatefulService statefulService() {  
            return new StatefulService();  
        }  
    }  
  
}
```

- 분명히 A는 order()를 10000원으로 했는데, 같은 객체 StatefulService를 사용해 버려서, order를 하게 되면 private int price의 값이 바뀌게 되고, 그게 공유가 된다.(싱글톤 방식이라 객체가 하나만 떠 있기 때문에)
- 웹에서 요청이 오면 thread가 할당이 된다.
- <span style="background:#d4b106">공유되는 필드의 값을 변경해버리는 것이 문제.</span>
- 꼭 한번씩 문제가 생기는 경우가 있다.
- <span style="background:#d4b106">스프링은 진짜 무조건 무상태(Stateless)로 설계해야한다,</span>
