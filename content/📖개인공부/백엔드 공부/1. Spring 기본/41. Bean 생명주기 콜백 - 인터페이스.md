


```java, title:"InitializingBean과 DisposableBean 인터페이스 사용", hl:6,33-36,38-41
package hello.example.core.lifecycle;  
  
import org.springframework.beans.factory.DisposableBean;  
import org.springframework.beans.factory.InitializingBean;  
  
public class NetworkClient implements InitializingBean, DisposableBean {  
  
    private String url;  
  
    public NetworkClient(){  
        System.out.println("생성자 호출, url = "+url);  
    }  
  
    public void setUrl(String url){  
        this.url = url;  
    }  
  
    // 서비스를 시작시 호출하는 메서드  
    public void connect(){  
        System.out.println("connect : "+url);  
    }  
  
    public void call(String message){  
        System.out.println("call : " + url + " message = " + message);  
    }  
  
    // 서비스 종료시 호출 되는 메서드  
    public void disconnect(){  
        System.out.println("close: "+url);  
    }  
  
    @Override  
    public void afterPropertiesSet() throws Exception {  
        connect();  
        call("초기화 연결 메세지");  
    }  
  
    @Override  
    public void destroy() throws Exception {  
        disconnect();  
    }  
}
```
``` title:"결과"
생성자 호출, url = null
connect : https://www.google.com
call : https://www.google.com message = 초기화 연결 메세지
close: https://www.google.com
```

- <span style="background:#fff88f">위 두개의 인터페이스의 afterPropertiesSet()과 Destroy()를 구현을 하면 된다.</span>
- 생성할 때, 종료될 때 Bean들이 생겨서 url값이 넘어가는 것을 볼 수 있다.


### 단점
1. **스프링 전용 인터페이스** -> 모든 코드가 스프링에 의존적이게 된다!
2. 초기화, 소멸 메서드의 **이름을 지정할 수 없음**
3. 내가 코드를 고칠 수 없는 **외부 라이브러리에 적용할 수 없음**

> 굉장히 초창기의 방법 -> 거의 사용 안함
