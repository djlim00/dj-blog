
### 1. 웹 스코프의 특징

- <font color="#ffff00">웹 환경에서만 동작함</font>
-  스프링이 <font color="#ffff00">종료시점까지 관리</font>한다. -> 즉 Destry() 메서드가 호출된다.


### 2. 웹 스코프 종류

- "request" : HTTP 요청 하나가 들어오고 나갈 때 까지 유지 되는 스코프, <font color="#ffff00">각각의 HTTP 요청마다 별도의 빈 인스턴스가 생성되고 관리</font>된다.

- "session" : <font color="#ffff00">HTTP Session</font>과 동일한 생명주기를 가지는 스코프

- "application" : 서블릿 컨텍스트("ServletContext)와 동일한 생명주기를 가지는 스코프

- "websocket" : <font color="#ffff00">웹 소켓</font>과 동일한 생명주기를 가지는 스코프

### request 스코프 예제 코드
```java title:"Mylogger 클래스"
package hello.example.core.common;  
  
  
import jakarta.annotation.PostConstruct;  
import jakarta.annotation.PreDestroy;  
import org.springframework.context.annotation.Scope;  
import org.springframework.stereotype.Component;  
  
import java.util.UUID;  
  
@Component  
@Scope(value = "request")  
public class MyLogger {  
  
    private String uuid;  
    private String requestURL;  
    public void setRequestURL(String requestURL) {  
        this.requestURL = requestURL;  
    }  
  
    public void log(String message){  
        System.out.println("[ "+uuid+" ]"+"[ "+requestURL+" ]"+"[ "+message+" ]");  
    }  
  
    @PostConstruct  
    public void init(){  
        String uuid = UUID.randomUUID().toString();  
        System.out.println("[ "+uuid+" ]"+"request scope bean create : "+this);  
    }  
  
    @PreDestroy  
    public void close(){  
        System.out.println("[ "+uuid+" ]"+"request scope bean close : "+this);  
    }  
  
}
```
- 스프링 컨테이너에 요청되면 요청하나 마다 생성됨
- uuid로 다른 HTTP 요청과 구분이 가능하다
- 생성 시, 소멸 시 함수가 호출된다.

```java title:"LogDemoController", hl:15
package hello.example.core.web;  
  
import hello.example.core.common.MyLogger;  
import jakarta.servlet.http.HttpServletRequest;  
import lombok.RequiredArgsConstructor;  
import org.springframework.stereotype.Controller;  
import org.springframework.web.bind.annotation.RequestMapping;  
import org.springframework.web.bind.annotation.ResponseBody;  
  
@Controller  
@RequiredArgsConstructor  
public class LogDemoController {  
  
    private final LogDemoService logDemoService;  
    private final MyLogger myLogger;  
  
    @RequestMapping("log-demo")  
    @ResponseBody  
    public String logDemo(HttpServletRequest request){  
        String requestURL = request.getRequestURL().toString();  
        myLogger.setRequestURL(requestURL);  
  
        myLogger.log("controller test");  
        logDemoService.logic("testId");  
  
        return "OK";  
    }  
}
```

> <font color="#ffff00">그냥 이 상태로 스프링 서버를 띄우면 오류가 발생한다.</font>
	-> 현재 의존관계 주입을 @RequiredArgsConstructor을 통해서 자동으로 MyLogger와 LogDemoService를 받고 있다
	-> 이때 MyLogger는 웹 스코프다 -> 웹 스코프의 생명주기는 HTTP 요청을 받았을 때 시작한다
	-> 그러나 <font color="#ffff00">현재 스프링 컨테이너를 만드는 시점에는 MyLogger가 존재하지 않기 떄문에 의존관계를 주입할 수 없어서 오류가 발생하는 것이다.</font>

- Logger가 잘 작동하는지 확인하는 테스트용 컨트롤러
- 여기서 HttpServletRequest를 통해서 요청 URL을 받는다
	- 'http://localhost:8080/log-demo'
- 이 URL값을 myLogger에서 set메서드로 저장을 한다
	- 이제 다른 요청들이랑 uuid로 구분이 가능하다
- 컨트롤러는 "controller test"라는 로그를 남긴다

> 참고로 위의 로깅 처리는 Controller보다는 공통처리가 가능한 Spring Interceptor나 서블릿 필터에서 처리하는 것이 좋다.



```java title:"LogDemoService", hl:11
package hello.example.core.web;  
  
import hello.example.core.common.MyLogger;  
import lombok.RequiredArgsConstructor;  
import org.springframework.stereotype.Service;  
  
@Service  
@RequiredArgsConstructor  
public class LogDemoService {  
  
    private final MyLogger myLogger;  
    public void logic(String id) {  
        myLogger.log("service id = "+id);  
    }  
}
```

> - 비즈니스 로직인 Service에서 logic()을 통해서 id를 출력한다.
> - <font color="#ffff00">여기서 request scope를 사용하지 않으면?</font>
	- 웹과 관련된 값을 넘기기 위한 파라미터가 많아짐 -> 코드 지저분해짐
	- 웹과 관련없는 서비스 계층까지 requestURL같은 웹 관련 정보가, 웹과 관련없는 서비스 계층까지 넘어가게 된다.
		- 웹과 관련된 부분은 Controller까지만 사용해야함
	- 위와 같은 문제로 유지보수를 위해서는 서비스 계층은 웹에 종속되지 않게 만드는 것이 중요
