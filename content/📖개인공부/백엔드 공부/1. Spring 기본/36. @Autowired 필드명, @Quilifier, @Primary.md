## 2. @Quilifier

- <font color="#ffff00">추가적인 구분자</font>를 붙여주는 방법이다. 빈 이름을 변경하는 것은 아님!
	1. @Quilifier끼리 매칭
	2. 빈 이름으로 매칭
	3. "NoSuchBeanDefinitionExeption" 발생


``` java title:"@Quilifier를 붙이고 이름을 준다.", hl:2
@Component  
@Qualifier("mainDiscountPolicy")  
public class RateDiscountPolicy implements DiscountPolicy {  
  
    private int discountPercent = 10;  
  
    @Override  
    public int discount(Member member, int price) {  
        if (member.getGrade() == Grade.VIP) {  
            return price * discountPercent / 100;  
        } else {  
            return 0;  
        }  
    }  
}
```

```java title:"앞에 붙여주면 된다", hl:9 
  
@Component  
public class OrderServiceImpl implements OrderService {  
  
    private final MemberRepository memberRepository;  
    private final DiscountPolicy discountPolicy;  
  
    @Autowired // - 생성자가 하나면 안써도 괜찮음  
    public OrderServiceImpl(MemberRepository memberRepository, @Qualifier("mainDiscountPolicy") DiscountPolicy discountPolicy) {  
        this.memberRepository = memberRepository;  
        this.discountPolicy = discountPolicy;  
    }  
    public MemberRepository getMemberRepository() {  
        return memberRepository;  
    }  
}
```

- @Quilifier가 만약에 못찾으면..? -> "mainDiscountPolicy"라는 과정을 다시 한번 하게 된다.
- 근데 솔직히 그렇게 짜는 것은 좋지 않다.<font color="#ffff00">@Quilifier는 @Quilifier를 찾는 용도로만 사용하는 것이 좋다.</font>

## 3. @Primary

- 여러 개가 매칭이 되면 @Primary가 매칭이 된다.

``` java title:"RateDiscountPolicy를 2개 이상일 때 선택"
@Component  
@Primary  
public class RateDiscountPolicy implements DiscountPolicy {  
  
    private int discountPercent = 10;  
  
    @Override  
    public int discount(Member member, int price) {  
        if (member.getGrade() == Grade.VIP) {  
            return price * discountPercent / 100;  
        } else {  
            return 0;  
        }  
    }  
}
```

- main DB가 있고 보조 DB가 있다고 하면, main을 가져오는게 거의 대다수의 코드이면 @Primary를 그냥 걸면 된다.
- 보조 db는 뭐 Quilifier를 걸던지 하면 된다.
