
그냥 설정 정보에
  <font color="#ffff00">@Bean(initMethod = "init", destroyMethod = "close")</font>
이런 식으로 이름을 알려주면 된다.


```java, title:"초기화, 소멸 콜백 지정", hl:28-38,59
  
public class NetworkClient {  
  
    private String url;  
  
    public NetworkClient(){  
        System.out.println("생성자 호출, url = "+url);  
    }  
  
    public void setUrl(String url){  
        this.url = url;  
    }  
  
    // 서비스를 시작시 호출하는 메서드  
    public void connect(){  
        System.out.println("connect : "+url);  
    }  
  
    public void call(String message){  
        System.out.println("call : " + url + " message = " + message);  
    }  
  
    // 서비스 종료시 호출 되는 메서드  
    public void disconnect(){  
        System.out.println("close: "+url);  
    }  
  
    public void init() throws Exception {  
        System.out.println("NetworkClient.init");  
        connect();  
        call("초기화 연결 메세지");  
    }  
  
    public void close() throws Exception {  
        System.out.println("NetworkClient.close");  
        disconnect();  
    }  
}

=============================================================================

public class BeanLifeCycleTest {  
  
    @Test  
    public void lifeCycleTest(){  
        // 그냥 ApplicationContext로 하게 되면 .close()가 안됨 -> 보통 할 일이 없음  
        // 그래서 기본 ApplicationContext에서는 지원하지 않아서  
        // ConfigurableApplicationContext or AnnotationConfigApplicationContext와 같은  
        // 상위의 인터페이스를 사용함  
        ConfigurableApplicationContext ac = new AnnotationConfigApplicationContext(LifeCycleConfig.class);  
        NetworkClient client = ac.getBean(NetworkClient.class);  
        ac.close();  
    }  
  
    @Configuration  
    static class LifeCycleConfig{  
  
        // 스프링 빈이 생성이 되고, url이 set된 상태로 호출이 된다.  
        @Bean(initMethod = "init", destroyMethod = "close")  
        public NetworkClient networkClient(){  
            NetworkClient networkClient = new NetworkClient();  
            networkClient.setUrl("https://www.google.com");  
            return networkClient;  
        }  
    }  
}
```
``` title:"결과"
생성자 호출, url = null
NetworkClient.init
connect : https://www.google.com
call : https://www.google.com message = 초기화 연결 메세지
NetworkClient.close
close: https://www.google.com
```

- 초기화, 소멸 메서드를 만들고 -> Config파일에 이름을 지정해주면 된다.

### 장점
1. 메서드 이름 지정가능
2. 스프링에 의존하지 않음
3. <font color="#ffff00">코드가 아니라, 설정 정보를 사용하기 때문에, 코드를 고칠 수 없는 외부 라이브러리에 초기화, 종료 메서드를 적용할 수 있음</font>

### 가장 중요한 장점 - <font color="#ffff00">종료 메서드 추론</font>

- 라이브러리는 대부분 'close', 'shuetdown'으로 되어있음
- destroyMethod의 default값이 '(inferred)' 즉, 추론으로 되어 있어서, 외부 라이브러리의 종료 메서드도 추론해서 호출해준다.
- 즉 따로 적지 않아도 웬만하면 잘 된다
