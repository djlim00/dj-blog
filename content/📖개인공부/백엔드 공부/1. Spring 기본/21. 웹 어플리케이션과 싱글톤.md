
- 스프링은 태생이 기업용 온라인 기술 지원을 위한 것이다. 
- 대부분의 스프링 애플리케이션은 웹 애플리케이션이다.(다른 것도 당연히 가능)
- <span style="background:#d4b106">보통 여러 고객이 동시에 요청을 한다.</span>

![[Pasted image 20241123192634.png]]
- 이렇게 같은 요청을 많이 요청하면 같은 요청이 3개가 생기게 된다.
	- 요청이 올 때 마다 계속 만들어야 함.

```java title:"스프링이 없는 순수한 DI컨테이너에서 같은 요청 2번"
public class SingletonTest {  
  
    @Test  
    @DisplayName("스프링 없는 순수한 DI컨테이너")  
    void pureContainer() {  
        AppConfig appConfig = new AppConfig();  
  
        // 1. 조회 : 호출할 때 마다 객체를 생성  
        MemberService memberService1 = appConfig.memberService();  
  
        // 2. 조회 : 호출할 때 마다 객체를 생성  
        MemberService memberService2 = appConfig.memberService();  
  
        // 참조값이 다를 것을 확인  
        System.out.println("memberService1 = " + memberService1);  
        System.out.println("memberService2 = " + memberService2);  

		Assertions.assertThat(memberService1).isNotSameAs(memberService2);
    }  
}
```
```java title:"결과 - 다른값을 참조함", hl:1-2
memberService1 = hello.example.core.member.MemberServiceImpl@70be0a2b
memberService2 = hello.example.core.member.MemberServiceImpl@2133814f
```
- 웹 애플리케이션은 정말로 많은 요청을 처리해야하는데 이렇게 계속해서 다른 객체를 생성하는 것은 효율적이지 않음.
- 심지어 우리의 코드에서는 하나의 객체를 생성하면 하나의 객체를 또 생성해야하므로, 두번 호출하게 되면 4개의 객체가 생성이 되는 것임!! -> 극한의 비효율

- <span style="background:#d4b106">그러니까 딱 하나의 객체만 생성한 다음에 공유하도록 하면 된다. => Singleton 패턴</span>
