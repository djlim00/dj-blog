- 싱글톤 빈인 clientBean은 스프링 컨테이너 생성 시점에 함께 생성
  -> 의존관계 주입
  -> 자동으로 의존관계가 주입되면서, 프로토타입 빈을 요청한다
  -> 포로토 타입빈을 생성해서 clientBean에 넘겨주고, 이제 이건 싱글톤 빈인 clientBean에서 관리한다.(count=0)

### ClientBean에 로직 요청 발생!
![[Pasted image 20241229122249.png]]
- 이제 clientBean의 logic()에서 프로토타입빈의 addCount()를 호출한다.
  -> count값이 증가되어 1이 된다.


### 또다른 Client가 ClientBean을 통해 로직 요청!
![[Pasted image 20241229122413.png]]
- 이렇게 되면 싱글톤 빈인 clientBean에 의해서 logic이 호출이 되고, addCount()가 호출된다.
- <font color="#ffff00">즉 count=2가 된다.</font>

한마디로 프로토타입 빈이기 때문에, 한번 생성시점에 생성되고, ClientBean은 싱글톤 빈이기 때문에 처음에 생성되서 받은 prototypBean만을 가지고 있음 -> count값이 공유가 됨.

```java title:"테스트 코드"  
public class SingletonWithPrototypeTest1 {  
  
    @Test  
    void prototypeFind(){  
        AnnotationConfigApplicationContext ac = new AnnotationConfigApplicationContext(PrototypeBean.class);  
        PrototypeBean prototypeBean1 = ac.getBean(PrototypeBean.class);  
        prototypeBean1.addCount();  
        Assertions.assertThat(prototypeBean1.getCount()).isEqualTo(1);  
  
        PrototypeBean prototypeBean2 = ac.getBean(PrototypeBean.class);  
        prototypeBean2.addCount();  
        Assertions.assertThat(prototypeBean2.getCount()).isEqualTo(1);  
  
    }  
  
    @Scope("prototype")  
    static class PrototypeBean{  
        private int count = 0;  
  
        public void addCount(){  
            count++;  
        }  
  
        public int getCount(){  
            return count;  
        }  
  
        @PostConstruct  
        public void init(){  
            System.out.println("PrototypeBean.init "+this);  
        }  
  
        @PreDestroy  
        public void destroy(){  
            System.out.println("PrototypeBean.destroy");  
        }  
    }  
  
    @Test  
    void singletonClientUserPrototype(){  
        AnnotationConfigApplicationContext ac =  
                new AnnotationConfigApplicationContext(PrototypeBean.class, ClientBean.class);  
  
        ClientBean clientBean1 = ac.getBean(ClientBean.class);  
        int count1 = clientBean1.logic();  
        Assertions.assertThat(count1).isEqualTo(1);  
  
        ClientBean clientBean2 = ac.getBean(ClientBean.class);  
        int count2 = clientBean1.logic();  
        Assertions.assertThat(count2).isEqualTo(2);  
  
  
    }  
  
    //@RequiredArgsConstructor  
    @Scope("singleton")  
    static class ClientBean{  
        private final PrototypeBean prototypeBean;  
  
        // 아래의 @Autowired는 사실 위에 @Required..하면 필요도 없음, 그리고 에러 나는건 IDE에서 인식 못하는 것  
        // 코드는 정상적으로 동작함  
        @Autowired  
        public ClientBean(PrototypeBean prototypeBean) {  
            this.prototypeBean = prototypeBean;  
        }  
  
        public int logic(){  
            prototypeBean.addCount();  
            int count = prototypeBean.getCount();  
            return count;  
        }  
    }  
}
```


___

#### <span style="background:#fff88f">그렇다면, 우리의 의도가 logic()을 호출할 때 마다 새로운 프로토타입 빈을 생성해서 쓰고 싶다면 어떻게 해야할까? (즉 클라이언트마다 다른 count값을 적용시키고 싶으면 어떻게 할까?)</span>
-> 애초에 위의 방식대로 할거면 그냥 싱글톤을 만들어서 쓰지 왜 굳이 프로토타입을 하겠나..?


- 그렇다고 뭐 logic()에서 AnnotationConfigApplicationContext로 가져와서 prototypeBean을 주입받는 방식은 너무 비효율적인 방식임(좋은 코드가 아님)
- 만약 클라이언트1, 클라이언트2 이렇게 하면 뭐 주입받을 때 각각 다른 프로토타입 빈을 받기는 함. -> 근데 뭐 이럴일이 없음
