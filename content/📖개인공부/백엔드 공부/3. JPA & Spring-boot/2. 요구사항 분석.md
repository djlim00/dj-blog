****회원**, **주문**, **상품의** **관계**:** 회원은 여러 상품을 주문할 수 있다. 그리고 한 번 주문할 때 여러 상품을 선택할 수 있으므로

주문과 상품은 다대다 관계다. 하지만 이런 <font color="#ff0000">다대다 관계는 관계형 데이터베이스는 물론이고 엔티티에서도 거의 사용하지 않는다.</font> <mark style="background: #FFF3A3A6;">따라서 그림처럼 주문상품이라는 엔티티를 추가해서 다대다 관계를 일대다, 다대일 관계로 풀어냈다.</mark>


![[Pasted image 20250407113922.png]]

1. **Member** : 임베디드 타입인 Address, Orders의 List를 가짐
2. **Order** : 한 번 주문 시 여러 상품을 주문할 수 있기 떄문에, Order와 OrderItem은 <mark style="background: #FFF3A3A6;">일대다 관계</mark>이다. status는 enum으로 ORER와 CANCEL이 있다.
3. **OrderItem** : 상품 정보와 orderPrice와 count(주문 수량)을 가진다.(보통 count보다는 OrderLine으로 표현)
4. **Item** : 이름, 가격, stockQuantity(재고수량)를 가지고 있음(주문하면 줄어듦). 종류로는 도서, 음반, 영화
5. **Delivery** : 주문 시 하나의 배송 정보 생성(주문과 일대일 관계)
6. **Category** : 상품과 다대다 관계를 맺음. parent, child로 부모, 자식 카테고리 연결
7. **Address** : 임베디드 타입. 회원과 배송에서 사용

# 2. @ManyToMany관계에서의 매핑
![[Pasted image 20250409103113.png]]
![[Pasted image 20250409103129.png]]
- <mark style="background: #FFF3A3A6;">객체는 다대다가 바로 가능한데, 실제 관계형 데이터베이스에서는 ManyToMany관계에서 중간의 하나의 테이블이 필요하다.</mark>
	- <font color="#ff0000">그렇기 때문에 @JoinTable(name="category_item")을 매핑해줘야함</font>
		- 보면 FK가 각각의 id를 가지고 있음


데이터 정합성이 중요하다면 FK를 달아주는 것이 좋다.
그런게 아니라면 안걸어도 되려나,,

- @Getter는 열어두는 것이 좋다.
- @Setter는 데이터를 변화하기 때문에 열어두는 것이 좋지 않다.
	- 꼭 수정이 필요한 경우에만 열어둔다,

```java title:"Category 엔티티"
@Entity  
@Getter@Setter  
public class Category {  
    @Id@GeneratedValue  
    @Column(name="category_id")  
    private Long id;  
  
    private String name;  
  
    @ManyToMany  
    @JoinTable(name="category_item",  
            joinColumns =@JoinColumn(name="category_id"),  
            inverseJoinColumns = @JoinColumn(name = "item_id")  
    )  
    private List<Item> items = new ArrayList<>();  
  
    @ManyToOne  
    @JoinColumn(name="parent_id")  
    private Category parent;  
  
    @OneToMany(mappedBy = "parent")  
    private List<Category> child = new ArrayList<>();  
}
```
- 카테고리는 부모-자식 관계로 되어있다. 
- 이때 둘다 동일한 Category인데 parent와 child로 나뉜다.
	- 하나의 부모 카테고리에 여러개의 자식 카테고리가 있기 때문에
	- @ManyToOne <-> @OneToMany
- 연관관계 매핑하는 방식은 동일하다.
1. Category parent : 연관관계의 주인 
	- @JoinColumn
2. chlid 
	- 거울 -> mappedBy = "parent"

```java title:"Address 임베디드 클래스"
@Embeddable  
@Getter  
public class Address {  
  
    private String city;  
    private String street;  
    private String zipcode;  
  
    // JPA 스펙을 위해서 만든 생성자  
    protected Address(){}  
  
    // 생성할 때만 값을 넣을 수 있음(Setter 못하게)  
    public Address(String city, String street, String zipcode) {  
        this.city = city;  
        this.street = street;  
        this.zipcode = zipcode;  
    }  
}
```
- 값을 밖에서 수정하지 못하게 생성하는 시점에서만 값을 설정하도록 생성자를 사용
- <mark style="background: #FFF3A3A6;">JPA 스펙을 위해 protected 기본 생성자가 필요</mark>
