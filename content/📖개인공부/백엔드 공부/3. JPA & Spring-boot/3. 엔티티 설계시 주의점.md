## 2. 모든 연관관계는 지연로딩으로 설정! 🙉

- 즉시 로딩(EAGER)는 예측이 어렵고 어떤 SQL이 실행될지 추적하기 어렵다. 특히 JPQL을 실행할 때 N+1문제가 생김
	- 연관된 모든 데이터를 끌고 오기 때문에 난리난다.
- <mark style="background: #FFF3A3A6;">실무에서 모든 연관관계는 지연로딩(LAZY)로 설정해야한다.</mark>
- <u>연관된 엔티티를 함꼐 DB에서 조회해야 하면, fetch join 또는 엔티티 그래프 기능을 사용한다.</u>
- <mark style="background: #FFF3A3A6;">@XToOne(@OneToOne, @ManyToOne, @)은 기본이 EAGER가서 직접 지연로딩으로 설정해야한다..!</mark>

```java title:"Order 엔티티 바꾸기", hl:11,18  
@Getter  
@Setter  
@Entity  
@Table(name = "orders")  
public class Order {  
  
    @Id @GeneratedValue  
    @Column(name="order_id")  
    private Long id;  
  
    @ManyToOne(fetch = FetchType.LAZY)  
    @JoinColumn(name = "member_id") // FK의 이름이 member_id임!  
    private Member member;  
  
    @OneToMany(mappedBy = "order")  
    private List<OrderItem> orderItems = new ArrayList<>();  
  
    @OneToOne(fetch = FetchType.LAZY)  
    @JoinColumn(name="delivery_id")  
    private Delivery delivery;  
  
    private LocalDateTime orderDate; // 주문시간  
  
    @Enumerated(EnumType.STRING)  
    private OrderStatus status; // 주문 상태  
}
```
- fethType을 바꿔줘야 한다!

## 3. 컬렉션은 필드에서 초기화 하자.

```java title:"생성자에서 초기화 하면 안되나?"
@OneToMany(mappedBy = "member")  
private List<Order> orders = new ArrayList<>();

------------------ 아래 방법은 안될까? ------------------

@OneToMany(mappedBy = "member")  
private List<Order> orders;

public Member(){
	orders = new ArrayList<>();
}
```

필드 주입의 장점
1. 초기화가 안되는 문제에서 자유로움(NULL 문제)
2. 컬렉션은 한번 생성하고 절대 바꾸지 않기(Hibernate가 관리하기 때문에 도중에 바뀌면 문제가 생긴다.)

##  4. 테이블, 컬럼명 생성 전략 

- Spring boot에서 Hibernate 기본 매핑 전략을 변경해서 실제 테이블 필드명은 다름.

### 👴 기존의 Hibernate의 구현
: <u>엔티티의 필드명을 그래도 테이블명으로 사용</u>

### 😎 신규 스프링 부트 설정(엔티티(필드) -> 테이블(컬럼명))
'SpringPhysicaNamingStrategy'
1. 카멜 케이스 -> 스네이크 케이스
2. .(점) ->  _ (언더스코어)
3. 대문자 -> 소문자
ex) orderDate -> order_date

## 3. Cascade(영속성 전이)

참고링크 : https://hongchangsub.com/jpa-cascade-2/
- 연관관계를 맻으면 <font color="#ff0000">연쇄적</font>으로 일어나게 하는 것을 말한다.

ex) 댓글을 삭제하면 하위의 대댓글도 삭제한다.
ex) A가 저장될 때 연관관계가 있는 B도 저장된다.

![[Pasted image 20250409153240.png]]
- 게시글의 사진과 같은 경우는 무조건 게시글이 삭제되면 같이 삭제되어야함.
- <mark style="background: #FFF3A3A6;">모든 OneToMany에 걸면 -> 예상치 못하게 저장, 삭제될 수 있음.</mark>

![[Pasted image 20250409153338.png]]

```java title:"Order 엔티티에서 Cascade", hl:1,4
@OneToMany(mappedBy = "order", cascade = CascadeType.ALL)  
private List<OrderItem> orderItems = new ArrayList<>();  
  
@OneToOne(fetch = FetchType.LAZY,cascade = CascadeType.ALL)  
@JoinColumn(name="delivery_id")  
private Delivery delivery;
```
- 이렇게 해놓으면, Order에 대해서 persist가 일어나면 OrderItem과 Delivery에서도 연쇄적으로 일어난다.
