### <span style="background:rgba(3, 135, 102, 0.2)">해결방법 -> 리다이렉트(Redirect)</span>
![[Pasted image 20250312114312.png|400]]
- 그렇다면, 상품저장을 했을 때, 상품 상세 페이지를 단순하게 뷰 템플릿을 띄우는 방식 즉
	- "return basic/item"을 하는 것이 아닌 다른 페이지로 redirect를 하면
	- 상품 상세 페이지에서 새로고침을 해도 GET요청으로 상품상세 페이지만 오기 때문에 Post요청에 의한 중복 상품 등록이 되지 않는다!

=> URL 자체를 완전히 다른 URL로 바꿔버리는 방법이라는 것이지


```java title:"PRG"
@PostMapping("/add")  
public String addItemV5(Item item){  
  
    // 이런 객체의 경우는 또 @ModelAttribute가 생락된거임!  
    itemRepository.save(item);  
    return "redirect:/basic/items/"+item.getId();  
}
```

- PRG를 통해서 /basic/item/{itemId}로 가기 때문에 괜찮음!
- <mark style="background: #FFF3A3A6;">"redirect:/basic/items/{itemId}" 이렇게 쓰지 않고 "redirect:/basic/items/"+item.getId() 이렇게 선언하면 URL 인코딩이 되지 않아서 위험하다.</mark>
	- RedirectAttribute를 이제 알아보자
