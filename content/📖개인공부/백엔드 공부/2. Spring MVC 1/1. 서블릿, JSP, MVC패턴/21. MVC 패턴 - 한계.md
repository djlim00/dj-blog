
- 뭔가 컨트롤러들은 중복되는 코드가 많고.. 딱히 필요하지가 않아보임
	- RequestDispatcher를 이용해서 forward를 하는 부분이 중복된다.
	- viewPath도 중복되는 내용이 많이 들어가 있음 "WEB-INF/views/..."까지는 전부 동일함
		- 만약에 파일 구조가 바뀌게 되면 모든 코드를 바꿔야함
	- response는 거의 사용되지 않고 있음
	- 테스트 코드를 작성하는 경우에 굉장히 귀찮음

### "공통처리가 어렵다"
: 프로젝트가 커질수록 공통처리하는 부분이 많다(로그처리, 코드중복...etc)
: <span style="background:#fff88f">컨트롤러 호출전에 공통기능을 전부 수행하는 "수문장 역할"을 하는 사람이 필요하다</span>
	-> 즉 <font color="#ffff00">"프론트 컨트롤러(Front Controller) 패턴"</font>을 도입한다!
	-> 이건 <font color="#ffff00">Spring MVC 패턴의 핵심</font>이다.
