# 1. MVC(Model-View-Controller)


#### 1.1 컨트롤러
: HTTP 요청을 받아서, 파라미터를 검증하고, 비즈니스 로직을 실행한다. 그리고 뷰에 
전달할 결과 데이터를 조회해서 모델에 담는다.

#### 1.2 모델
: 뷰에 출력할 데이터를 담아둔다. 뷰가 필요한 데이터를 모두 모델에 담아서 전달해주는 덕분에, 뷰는 비즈니스 로직이나, 데이터 접근을 몰라도 되고, 화면을 랜더링 하는 일에 집중할 수 있다.
- <font color="#ffff00">컨트롤러와 뷰의 의존관계를 끊음</font>


#### 1.3 뷰
: 모델에 담겨있는 데이터를 사용해서 화면을 그리는 일만 한다. 여기서는 HTML을 생성하는 부분을 말한다.


![[Pasted image 20250128113911.png]]
- 비즈니스 로직 -> 서비스에서 코드가 작성이 된다. -> 데이터 관련된 코드는 레포지토리를 통해서 처리
- 그러니까, HTTP 요청이나 응답에 대한 일은 컨트롤러가 한다. 그럼 이제 서비스와 레포지토리를 통해서 서비스 로직을 실행하고 데이터를 가져온다
- 그 이후에 Model을 만들어서 이를 뷰 로직에 전달한다
- 뷰 로직은 데이터를 받아와서 그걸 이용해서 화면을 만들어서 응답한다.
	- 보통 서비스의 비즈니스 로직이 수정되면, 컨트롤러의 코드가 수정될 가능성이 높다.
