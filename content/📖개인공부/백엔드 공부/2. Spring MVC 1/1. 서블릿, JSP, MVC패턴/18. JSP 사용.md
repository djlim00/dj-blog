```html title:"JSP로 작성한 HTML-form 화면"

    <meta charset="UTF-8">  
    <title>Title</title>  
</head>  
<body>  
<form action="/jsp/members/save.jsp" method="post">  
    username: <input type="text" name="username" />  
    age: <input type="text" name="age" />  
    <button type="submit">전송</button>  
</form>  
</body>
```

- 회원의 이름과, 나이를 Form으로 입력 받는다
- -> <font color="#ffff00">"/jsp/members/save.jsp"으로 POST 메소드로 redirect된다!</font>

```h title:"JSP 사용 예제 입"
<%@ page import="hello.servlet.domain.member.Member" %>  
<%@ page import="hello.servlet.domain.member.MemberRepository" %>  
<%@ page contentType="text/html;charset=UTF-8" language="java" %>  


	<%  
		  // request, response는 그냥 사용가능 -> 자동으로 서블릿으로 변환되기 때문에

	  System.out.println("MemberRepository.service");  
	  String username = request.getParameter("username");

<html>

성공<ul>

<a href="/index.html">메인</a>  
</body>  
</html>"
```

- <% %> 을 이용해서 로직을 작성한다
- 아래에는 html이 위의 로직을 이용해서 동적으로 생성된다
	- response와 request는 그냥 사용가능하다.

# 서블릿 + JSP 사용의 단점

1. html은 깔끔해짐(중간에 동적인 코드 좀만 넣으면 됨) 그러나!!
2. JAVA코드, 레포지토리 코드가 전부 JSP에 노출 - 코드가 너무 김
	- <font color="#ffff00">즉 JSP에 너무 많은 역할이 있음!</font>
