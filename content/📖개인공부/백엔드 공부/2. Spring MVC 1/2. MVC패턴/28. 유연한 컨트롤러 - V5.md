지금은 Controller가 어떤 형식을 사용하는지(V1,V2,V3..) 한번 결정하면 바꿀 수가 없음
-> 하나의 인터페이스로 모든 컨트롤러가 묶여있기 때문에

이런 것을 <mark style="background: #FFF3A3A6;">다양한 형태의 컨트롤러를 지원</mark>할 수 있게 바꾸는 
유연한 컨트롤러를 만들어보자.

## **어댑터 패턴**
지금까지 우리가 개발한 프론트 컨트롤러는 한가지 방식의 컨트롤러 인터페이스만 사용할 수 있다.
ControllerV3와 ControllerV4는 완전히 다른 인터페이스다.
따라서 호환이 불가능하다. 
이럴 때 사용하는 것이 바로 **"어댑터**"이다.

![[Pasted image 20250221154244.png]]
- 컨트롤러를 찾아올 수 있는 어댑터를 찾아옴(해당 컨트롤러를 처리할 수 있는 어댑터)
- 프론트 컨트롤러도 핸들러 어댑터를 통해서 컨트롤러를 호출해야함
- 그냥 진짜로 사이에 껴서 프론트 컨트롤러에는 같은 형식의 값을 주고, 각각의 방법에 따른 값을 하나로 통일해 준다고 생각하면 될 것 같다
- 핸들러 -> 컨트롤보다 넓은 범위
	- 꼭 컨트롤러가 아니라 다른 것들도 처리가 가능하다

### 파일구조
![[Pasted image 20250221172605.png]]

```java title:"MyHandlerAdapter - 인터페이스"  
public interface MyHandlerAdapter {  
  
    boolean supports(Object handler);  
  
    ModelView handle(HttpServletRequest request, HttpServletResponse response,Object handler) throws ServletException, IOException;  
}
```
1. <mark style="background: #FFF3A3A6;">support() : 해당 요청(request)를 handle 가능한 핸들러어댑터인가?를 판단해주는 함수</mark>
2. <mark style="background: #FFF3A3A6;">handle() : 핸들러를 호출해서 ModelView를 반환하는 함수</mark>


# V3를 먼저 구현해보자자
```java title:"ControllerV3HandlerAdapter"
public class ControllerV3HandlerAdapter implements MyHandlerAdapter {  
    @Override  
    public boolean supports(Object handler) {  
        return (handler instanceof ControllerV3);  
    }  
  
    @Override  
    public ModelView handle(HttpServletRequest request, HttpServletResponse response, Object handler) throws ServletException, IOException {  
        // 이미 supports 에서 한번 ControllerV3인지 거르기 때문에 캐스팅 해도 안전하다  
        ControllerV3 controller = (ControllerV3) handler;  
  
        Map<String, String> paramMap = createParamMap(request);  
        ModelView mv = controller.process(paramMap);  
  
        return mv;  
    }  
  
    private static Map<String, String> createParamMap(HttpServletRequest request) {  
        Map<String, String> paramMap = new HashMap<>();  
        request.getParameterNames().asIterator()  
                .forEachRemaining(paramName -> paramMap.put(paramName, request.getParameter(paramName)));  
        return paramMap;  
    }  
  
  
}
```
#### 이건 V3에 해당하는 핸들러 어댑터이다.
- 즉 V3에 해당하는 핸들러 어댑터인지 확인하고
- <mark style="background: #FFF3A3A6;">req, res, 핸들러를 넘겨받아서, ParamMap을 생성하고 -> 핸들러를 통해서 ModelView를 반환받는다.</mark>

```java title:"MemberFormControllerV3"
public class MemberFormControllerV3 implements ControllerV3 {  
  
    @Override  
    public ModelView process(Map<String, String> paramMap) {  
        return new ModelView("new-form");  
    }  
}
```
#### 이전에 V3에서 만든 컨트롤러이다 -> 지금은 핸들러🥸
- 이게 결국은 핸들러 어댑터를 통해 호출당해서 로직을 실행하는 "핸들러"다.


```java title:"FronControllerServletV5"
@WebServlet(name = "fronControllerServletV5", urlPatterns = "/front-controller/v5/*")  
public class FrontControllerServletV5 extends HttpServlet {  
  
    private final Map<String, Object> handlerMappingMap = new HashMap<>();  
    private final List<MyHandlerAdapter> handlerAdapters = new ArrayList<>();  
  
    public FrontControllerServletV5(){  
        initHandlerMappingMap();  
        initHandlerAdapters();  
    }  
  
    private void initHandlerAdapters() {  
        handlerAdapters.add(new ControllerV3HandlerAdapter());  
    }  
  
    private void initHandlerMappingMap() {  
        // 우선 V3를 지원부터 만듬  
        handlerMappingMap.put("/front-controller/v5/v3/members/new-form",new MemberFormControllerV3());  
        handlerMappingMap.put("/front-controller/v5/v3/members/save",new MemberSaveControllerV3());  
        handlerMappingMap.put("/front-controller/v5/v3/members",new MemberListControllerV3());  
    }  
  
    @Override  
    protected void service(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {  
  
        Object handler = getHandler(req);  
  
        if(handler==null){  
            resp.setStatus(HttpServletResponse.SC_NOT_FOUND);  
            return;  
        }  
  
        // 해당 핸들러에 맞는 핸들러 어댑터를 찾기  
        MyHandlerAdapter adapter = geHandlerAdapter(handler);  
  
        // adpter의 handle == Controller의 process        ModelView mv = adapter.handle(req, resp, handler);  
  
        // 위에서는 논리 이름만 얻기 때문에 뷰리졸버로 물리 주소가 필요함  
        String viewName = mv.getViewname();  
  
        // "/WEB-INF/views/new-form.jsp" -> 이런식으로 만들어짐  
        MyView view = viewResolver(viewName);  
  
        // 렌더링 작업  
        view.render(mv.getModel(),req,resp);  
    }  
  
    private MyHandlerAdapter geHandlerAdapter(Object handler) {  
  
        for (MyHandlerAdapter adapter : handlerAdapters) {  
            if(adapter.supports(handler)) return adapter;  
        }  
  
        throw new IllegalArgumentException("handler adapter를 찾을 수 없습니다.");  
    }  
  
    private Object getHandler(HttpServletRequest req) {  
        String requestURI = req.getRequestURI();  
        Object handler = handlerMappingMap.get(requestURI);  
        return handler;  
    }  
  
    private static MyView viewResolver(String viewName) {  
        return new MyView("/WEB-INF/views/" + viewName + ".jsp");  
    }  
}
```

- handlerAdapters : 핸들러 어댑터 리스트이다. -> 이제 이걸 돌면서 지금 요청을 suport()하는
