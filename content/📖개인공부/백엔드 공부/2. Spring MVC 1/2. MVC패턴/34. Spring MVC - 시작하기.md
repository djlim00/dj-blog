- 유연하고 실용적이다

### "@RequestMapping"
: Annotation을 활용한 매우 유연하고 실용적인 컨트롤러 -> @RequestMapping
: 과거에는 MVC 웹 기능은 스트럿츠 같은 다른 프레임워크를 사용했다.
: 그러나 @RequestMapping이 나오면서 그냥 다른 프레임워크는 멸망했다.


#### "@RequestMapping"
- **"RequestMappingHandlerMapping"** : 우리의 컨트롤러(핸들러)를 찾아줌
- **"RequestMappingHandlerAdapter"** 
<span style="background:#fff88f">=> 가장 우선순위가 높은 핸들러 매핑과 핸들러 어댑터이다.</span>

```java title:"ContollerV1 - Spring 어노테이션"
@Controller  
public class SpringMemberFormControllerV1 {  
  
    @RequestMapping("/springmvc/v1/members/new-form")  
    public ModelAndView process(){  
        return new ModelAndView("new-form");  
    }  
}
```
1. @Controller : 스프링이 자동으로 스프링 빈으로 등록
	- <span style="background:#fff88f">@Controller 내부에 @Component가 있어서 자동으로 스프링 빈에 등록됨</span>
	- 즉, 컴포넌트 스캔의 대상이 된다.
	- 또한 Controller로  인식을 하기 때문에 자동으로 <font color="#d83931">핸들러로 인식</font>(핸들러 매핑)
2. @RequestMapping : 요청 정보를 매핑. 
	- 해당 URL이 호출되면 이 메서드가 호출.
	- <span style="background:#fff88f">애노테이션을 기반으로 동작하기 때문에, 메서드의 이름은 아무렇게나 상관없음</span>
	- <u>즉 메서드의 이름을 기반으로 빈을 찾는게 아님!</u>
3. ModelAndView : 모델과 뷰의 정보를 담아서 반환
