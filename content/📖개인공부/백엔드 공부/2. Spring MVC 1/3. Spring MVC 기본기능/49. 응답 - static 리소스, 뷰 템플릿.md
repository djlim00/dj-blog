# 뷰 템플릿 - 동적 HTML
____
- 뷰 템플릿을 거쳐서 HTML이 생성되고, 뷰가 응답을 만들어서 전달
	- 일반적으로 동적 HTML
	- 뷰 템플릿이 지원하는 여러가지 기능이 있음

```html title:"뷰 템플릿 resources/templates/response/hello.html", hl:8
<!DOCTYPE html>  
<html xmlns:th="http://www.thymeleaf.org">  
<head>  
    <meta charset="UTF-8">  
    <title>Title</title>  
</head>  
<body>  
<p th:text="${data}">empty</p>  
</body>  
</html>
```


```java title:"뷰를 반환하는 방법", hl:12,13,20,21
@Controller  
public class ResponseViewController {  
  
    @RequestMapping("/response-view-v1")  
    public ModelAndView responseViewV1(){  
        ModelAndView mav = new ModelAndView("response/hello")  
                .addObject("data","hello");  
  
        return mav;  
    }  
  
    // @Controller면서 String을 반환하면 -> 뷰의 논리이름  
    // @ResonseBody하면 안됨!! -> HTTP Body에 들어감  
    @RequestMapping("/response-view-v2")  
    public String responseViewV2(Model model){  
        model.addAttribute("data","hello!");  
        return "response/hello";  
    }  
  
    // 너무 불명확해서, 비추  
    // url과 파일의 위치가 동일하면 가능  
    @RequestMapping("/response/hello")  
    public void responseViewV3(Model model){  
        model.addAttribute("data","hello!");  
    }  
}
```
- <mark style="background: #FFF3A3A6;">뷰를 반환하는 경우에는 @ResponseBody하기 않기!!</mark>

### Thymeleaf **스프링** **부트** **설정**
____
다음 라이브러리를 추가하면(****이미** **추가되어** **있다**.**)
```java title:"build.gradle"
implementation 'org.springframework.boot:spring-boot-starter-thymeleaf'
```

스프링 부트가 자동으로 `ThymeleafViewResolver` 와 필요한 스프링 빈들을 등록한다. 그리고 다음 설정도 사용한다. 이 설정은 기본 값 이기 때문에 변경이 필요할 때만 설정하면 된다.
```java title:"application.properties"
spring.thymeleaf.prefix=classpath:/templates/spring.thymeleaf.suffix=.html
```
