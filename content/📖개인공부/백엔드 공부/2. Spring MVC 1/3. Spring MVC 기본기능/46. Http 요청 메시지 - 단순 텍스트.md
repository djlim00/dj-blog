Http message body에 데이터를 직접 담아서 요청
1. JSON, XML, TEXT
2. 데이터 형식은 주로 JSON을 사용함
3. POST, PUT, PATCH

- @RequestParam이나 @ModelAttribute를 사용하는 방식 -> 쿼리 파라미터
- 다른 형식은 HTTP body를 봐야한다.

```java title:"HTTP Body 요청 처리", hl:22-30
@Slf4j  
@Controller  
public class RequestBodyStringController {  
  
    @PostMapping("/request-body-string-v1")  
    public void reqeustBodyString(HttpServletRequest req, HttpServletResponse res) throws IOException {  
        ServletInputStream inputStream = req.getInputStream();  
        String messageBody = StreamUtils.copyToString(inputStream, StandardCharsets.UTF_8);  
  
        log.info("messageBody:{}",messageBody);  
        res.getWriter().write("ok");  
    }  
  
    @PostMapping("/request-body-string-v2")  
    public void reqeustBodyStringV2(InputStream inputStream, Writer responseWriter) throws IOException {  
        String messageBody = StreamUtils.copyToString(inputStream, StandardCharsets.UTF_8);  
  
        log.info("messageBody:{}",messageBody);  
        responseWriter.write("ok");  
    }  
  
    @PostMapping("/request-body-string-v3")  
    public HttpEntity<String> reqeustBodyStringV3(HttpEntity<String> httpEntity) throws IOException {  
  
        String messageBody = httpEntity.getBody();  
  
        log.info("messageBody:{}",messageBody);  
        return new HttpEntity<>("ok");  
    }  
}
```
- v1과 v2는 진짜 개에바 코드다 -> 너무 지저분함
- <mark style="background: #FFF3A3A6;">HttpEntitiy를 사용하면 바디 그 자체를 주고 받는 느낌으로 할 수 있음</mark>

### 최종진화형 - @RequestBody
```java
@ResponseBody  
@PostMapping("/request-body-string-v4")  
public String reqeustBodyStringV4(@RequestBody String messageBody) throws IOException {  
  
    log.info("messageBody:{}",messageBody);  
    return "ok";  
}
```
🙉 이제 슬슬 헷갈리기 시작한다.

1. @RequestBody는 클라이언트에서 보내는 요청이 HttpBody에 있음!
2. 그니까 이제 그걸 받아오기 위해서 그냥 @RequestBody String messageBody로 받아오기만 하면 됨.(받아온게 로그에 찍힘)
3. 그러고 @ResonseBody는 Http body에 나의 응답값을 넣겠다는 것이다(뷰 반환이 아니라)
