```java titlel:"@RequestParam 예시"
// @ResponseBody를 쓰면 그냥 뷰가 아니라 문자열을 body에 들어가게 된다.  
@ResponseBody  
@RequestMapping("/request-param-v2")  
public String requestParamV2(  
        @RequestParam("username")  String memberName,  
        @RequestParam("age") int memberAge){  
  
    log.info("username={}, age={}", memberName, memberAge);  
    // 이렇게 하면 ok라는 뷰를 찾게 된다.  
    return "ok";  
}
```

- 그냥 결론적으로
	- @RequestParam("username") String username
	- 은 아래와 동일한 것이다.
	- String username = req.getParamater("username");


### 필수 파라미터 여부- (required = true)

```java title:"username은 필수, age는 필수 아님", hl:5
@ResponseBody  
@RequestMapping("/request-param-v4")  
public String requestParamV5(  
        @RequestParam(required = true) String username,  
        @RequestParam(required = false) Integer age){  
  
    log.info("username={}, age={}", username, age);  
    // 이렇게 하면 ok라는 뷰를 찾게 된다.  
    return "ok";  
}
```
- 근데 V5에 int로 선언하면 age에 null값을 넣어줘야지 없다는 뜻이고 완전히 아무 값도 없으면 안됨
	- <mark style="background: #FFF3A3A6;">Integer로 선언을 해야지 완전히 아무값도 없을 수 있음</mark>

⚠️ ""은 빈문자로 들어와서 들어왔다고 본다. null이 진짜 없다는 의미다



### Default value
```java title:"기본값 설정"
@ResponseBody  
@RequestMapping("/request-param-default")  
public String requestParamDefault(  
        @RequestParam(required = true,defaultValue = "guest") String username,  
        @RequestParam(required = false, defaultValue = "-1") int age){  
  
    log.info("username={}, age={}", username, age);  
    // 이렇게 하면 ok라는 뷰를 찾게 된다.  
    return "ok";  
}
```

### ParamMap으로 한번에 가져오기
```java
@ResponseBody  
@RequestMapping("/request-param-map")  
public String requestParamMap(  
        @RequestParam Map<String, Object> paramMap){  
    log.info("username={}, age={}", paramMap.get("username"), paramMap.get("age"));  
    // 이렇게 하면 ok라는 뷰를 찾게 된다.  
    return "ok";  
}
```
