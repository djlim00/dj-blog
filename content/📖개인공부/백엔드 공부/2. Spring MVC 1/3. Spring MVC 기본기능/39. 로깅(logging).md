### 로깅 라이브러리
- 스프링 부트 라이브러리를 사용하면 스프링 부트 로깅 라이브러리 spring-boot-starter-logging가 함께 포함된다.
- 기본적으로 다음 로깅 라이브러리를 사용한다.
	- SLF4J - http://www.slf4j.org - 인터페이스
	- Logback - http://logback.qos.ch - 구현체

수많은 라이브러리가 있지만 SLF4J를 사용한다!

```java title:"LogTestController"
@RestController  
public class LogTestController {  
  
    private final Logger logger = LoggerFactory.getLogger(getClass());  
  
    @RequestMapping("/log-test")  
    public String logTest(){  
        String name = "Spring";  
  
        System.out.println("name = " + name);  
        logger.trace(" trace log={}",name);  
		logger.debug(" debug log={}",name);  
		logger.info(" info log={}",name); 
		logger.warn(" warn log={}",name);  
		logger.error(" error log={}",name);   
  
        return "ok";  
    }  
}
```

**@RestController**
: String으로 반환할 때 뷰 이름이 아니라, String 그 자체가 반환된다.(Body에 String을 넣음)
: @Controller는 String반환값이 뷰로 인식한다.

- <mark style="background: #FFF3A3A6;">log로 찍으면, 시간, 프로세스ID,쓰레드, 컨트롤러 이름, 메시지 가 출력된다!</mark>
	- 굉장히 많은 정보를 볼 수가 있음! -> 디버깅도 빠르다
- 로깅의 레벨을 정할 수가 있음
	- sout을 쓰게 되면 운영서버에서 너무 많은 로그가 남는 문제가 생길 수 있다
	- 개발 서버에서는 debug 레벨로 설정을 하고, 운영서버에서는 info레벨로 설정을 해서 로깅을 줄이는 것이다.
		- <span style="background:#fff88f">LEVEL : TRACE > DEBUG > INFO > WARN > ERROR</span>
		- <span style="background:#fff88f">상위 레벨을 선택하면 하위 레벨도 나오게 된다는 뜻.</span>

개발 -> Debug 레벨
운영 -> Info 레벨

### 😎 Lombok의 @Slf4j 
```java title:"선언할 필요가 없다"
@Slf4j  
@RestController  
public class LogTestController {  
  
    //private final Logger logger = LoggerFactory.getLogger(getClass());  
  
    @RequestMapping("/log-test")  
    public String logTest(){  
        String name = "Spring";  
  
        System.out.println("name = " + name);  
  
        log.trace(" trace log={}",name);  
        log.debug(" debug log={}",name);  
        log.warn(" warn log={}",name);  
        log.error(" error log={}",name);  
        log.info(" info log={}",name);  
  
        return "ok";  
    }  
}
```

#### 로그의 올바른 사용법
- <span style="background:rgba(205, 244, 105, 0.55)">무조건 log.debug(" trace log={}",name) 이렇게 써야한다</span>
	- log.debug("trace log="+name) 이렇게 쓰면 <font color="#d83931">+로 인해서 연산이 미리 일어나게 되고</font>, 쓸모없는 리소스를 사용하게 된다.
	- {}으로 파라미터로 하면 가지고 있지 않는다~


### 로그 사용의 장점
1. 쓰레드 정보, 클래스 이름과 같은 부가 정보를 볼 수 있음
2. 출력 형식을 조정가능
3. 로그 레벨에 따라 구동 환경에 따라서 로그 출력수준 조절 가능
4. 파일이나 네트워크와 같이 별도의 위치에 로그 저장 가능 -> 파일(일별, 월별..)
5. 성능이 좋다 -> 내부 버퍼링, 멀티 쓰레드 -> <span style="background:rgba(205, 244, 105, 0.55)">무조건 로그를 쓰자!</span>
