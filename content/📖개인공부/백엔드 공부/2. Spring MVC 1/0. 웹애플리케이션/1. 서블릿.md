# 1. 서블릿

- TCP/IP 연결 대기 -> 소켓 연결 -> HTTP 요청 메세지 파싱해서 읽기 -> HTTP 메서드 확인해서, 읽고 어떤 데이터인지도 읽고 -> HTTP 바디 내용 파싱하고 -> <font color="#92d050">{비즈니스 로직 실행}</font> -> HTTP 응답 메세지 생성(start-line,header,body) -> TCP/IP 응답 전달 -> 소켓 종료

- <font color="#92d050">초록색 부분을 제외한 모든 것을 해준다</font>

## 1.2 서블릿 컨테이너 - WAS 내부

![[Pasted image 20250104192623.png]]

1. 톰캣 처럼 서블릿을 지원하는 WAS를 서블릿 컨테이너
2. 서블릿 객체를 생성, 초기화, 호출, 종료하는 생명주기 관리
3. <font color="#ffff00">서블릿 객체를 싱글톤으로 관리</font>
	- 최초의 로딩 시점의 서블릿 객체를 미리 만들어서 재활용
		- 요청, 응답은 항상 다르지만, 서블릿 객체는 재활용 가능하니까
	- <font color="#ffff00">공유변수 사용 주의!</font>
	- 서블릿 컨테이너 종료시 함께 종료
4. JSP도 서블릿으로 변환 되어서 사용
5. <span style="background:#fff88f">동시 요청을 위한 멀티 쓰레드 처리 지원</span>
