- 주로 회원가입, 상품 주문 등에서 사용하는 방식
- 우선 basic/hello-form.html을 만들면
	- 정적 리소스를 바로 볼 수 있음


![[Pasted image 20250105222626.png]]
- 이렇게 그냥 <font color="#ffff00">basic/hello-form.html하면 된다!</font>


```html title:"hello-form.html 파일"
<!DOCTYPE html>  
<html>  
<head>  
  <meta charset="UTF-8">  
  <title>Title</title>  
</head>  
<body>  
<form action="/request-param" method="post">  
  username: <input type="text" name="username" />  
  age: <input type="text" name="age" />  
  <button type="submit">전송</button>  
</form>  
</body>  
</html>
```

- 보면 <font color="#ffff00">action="/request-param"</font>으로 보내고 있다 -> 아까 전 시간에 GET 쿼리 파라미터 방식
	- 즉 HTML Form 방식은 POST 요청으로 쿼리 파라미터의 형식으로(같은 방식으로) 보내기 때문에 아까의 GET 쿼리파라미터 방식으로 리다이렉션을 걸어서 보내면 된다

```java title:"결과"
[전체 파라미터 조회] - start
username=kim
age=20
[전체 파라미터 조회] - end

[단일 파마미터 조회] - start
username = kim
age = 20
[단일 파마미터 조회] - end

[이름이 같은 복수 파라미터 조회] - start
username = kim
[이름이 같은 복수 파라미터 조회] - end
```

- <span style="background:#fff88f">즉 POST HTTP Form 방식도 그냥 req.getParameter()를 쓰면 된다</span>


> 차이점이라면 GET 방식은 Body 자체가 없기 때문에 Content-Type이 존재하지 않는다.
> 
> 위의 POST HTTP Form방식을 사용하면 뭐 JSON이라든지, text라든지 방식으로 Body에 넣어서 보내기 때문에 Content-Type이 존재한다.
