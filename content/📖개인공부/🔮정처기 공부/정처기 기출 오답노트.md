1. **GoF 디자인 패턴** 
	- 생성 패턴 : 빌더, 싱글톤, 팩토리, 프로토타입
	- 구조 패턴 : ~ er 가 붙은 경우(이어주는 느낌)
	- 행위 패턴 : ~자 패턴(이름이 어렵거나, ~하는 사람 느낌)

2. **미들웨어**
	- DB : 2-Teir 아키텍처
	- RPC : 원격을 마치 로컬 처럼
	- MOM : 메시지 기반
	- TP-Monnitor : 항공기, 온라인 트랜잭션(빠른 응답)
	- ORB : CORMA 표준 스펙
	- WAS : 웹 어플리케이션 서버
3. **XP (걍 뭔가 추상적인 애들은 XP임, 빠르고 간단하게 하자)**
	- 용기
	- 의사소통
	- 단순성
	- 존중
	- 피드백
	- Pair, 공동 코드 소유, TDD, Whole Team, CI, 리팩토링, small releases
4. **결합도(대충 어떤 느낌인지 알아두자)**
	- 자료 결합도 : 자료요소만 구성
	- 스탬프 결합도 : 자료구조 전달
	- 제어 결합도 : 제어신호
	- 외부 결합도 : 외부의 다른 모듈 참조
	- 공통 결합도 : 공유되는 데이터 영역
	- 내용 결합도 : 내부 기능 및 그 내부 자료를 직접 참조
5. **파티션(DB를 나누는 방법)**
	- 범위 분할 (단순히 쪼갬)
	- 해시 분할 (해시값으로 나눔)
	- 조합 분할(범위+해시)
	- 목록 분할(열에 따라)
	- 라운드 로빈 분할
6. **릴레이션 특징**
	- 중복 없음
	- 속성은 유일한 값
	- 값은 모두 원자값(더 이상 나눌 수 있는 어떤 값이 아님)
	- 튜플에 순서 존재X
7. **뷰(View)**
	- RESTRICT옵션 : 다른 곳에서 참조하면 삭제하지 않음
	- CASCADE옵션 : 참조까지 다 지움
	- ALTER VIEW 불가능 -> 정의 변경 불가능
8. **단계별로 전문가 -> 델파이 기법**
9. **똑같 패스워드 다른 암호값 -> Salt**
10. SSH
	- 22번 포트
	- 암호화
	- 원격
	- 공개키 등록해야함
11. 출발지 도착지 다르게 공격 -> Land
12. 상향식 & 하향식 설계 방법
	- 하향식 -> 통합 테스트에서 인터페이스가 이미 정의 되어 있음
	- 상향식도 인터페이스가 있어야지, 기능 추가가 좋음
13. **버블정렬, 삽입정렬, 선택정렬**
	- 선택 정렬 : 최솟값을 맨앞으로 이동, 두번쨰 최솟값을 맨앞으로 이동 -> PASS 1,2,3..
	- 삽입 정렬 : 1번부터 보면서 왼쪽에서 자기 자시 찾아가기
	- 버블 정렬 : 교환교환교환 -> 끝까지 가면 1회전
14.  **소스 코드 품질 분석 도구**
	- 동적 분석 도구 : Avalache, Valgrind
	- 정적 분석 도구 : pmd, cppCheck, SonarQube, checkstyle, ccm, cobertura
15. **정규화**
	- 두부이걸다죠
	- 도메인 원자값 ( -> 1)
	- 부분 함수 종속 ( -> 2)
	- 이행적 함수 종속 ( -> 3)
	- 결정키 (-> BCNF)
	- 다치 종속 (-> 4)
	- 조인 종속 (-> 5)
