# 1. 원자성(Atomicity)
___
- "all or nothing"
- <span style="background:rgba(3, 135, 102, 0.2)">전부 수행되었거나, 전부 수행되지 않았거나를 보장함</span>
- 작업의 중간 과정을 알 수 없고, 하나의 트랜잭션은 하나의 operation으로서 중간에 취소되면, 트랙잭션의 수행전으로, 완료되면 트랙잭션이 커밋된 이후로 이 둘 중 하나만의 상태만 존재함을 보장


# 2. 일관성(Consistency)
___
- <span style="background:rgba(3, 135, 102, 0.2)">'허용된 방식'으로만 데이터를 변경해야함</span>
- 모든 데이터를 여러가지 조건, 규칙에 따라 유효함
	- ex) 통장이 0원인 계좌에서는 100원을 출금할 수 없음


# 3. 격리성(Isolation)
___
- <span style="background:rgba(3, 135, 102, 0.2)">트랜잭션 수행 시 서로 끼어들지 못하는 것을 말함</span>
- 복수의 <font color="#0070c0">병렬 트랜잭션</font>은 서로 격리되어 마치 순차적으로 실행되는 것 처럼 작동되어야 함
- 데이터베이스는 여러 사용자가 같은 데이터에 접근할 수 있어야 함.
	- 순차적으로 수행하면 성능이 나쁨..
- <u>격리성은 여러 개의 격리 수준으로 나뉘어 격리성을 보장함</u>


# 4. 지속성(Durabilty)
___
- <span style="background:rgba(3, 135, 102, 0.2)">성공적으로 수행된 트랜잭션은 영원히 반영되어야 하는 것의 의미</span>
	- 데이터베이스에 시스템 장애가 발생해도, 원래 상태로 복구하는 회복 기능이 있어야함
	- 체크섬, 저널링(로깅비슷), 롤백...etc


___
### 무결성
- 데이터의 정확성, 일관성, 유효성을 유지하는 것
- 저장된 데이터 값과 그 값에 해당하는 현실 세계의 실제 값이 일치여부에 대한 신뢰가 생김
1. **개체 무결성** : PK는 NOT NULL
2. **참조 무결성** : 참조된 값과 동일 해야함
3. **고유 무결성** : 특성 속성에 대해 고유한 값을 가지도록 조건이 주어진 경우, 그 속성은 모두 고유한 값을 가짐
4. **NULL 무결성** : NOT NULL인 경우  NULL일 수 없음
