##  내부 조인(Inner Join)
____
<mark style="background: #FFF3A3A6;">두 테이블에서 일치하는 데이터만 반환(교집합)</mark>
A와 B가 있을 때, A와 B 모두에 존재하는 데이터만 결과로 나타낸다.

### 왼쪽 외부 조인(Left Outer Join)
![[Pasted image 20250304131720.png]]
- <mark style="background: #FFF3A3A6;">첫 번째(왼쪽) 테이블의 모든 레코드 + 두 번째(오른쪽) 테이블의 일치하는 레코드를 반환</mark>
-> <font color="#d83931">일치하는 테이블이 없다면, 왼쪽 테이블의 레코드와 NULL 값으로 구성</font>

cris는 Employees에는 있는데 Department에는 없으니까 NULL이 들어감


#### 오른쪽 외부 조인(Right Outer Join)
![[Pasted image 20250304131832.png]]
- <mark style="background: #FFF3A3A6;">두 번째(오른쪽) 테이블의 모든 레코드와 첫 번째(왼쪽) 테이블의 일치하는 레코드를 반환합니다.</mark>
-> 왼쪽 테이블에 일치하는 레코드가 없다면, 결과는 오른쪽 테이블의 레코드와 NULL 값으로 구성

-> department에는 csnote가 있는데 ID가 동일한 값이 Emplyees에는 없음!

### 전체 외부 조인(Full Outer Join)
![[Pasted image 20250304131940.png]]
- <mark style="background: #FFF3A3A6;">두 테이블의 모든 레코드를 반환한다.</mark><span style="background:#d3f8b6"> 한 테이블에만 존재하는 레코드는 다른 테이블에 해당하는 부분이 NULL 값으로 채워짐</span>
- MySQL은 Full Outer Join이 없어서 left랑 right를 하고 UNION하면 된다.

## 성능차이
___
- 일반적으로 Inner Join이 Outer Join보다 빠르다
- 보통 Inner Join은 교집합만이 필요하기 때문에 처리해야할 데이터가 상대적으로 적음
	- <span style="background:#d3f8b6">다만 실제 성능은 쿼리, 데이터베이스 구조, 인덱스 구조 등에 따라 다름</span>
