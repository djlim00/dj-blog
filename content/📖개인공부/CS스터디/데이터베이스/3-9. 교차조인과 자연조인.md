## 교차조인(Cross Join)
___
- 두 개 이상의 테이블에서 <u>가능한 모든 행의 조합을 생성</u>하는 조인방법
- 두 테이블 간에 어떠한 조인 조건이 없을 때 사용되며
	- <span style="background:#d3f8b6">두 테이블의 모든 행 간에 카테시안 곱(Cartesian product)를 생성함</span>
![[Pasted image 20250304132851.png]]
=> 3개의 행이 있고 다른 테이블에 3개의 행이 있다면, 총 9(3x3)의 행으로 구성됨
### 성능
: <mark style="background: #FFF3A3A6;">매우 많은 데이터를 생성하기 떄문에 성능이 좋지 않음.</mark>
: POC를 하거나 테스팅을 할 때 사용함
: 성능저하로 인해 limit을 거는 경우가 많음
![[Pasted image 20250304134004.png|400]]
![[Pasted image 20250304133227.png]]
- 3개의 옷 종류와 3개의 색의 조합 -> 9개!


## 자연조인(Natural Join)
___
- <mark style="background: #FFF3A3A6;">두 테이블 간의 동일한 이름을 가진 모든 열을 기반으로 조인</mark>
	- 명시적으로 조건을 지정 X
	- 두 테이블에서 이름이 같은 모든 열에 대해 내부 조인(Inner Join)을 수행
![[Pasted image 20250304133351.png]]
ex) 두 테이블 모두 'userID'열이 있다면, 이 열을 자동으로 매칭해서 조인
![[Pasted image 20250304133928.png|400]]
![[Pasted image 20250304133617.png]]
- CustomerID가 같기 때문에, 조건을 지정하지 않아도, CustomerID 기준으로 합침

### 성능
: 교차조인에 비해 훨씬 적은 수의 행을 결과로 반환
: <mark style="background: #FFF3A3A6;">교차 조인에 비해서는 성능이 좋음</mark>
: <span style="background:#d3f8b6">간단하게 뭉뚱그려 데이터를 통합할 때 사용</span>
