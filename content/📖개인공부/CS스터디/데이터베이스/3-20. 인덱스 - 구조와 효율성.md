## 인덱스란?
: <span style="background:rgba(3, 135, 102, 0.2)">데이터 접근(검색, 삽입, 삭제 등) 속도를 향상시키기 위해 사용하는 데이터 구조</span>

ex) 사전의 색인(ㄱ,ㄴ,ㄷ,ㄹ...ㅎ)
특정 단어를 찾으려고 할 때, 색인을 보고 빠르게 찾을 수 있음
-> <mark style="background: #FFF3A3A6;">데이터베이스에서도 인덱스를 사용하면 모든 행을 검색하지 않고도 찾을 수 있음</mark>



## B-트리
___
- <span style="background:rgba(3, 135, 102, 0.2)">일반적으로 DB에서 쓰이는 인덱스 구조</span>

	B트리는 이진검색트리(BST)를 일반화한 자료구조로, 2개 이상의 자식을 가진 노드를 허용하고 탐색에 평균 O(logN) 시간이 걸리며 대수확장성이란 특징으로 인해 <mark style="background: #FFF3A3A6;">더 빠른 시간안에 많은 양의 데이터를 빠르게 찾을 수 있는 구조</mark>이다.
	ex) MySQL, PostgreSQL, Oracle

![[Pasted image 20250305131800.png]]
- 리프노드의 레벨이 같다.
- 차수 : 노드가 가질 수 있는 최대 자식 수
- 노드 내에 여러 키를 저장 -> 자식 노드들 사이의 값을 구분하는 역할
	- 7과 16 사이에는 7이상 16이하의 값이 있음
- 노드가 꽉차면, 삽입을 위해 분할. / 삭제되면 인접 노드와 병함


### B-트리의 작동과정#1
___
- B-트리는 루트노드, 브랜치노드(내부노드), 리프노드로 구성된 트리지만 내부노드가 없다고 가정해서 찾는 과정을 설명하겠습니다.
- 핵심원리는 요소들을 선형적으로 탐색하는 것이 아닌 트리라는 자료구조를 이용해 **“있을법한 노드” 를 기반으로 찾고자 하는 요소를 빠르게 찾는 것입니다.**

ex) E라는 요소를 찾는다고 하자. E가 있을 법한 노드인 2번째 노드로 가서 해당 요소를 찾음

- D -> E 2번만 하면 된다 / 원래는 A, B, C, D, E 5번이 필요함

### B-트리의 대수확장성
___
- **대수확장성이란?** : 트리깊이가, 리프 노드 수에 비해 매우 느리게 성장하는 것.

<span style="background:rgba(3, 135, 102, 0.2)">4차 B-트리의 경우는 기본적으로 엔덱스가 한 깊이씩 증가할 때 마다 최대 인덱스 항목의 수는 4배씩 증가(자식이 차수N에 대해서 N^2씩 늘어나기 때문에)</span>
-> 트리의 높이를 낮추기 위해 그저 자식노드가 2개가 아니라 보다 많은 키를 노드에 넣었음


ex) 차수가 4인 경우 4^10 = 1048576
