- ERD는 DB를 구축할 때 가장 기초적인 뼈대 역할
- 릴레이션 간의 관계들을 정의한 것
- ERD는 시스템의 요구 사항을 기반으로 작성
- ERD를 기반으로 DB를 구축

# ex1) 무무오브레전드의 ERD 구성
___
1. 선수들은 1명의 챔피언을 고를 수 있음
2. 챔피언은 한개 이상의 스킬을 가진다
3. 스킬은 한개 이상의 특성을 갖는다

선수 : 챔피언 = 1 : 1
챔피언은 : 스킬 = 1 : N
스킬 : 특성 = 1 : N

# ex2) 승원부서의 ERD
___
1. 영업사원은 0~n명의 고객을 관리
2. 고객은 0~n개의 주문을 넣을 수 있음
3. 주문은 1~n개의 상품이 들어감

영업사원 : 고객 = 1(0) : N
고객 : 주문 = 1(0) : N
주문 : 상품 = 1 : N

![[Pasted image 20250303225215.png]]
![[Pasted image 20250303225318.png]]
- <span style="background:rgba(205, 244, 105, 0.55)">실제로는</span> 주문 입장에서는 1개의 주문에 N개의 상품이 들어갈 수 있고
- 상품 입장에서는 1개의 상품이 여러개의 주문에 들어갈 수 있으니
- N : M 관계를 가진다.
- <span style="background:rgba(205, 244, 105, 0.55)">OrderDetail에는 주문ID, 상품ID를 FK로 가지고 있는 하나의 테이블이 더 있다</span>



```sql hl:31
-- 영업사원 테이블
CREATE TABLE Salesperson (
	SalespersonID INT PRIMARY KEY,
	Name VARCHAR(100),
	Contact VARCHAR(100)
);
-- 고객 테이블
CREATE TABLE Customer (
	CustomerID INT PRIMARY KEY,
	Name VARCHAR(100),
	Address VARCHAR(200),
	Contact VARCHAR(100),
	SalespersonID INT,
	FOREIGN KEY (SalespersonID) REFERENCES
	Salesperson(SalespersonID)
);
-- 주문 테이블
CREATE TABLE Orders (
	OrderID INT PRIMARY KEY,
	OrderDate DATE,
	ShippingAddress VARCHAR(200),
	CustomerID INT,
	FOREIGN KEY (CustomerID) REFERENCES Customer(CustomerID)
);
-- 상품 테이블
CREATE TABLE Product (
	ProductID INT PRIMARY KEY,
	Name VARCHAR(100),
	Price DECIMAL(10, 2)
);
-- 주문 세부 정보 테이블 (주문과 상품 간의 N:M 관계를 나타냄)
CREATE TABLE Order_Details (
	OrderID INT,
	ProductID INT,
	Quantity INT,
	FOREIGN KEY (OrderID) REFERENCES Orders(OrderID),
	FOREIGN KEY (ProductID) REFERENCES Product(ProductID),
	PRIMARY KEY (OrderID, ProductID)
);
```

- OrderDetail이라는 상품과 주문을 관리하기 위한 테이블이 하나 있다.
- PK로는 {주문ID, 상품ID}를 가진다.
