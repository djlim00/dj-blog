# 문자열 : CHAR
___
- <mark style="background: #FFF3A3A6;">테이블을 생성할 때 고정된 길이로 고정됨</mark>
	- CHAR(5)로 선언하면 'a'(1바이트)를 저장해도 5바이트가 사용됨.
- 길이는 0~255 사이의 값
- 마치 <u>배열 처럼</u> 선언하면 그 만큼의 데이터 크기를 고정해서 저장


# 문자열 : VARCHAR
___
- <mark style="background: #FFF3A3A6;">가변 길이 문자열 (0~65,535)</mark>
- List 처럼 용량이 문자열 크기에 따라서 저장
- <mark style="background: #FFF3A3A6;">VARCHAR(1000)으로 선언해도 'abc'를 저장하면 3 + 1(데이터 길이기록용) -> 4바이트로 저장</mark>
	- 255이상의 문자열이면 데이터 길이기록용 데이터가 2바이트



![[Pasted image 20250303141455.png]]
- CHAR는 해당 데이터 크기를 초과하면 그냥 잘려서 저장된다.

### VARCHAR 최대길이로 설정할 때 주의할 점
1. MySQL의 행의 최대 크기는 65,535임
2. MySQL이 사용하는 문자인코딩 방식은 5.5.3 버전부터 UTF-8을 지원하는 utf8mb4임
	- 이모지(4바이트), 한글(3바이트) 모두 커버
3. <mark style="background: #FFF3A3A6;">괄호안에 들어가는 숫자는 바이트가 아니라 길이제한임.</mark>
	- VARCHAR(65000) - 길이임
