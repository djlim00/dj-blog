# 데이터베이스의 정의와 특징
___
- 일정한 <mark style="background: #FFF3A3A6;">규칙, 규약을 통해 구조화되어 저장되는 데이터의 모음</mark>
- 이러한 데이터베이스를 관리하는 시스템이 DBMS
- 쿼리언어를 이용해서 데이터의 CRUD(Create, Read, Update, Delete)가 가능
	- SQL
	- JSON기반
1. 트랜잭션
2. 동시공유
3. 사용자에 따른 데이터접근 권한 부여
4. 데이터 암호화
5. 백업 및 복구


# 엔터티(Entitiy)
___
- 사람, 장소, 물건, 사건, 개념 등 여러 속성을 지닌 명사
- 데이터베이스에서 실제 세계의 객체나 개념을 모델링한 것
	- 서비스에서 <font color="#d83931">필요한 정보만이 엔터티</font>가 된다
	- 모든 속성을 가져오는 것이 아님
- <mark style="background: #FFF3A3A6;">DB내에서 하나의 고유하게 식별될 수 있는 속성을 가짐</mark>


### 약한 엔터티와 강한 엔터티
- **약한 엔터티** : 독립적으로는 존재할 수 없음. 다른 객체의 존재 여부에 따라서 종속적임(<font color="#d83931">의존적</font>)
	- ex) 강의등록 - 학생 / 강의 와 같은 <mark style="background: #FFF3A3A6;">다른 엔터티가 필요함</mark>
- **강한 엔터티** : 독립적으로 존재할 수 있음.
	- ex) 학생(ID, 번호, 이름, 성별, 이름...etc)



# 릴레이션(Relation)
___
- 데이터베이스에서 정보를 구분하여 저장하는 기본단위
- <mark style="background: #FFF3A3A6;">엔터티의 집합과 그 엔터티의 속성을 표현하는 구조</mark>
=> 엔터티가 데이터베이스에 올라가면 릴레이션이 됨.

관계형데이터베이스(RDBMS)의 릴레이션 => **테이블**
NoSQL 데이터베이스의 릴레이션 =>  **컬렉션**
Redis => key : value

# 속성과 도메인
___

- **속성** : 릴레이션에서 관리하는 구체적이며 <mark style="background: #FFF3A3A6;">고유한 이름을 갖는 정보</mark>
	- ex) 자동차
		- 차 번호
		- 바퀴 수
		- 차 색깔
- **도메인** : 릴레이션의 <mark style="background: #FFF3A3A6;">속성들이 가질 수 있는 값의 집합</mark>
	- ex) 성별
		- {남, 여}
