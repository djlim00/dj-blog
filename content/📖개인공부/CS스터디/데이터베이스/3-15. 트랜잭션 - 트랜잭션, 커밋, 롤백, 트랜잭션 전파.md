# 트랜잭션(Transaction)
___
<span style="background:rgba(3, 135, 102, 0.2)">데이터베이스에서 하나의 논리적 기능을 수행하기 위한 작업의 단위</span>
- 데이터베이스에 접근하는 방법은 쿼리 -> 즉 <span style="background:rgba(3, 135, 102, 0.2)">여러 개의 퀴리들을 하나로 묶는 단위</span>



# 커밋(Commit)
___
<span style="background:rgba(3, 135, 102, 0.2)">여러 쿼리가 성공적으로 처리되었다고 확정하는 명령어</span>
- 트랜잭션 단위로 수행되면 변경된 내용이 모두 영구적으로 저장되는 것을 말함
- "커밋이 수행되었다" == "하나의 트랜잭션이 성공적으로 수행되었다"
![[Pasted image 20250305091922.png]]
- update + insert + delete 쿼리를 하나에 묶어서 트랜잭션 -> 커밋 종료 -> DB에 저장


# 롤백(Rollback)
___
- <span style="background:rgba(3, 135, 102, 0.2)">트랜잭션으로 처리한 하나의 묶음 과정을 일어나기 전으로 돌리는 일(취소)</span>

-> 커밋과 롤백 덕에 <font color="#0070c0">데이터베이스 무결성이 보장</font>된다.
-> 데이터 변경 전에 변경 사항을 쉽게 확인할 수 있고, 해당 작업을 그룹화 할 수 있음,
⚠️ <span style="background:rgba(3, 135, 102, 0.2)">트랜잭션으로 묶는 로직에 외부 API 호출은 자제해야함(롤백도 생각해야 하기 때문에)</span>



# 트랜잭션 전파
___
- 트랜잭션을 수행할 때, 커넥션 단위로 수행하기 때문에 커넥션 객체를 넘겨서 수행해야함.
	- 매번 넘겨주기 귀찮고 어려움
	- <span style="background:rgba(3, 135, 102, 0.2)">따라서 여러 트랜잭션 관련 메서드의 호출을 하나의 트랜잭션에 묶이도록 하는 것을 트랜잭션 전파라고 함</span>
		- ex) Spring의 <u>@Transactional</u>
