
DB에서 조인을 할 때 쓰이는 알고리즘(원리)를 알아보자.
- <mark style="background: #FFF3A3A6;">MySQL에서는 특정한 조인 알고리즘, 예를 들어 중첩루프 조인(Nested Loop Join)을 명시적으로 설정하는 것은 불가능하다.</mark>
- <span style="background:#d3f8b6">해당 DB의 query optimizer가 쿼리를 실행할 때, 조인의 종류 중 자동으로 가장 성능이 좋은 알고리즘으로 결정한다.</span>
	- MySQL뿐만 아니라 PostgreSQL, Oracle, SQL Server

어느정도 조정은 가능 가능함 -> 그러나 어떤 알고리즘을 지정할 수는 없음
- MySQL에서는 <u>해시조인을 쓰고 싶지 않다면 힌트설정 가능</u>
```sql
SET optimizer_switch="hash_join=off";
```


### 중첩루프조인(Nested Loop Join)
___
: <span style="background:#d3f8b6">한 테이블의 각 행에 대해 다른 테이블의 모든 행을 순차적으로 검사하다 조인하는 방법</span>
![[Pasted image 20250304163403.png]]
**시간복잡도 = O(MN)** : 조인된 테이블의 행 수 M,N
**성능** : 소규모 데이터셋에서는 잘 작동하지만, 커질 수록 비효율

![[Pasted image 20250304163752.png]]
- 브루트포스


```sql hl:1
[참고] 블록 중첩 루프조인(BNL, Block Nested Loop)
중첩루프조인에서 발전한 조인 알고리즘으로 한 테이블의 블록(여러 행의 집합)을 메모리에 한번에 로드 ->
조인 버퍼라는 메모리영역에 저장합니다. 이후 해당 블록에 대해 다른 테이블의 각 행에 대해 순회하면서 조건에 맞는 행이 있는 지를 확인 및 행을 조인하는 방법입니다.
```
