- <mark style="background: #FFF3A3A6;">한 테이블(보통 더 작은 테이블)의 행을 사용해서, 해시 테이블을 만든 다음</mark>다른 테이블을 순회하며 해시 테이블을 사용하여 조인한다.
- 하나의 테이블이 서버 메모리에 온전히 들어간다면 중첩루프조인(Nested Loop Join)보다 효율적이지만, <mark style="background: #FFF3A3A6;">메모리가 부족하면 오히려 성능이 낮다.</mark>
- <mark style="background: #FFF3A3A6;">동등(=)조인</mark>에서만 사용가능하다

## 작동방식
___
### 1. 빌드 단계
: 바이트가 더 작은 테이블로 **해시 테이블을 생성**
: <mark style="background: #FFF3A3A6;">각 행의 조인 키에 해시 함수를 적용하여 해시 테이블을 저장</mark>

### 2. 프로브 단계
: 다른 테이블의 각 행에 대한 조인키를 해싱하여
: 앞서 만든 해시값과 비교해서 해시 테이블에서 매칭되는 행을 찾으면
: 해당 행들을 조인결과테이블에 추가

작은 테이블로 테이블 생성 -> 큰 테이블의 조인키를 해싱해서, 해시 테이블에 일치하는 값이 있으면 -> 조인을 함
