- RDBMS는 스키마가 굉장히 엄격하게 지정을 해줘야함
- SQL만을 사용하는 관계형과 다르게 JSON, APQ, Cyppher와 같은 다양한 형식으로 데이터에 접근함
- <mark style="background: #FFF3A3A6;">트랜잭션은 모두 지원</mark>
- 격리성은 DBMS마다 다름
- 스케일링 : 
	- MySQL : 수직 스케일링이 쉬움(서버 성능향상)
	- NoSQL : 수평 스케일링이 쉬움(서버 대수 증가)


#### 스케일링이란
: 데이터베이스 용량을 증가하거나, 그로인한 성능 향상을 위한 기법.
	수직, 수평이 있음

##### 수직 스케일링
-> 데이터베이스 서버의 하드웨어 자원을 확대하는 방법(더 큰 메모리)
	장점 : 구축하려는 시스템에 대한 요구사항을 맞추기 쉬움. 단일 서버에서 자원 증가
	단점 : 서버 자원의 한계. 일정 수준 이상부터는 가성비가 안나옴
-> 요청 처리량이 증가하는 경우에만 적절.
(오토 스케일링)
##### 수평 스케일링
-> 데이터베이스를 여러개의 서버로 분산하는 방법
	장점 : 서버 자원의 한계를 극복. 서버 구축 비용도 적음
	단점 : 시스템 구성이 복잡해지고, 서버 간의 관리와 병렬 처리가 필요. 관리 비용이 많이 듦
-> 요청 처리량이 증가하면 서버를 추가해서 요청 처리를 분산 처리하는 경우 적절함
(오토 스케일링)
