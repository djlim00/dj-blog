# 키(Key)
___
- 데이터 베이스를 설계할 때, key를 기반으로 테이블 간의 관계를 정의하는데 사용
- 기본키, 외래키, 대체키, 후보키, 슈퍼키, 복합키가 있음

![[Pasted image 20250303153013.png]]
- 슈퍼키(**유일성**) -> 후보키(**최소성**) -> 기본키 / 기본키가 못된 키는 대체키
	- <mark style="background: #FFF3A3A6;">유일성</mark> - 중복되는 값 X
	- <mark style="background: #FFF3A3A6;">최소성</mark> - 필드를 조합하지 않고, 최소의 필드를 써서(보통 한개) 키를 형성할 수 있음


### 1. 기본키(Primary key, PK)

- 유일성과 최소성을 만족하는 키이자
- <mark style="background: #FFF3A3A6;">테이블의 각 행(row)를 고유하게 식별하는 필드</mark>
- 각 테이블은 하나의 PK만 있을 수 있고, 보통 'ID'나 '번호' 필드가 기본키

ex) id가 기본키, {id, name}으로도 유일성이 보장되지만, 최소성이 아니다.
![[Pasted image 20250303153403.png]]
```sql hl:5
CREATE TABLE Students (
StudentID INT NOT NULL,
Name VARCHAR(100),
...
PRIMARY KEY (StudentID)
);
```

### 자연키와 인조키
- PK는 보통 자연키와 인조키 중에 골라서 설정(<mark style="background: #FFF3A3A6;">보통은 인조키</mark>)

#### 🌴 자연키(Natural Key)
- 이미 데이터에 존재하고, 실제로 의미가 있는 필드(혹은 필드의 조합)으로 행(row)를 고유하게 식별하는 키
- 자연스럽게 데이터 내에서 발견되는 유일한 값이자 의미있는 정보
- <mark style="background: #FFF3A3A6;">그러나 언젠가는 변하는 속성을 지닙니다(미래에 문제가 생김)</mark>
ex) 주민등록번호, 차량번호, 전화번호


#### 🤖 인조키(Surrogate Key)
- PK의 역할(유일성, 최소성)을 위해 특별히 생성된, 일반적으로 숫자로 이루어진 고유하게 식별되는 키
- 테이블 내에서 자동으로 생성(Auto Increment)되며, 아무 실직적 의미가 없음

### 2. 외래키 (Foreign Key)

- 다른 키의 기본키(PK)를 참조하는 필드
- <mark style="background: #FFF3A3A6;">참조하는 키본키의 값과 일치해야하며, 중복될 수 있고, NULL값을 가질 수도 있음</mark>
![[Pasted image 20250303153950.png]]
- **또 따로 저장하지 않고, 참조 -> 메모리 효율성이 높아짐**

### 3. 대체키 (Altenamte Key)
: PK로 선택되지 않은 후보키


### 4. 후보키 (Candidate Key)
: PK가 될 수 있는 후보들이며, 유일성과 최소성을 만족하는 것
![[Pasted image 20250303154744.png]]
<span style="background:#d3f8b6">=> user_id는 대체키이며 후보키가 된다.</span>


### 5. 슈퍼키(Super Key)
: 테이블 내의 행을 고유하게 식별할 수 있는 유일성을 갖춘 키


### 6. 복합키 (Composite Key)
: 두 개 이상의 필드를 조합하여 만든 키
: 이 키는 각 필드 단독으로는 행을 고유하게 식별 못하지만, 조합하면 식별이 가능함.
