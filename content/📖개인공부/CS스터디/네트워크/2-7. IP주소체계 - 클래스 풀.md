
# 네트워크 구별을 해보자


# 클래스풀
____
- IP주소는 <span style="background:rgba(205, 244, 105, 0.55)">네트워크 주소</span> + <span style="background:rgba(205, 244, 105, 0.55)">호스트 주소</span>
	- 네트워크 주소가 동일 -> 같은 로컬네트워크
	- 호스트 주소 : 호스트를 구분하기 위한 주소(이 네트워크의 어떤 컴퓨터냐?)
	- 네트워크 호스트 = 컴퓨터 네트워크에 연결된 컴퓨터나 기타장치

## 정의
: 네트워크 주소를 매기고, 그에 따라 네트워크의 크기를
: 다르게 구분해서 클래스를 할당하는
: 주소 체계
: 구분하는 기준자(1,2,3옥텟)을
: <mark style="background: #FFF3A3A6;">서브넷 마스크</mark>라고 함.
![[Pasted image 20250217103722.png]]
- <mark style="background: #FFF3A3A6;">클래스 </mark><mark style="background: #FFF3A3A6;">A,B,C는 1,2,3 옥텟까지 네트워크 아이디를 사용</mark>
![[Pasted image 20250217103828.png]]
- 1.0.0.0 이면 A클래스에서는 1에 해당되는 부분만 네트워크 주소, 0.0.0은 호스트 주소

### 클래스 A
![[Pasted image 20250217104138.png]]
- 총 8 x 3 = 24 비트를 호스트 주소로 사용가능
- <span style="background:rgba(205, 244, 105, 0.55)">여기서 127은 루프백 주소 -> 127.0.0.1(localhost)은 본인의 아이피를 지칭하는 주소</span>
- 0.0.0.0은 알 수 없는 대상이라는 의미
- 총 8비트 중에 앞의 비트는 0이므로 2^7-1 = 128-1인 127


### 클래스 B
![[Pasted image 20250217104351.png]]
- 128.1.x.x와 128.0.x.x은 <u>다른 로컬 네트워크</u>를 의미함
- 16비트만 호스트에 할당 가능함(-2는 클래스A와 마찬가지)

### 클래스 C
![[Pasted image 20250217110151.png]]
- 192.0.0.1과 192.0.0.2이 다르다 
- 254개의 호스트를 가질 수 있음

### 계속 주소 중 2개가 빠지는 이유
- <mark style="background: #FFF3A3A6;">맨 앞자리는 네트워크 주소</mark> : 이 네트워크가 뭐야? 192.0.0.0 입니다! 여기서 어떤 호스트를 할당하면 안됨!
- <mark style="background: #FFF3A3A6;">마지막은 브로드캐스팅 주소</mark> : 192.0.0.1 ~ 192.0.0.254까지 전부 보내기 위해서 쓰는 주소
- 즉 2개는 항상 빠짐
