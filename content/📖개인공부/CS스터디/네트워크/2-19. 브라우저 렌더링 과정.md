## 1. DOM 트리와 CSSOM트리 구축
___
### DOM 트리 구축(Document Object Model Tree)
: 하나의 html 페이지는 <mark style="background: #FFF3A3A6;">div, span 등의 요소</mark>를 가짐!
: 이런 요소들이 HTML 파서에 의해서 '구문분석이 됨'
: <mark style="background: #FFF3A3A6;">이러한 하나하나의 요소가 Node인 트리가 저장됨 => DOM 트리</mark>
ex)  div > span, span이 있다면 div가 부모노드고 자식이 span 두개
![[Pasted image 20250224112753.png]]


## CSSON 트리 구축

:<mark style="background: #FFF3A3A6;"> DOM 구축과 **동시에** CSS에 대한 트리도 만들어짐</mark>

## 2. 렌더트리와 렌더레이터 생성
_____
: <mark style="background: #FFF3A3A6;">DOM + CSSOM == 렌더객체</mark>
: 렌더객체의 트리 -> 렌더트리
![[Pasted image 20250224113212.png]]

- display:noneㅇ이 포함된 노드 제거
- font-size 등 상속적인 스타일은 부모노드에만 위치 -> <u>최적화</u>
=> <mark style="background: #FFF3A3A6;">렌더 레이어 완성</mark>
> display:hidden은 여전히 포함됨

![[Pasted image 20250224113341.png]]
> 이때 GPU에서 처리되는 부분은 강제적으로 그래픽 레이어로 분리됨
> -> 너무 많이 그래픽 레이어로해도 좋지 않음

>**Q. 자주 묻는 면접질문 : 렌더트리와 DOM트리는 1:1 대응일까요?**
	아닙니다. DOM트리 > 렌더객체 > 렌더트리가 되는 과정에서 <mark style="background: #FFF3A3A6;">display:none으로 사라지는
	렌더 객체(노드)</mark>들이 있을 수 있기 때문에 <font color="#d83931">1 : 1 대응이 아닙니다.</font>
