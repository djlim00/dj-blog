# 1. 애플리케이션 계층
___
## <span style="background:rgba(205, 244, 105, 0.55)">L7 스위치 == 로드밸런서</span>

: <mark style="background: #FFF3A3A6;">서버의 부하를 분산하는 기기</mark> 여러개의 서버가 있을 때
: 서버 이중화, 보안에 강점이 있음
: IP, Port 뿐만 아니라<mark style="background: #FFF3A3A6;"> URL, 헤더, 쿠키</mark> 등을 기반으로
: 트래픽을 분산
: <mark style="background: #FFF3A3A6;">헬스체크</mark>를 통해서 장애가 발생한 서버를 확인하고
: 해당 서버로 트래픽을 보내지 못하게 함
![[Pasted image 20250220102523.png]]
#### <span style="background:rgba(205, 244, 105, 0.55)">헬스체크</span>
: L4 스위치 또는 L7 스위치 모두 
: 주기적으로 3-way handshake와 같이
: <span style="background:rgba(205, 244, 105, 0.55)">서버에 요청을 보내서 확인</span>

# 2. 전송계층
___
## <span style="background:rgba(205, 244, 105, 0.55)">L4 스위치</span>
: 앞서 설명한 로드밸런서의 특징인 트래픽 분산 등이 가능
: 패킷의 <mark style="background: #FFF3A3A6;">IP주소와 Port번호를 참고해서 적절히 트래픽 분산</mark>
: (L7같은 경우는 URL, 헤더, 쿠키도 참고)
: <mark style="background: #FFF3A3A6;">TCP, UDP의 헤더를 기반으로 우선순위를 판단</mark>해서
: 분산도 가능
: <span style="background:rgba(205, 244, 105, 0.55)">헬스체크</span>도 가능

# 3. 인터넷 계층
___
## <span style="background:rgba(205, 244, 105, 0.55)">라우터</span>
![[Pasted image 20250220112548.png]]
: 라우팅은 <font color="#d83931">하나 이상의 네트워크</font>에서
: <mark style="background: #FFF3A3A6;">경로를 선택하는 프로세스</mark>
: 즉, 이 라우팅을 하는 장치
: 다른 네트워크에 존재하는 장치끼리 데이터를 주고받을 때
: <mark style="background: #FFF3A3A6;">"패킷소모 최소화", "경로 최적화"</mark>

: IP주소를 담아 보냄
: <font color="#d83931">하나의 네트워크 안에서만</font>

: 목적지MAC 주소로 패킷을 보내는 역할


## 브리지
![[Pasted image 20250220113404.png|400]]

# 5. 물리계층
___
## NIC == 랜카드
![[Pasted image 20250220113445.png|400]]
- LAN카드라고 하는 네트워크 인터페이스 카드NIC
- <mark style="background: #FFF3A3A6;">네트워크에 연결하기 위해 PC에 설치된 회로기판</mark>
- <mark style="background: #FFF3A3A6;">PC고유의 식별번호인 MAC주소가 있음</mark>


## 리피터
: 들어오는 약해진 <mark style="background: #FFF3A3A6;">신호 정도를 증폭</mark>해서 다른 쪽으로 전달


## AP(Access Point)
: <mark style="background: #FFF3A3A6;">패킷을 복사하는 장치</mark>
: AP에 유선LAN을 연결한 후 
: 무선 LAN기술을 기반으로 <mark style="background: #FFF3A3A6;">무선 네트워크망을 구축</mark>
