
# 1. 로컬 스토리지
___
##### <span style="background:rgba(205, 244, 105, 0.55)">웹 스토리지 객체로 브라우저(like 크롬) 내에 { key : value } 형태로 오리진에 종속되어 저장되는 데이터(오리진이 같은 브라우저 내에 공유)</span>

1. 크롬에 있는 데이터를 인터넷 익스플로러에서 못 봄!(같은 오리진 내에서만 공유)
2. 하나의 key에 하나의 값만 저장
3. 데이터는 수동으로 삭제하지 않는 한, 없어지지 않음! 만료 날짜가 없음
	- 심지어 컴퓨터를 종료해도 만료X
	- 최대 5MB
4. <mark style="background: #FFF3A3A6;">사용자의 행위를 기억할 때 사용</mark> ex)로그인을 위한 값
5. 서버에 저장되지 않음!(쿠키는 서버에 자동전송)

## 오리진에 종속? 그렇다면 오리진은 뭔가?
___
![[Pasted image 20250219171619.png]]
- 주소창에 주소를 쳐서 들어가는 url에 있는 origin이 있음
	- https://site.com:8080 -> 여기까지가 origin

근데 이제 보통 ww.naver.com 여기까지만 쳐도 https://랑 port번호는 생략이 가능해서 오리진은 그냥 주소라고 생각하면 편함
- http: 80
- https: 443
#### <span style="background:rgba(205, 244, 105, 0.55)">즉 오리진만 같다면 local storage는 공유가 된다!</span>


# 캐싱 - local storage의 쓰임
___
### local storage의 쓰임
1. 로그인 유지 - Token을 local storage에 저장을 함 ->로그인을 유지하고 싶다면 ->  Token을 header-Authorization헤더를 이용해서 서버에 전달함 -> 로그인 유지!!
2. **캐싱** : 사용자가 브라우저 입력하고, 설정한 값을 저장해둠
	- 자동완성 -> UX 증가
	- 새로고침을 해도 유지



# 2. 세션 스토리지
___
- 세션 스토리지는 로컬 스토리지와 매우 유사
- <span style="background:rgba(205, 244, 105, 0.55)">key : value로 오리진에 종속되어서 저장됨</span>
- <mark style="background: #FFF3A3A6;">다만 동일한 오리진에서도 브라우저의 각 탭에 독립적으로 저장됨</mark>
	- 즉 다른 탭에서 세션 스토리지에 저장된 데이터에 접근할 수 없음

##### 1. 하나의 key에 오로지 하나의 value만 저장
##### 2. 최대 저장용량 5MB
##### 3. <span style="background:rgba(205, 244, 105, 0.55)">사용자가 브라우저에서 탭을 닫으면 데이터는 만료</span>



# 쿠키
___
- 사용자가 서버에 요청을 함
- <mark style="background: #FFF3A3A6;">서버가 setCookie라는 응답헤더에 값을 설정함</mark>
- 다음부터는 사용자의 브라우저는 쿠키값을 가짐
- <mark style="background: #FFF3A3A6;">앞으로는 요청헤더에 자동으로 쿠키값이 포함되서 요청에 들어감</mark>

![[Pasted image 20250219173542.png]]
- **Set-Cookie랑 Cookie라는 헤더를 보자!** -> 헤더를 통해서 <font color="#ff0000">추가적인 정보</font>를 보냄
- 서버에서 응답해더로 Set-Cookie헤더를 설정
- 클라이언트에서 요청헤더로 Cookie헤더를 설정
-> 주로 로그인, 장바구니, 사용자 커스터마이징, 사용자 행동분석 등등에 쓰임

#### 서버가 쿠키를 설정하는 것이 바람직하고, 클라이언트가 쿠키를 설정하는 것은 권장하지 않음!
-> 민감한 정보(쿠키에 있는) 해킹 위험이 서버에 비해서 사용자가 훨씬 높음

### 세션쿠키
___
- Expires or Max-Age 속성을 지정하지 <u>않은 것</u> -> <span style="background:rgba(205, 244, 105, 0.55)">브라우저 종료되면 사라짐</span>


### 영구 쿠키
___
- Expires 또는 Max-Age 속성을 <u>지정한 것</u> -> <span style="background:rgba(205, 244, 105, 0.55)">일정 시간이 지나야지만 삭제</span>(브라우저랑은 상관X)


### 쿠키관련 문법
___
![[Pasted image 20250219174700.png]]
- **secure** : https로만 쿠키를 주고 받을 수 있게 함
- **httponly** : 쿠키를 JS로 빼낼 수 없게 함(document.cookie로 접근X)
- **samesite** : 요청이 동일한 도메인에서 시작된 경우에만 쿠키가 애플리케이션으로 전송되도록 허용
![[Pasted image 20250219174948.png]]
- Set-Cookie헤더를 보면, httponly옵션, Secure옵션(Https서버에만 가능하게) 붙였음
- httponle를 붙인 kundol = amumu는 document.cookie로 추출이 불가능하고, loltier = master만 조회가 가능하다

#### <쿠키의 시큐어 코딩>
- 쿠키 - 세션으로 로그인을 처리한다면 다음과 같은 시큐어 코딩을 해야 합니다.
1. cookie에 세션ID를 담을 때 이 세션ID기반으로 클라이언트의 개인정보를 유추할 수 없게 해야 합니다.
2. 자바스크립트로는 파악할 수 없게 http only 옵션을 걸고 https로만 쿠키를 주고받을 수 있게 secure옵션을 걸어야 합니다.
3. <span style="background:rgba(205, 244, 105, 0.55)">일정시간의 세션 타임아웃 걸어야 합니다.</span>
![[Pasted image 20250219215748.png]]
- 이렇게 민감한 금융 쪽 사이트는 이렇게 session timeout을 빡시게 설정해야한다.
