# 🛜 OSI 7 계층이란?

![[Pasted image 20250203150847.png]]
- **<span style="background:#fff88f">네트워크 통신이 일어나는 과정을 7단계로 나눈 것.</span>**
	- 계층을 나눔으로써 통신의 과정을 단계별로 파악할 수 있다.
	- 통신에 이상이 생기면 다른 계층의 소프트웨어와 장비를 건드리지 않고, 문제가 생긴 계층만 고칠 수 있다.
___
# 1계층 - 물리계층(Physical Layer)

- 주로 전기적, 기계적, 기능적인 특성을 이용해서 <font color="#ff0000">통신 케이블(물리적인 연결)</font>로 데이터를 전송한다.
	- 통신 단위 : 0,1로 이루어진 **비트**(전기적으로 On, Off 상태)
- 데이터를 전송하거나 받을 뿐(전기적 신호를 보내거나 쏘기만 함) **데이터가 무엇인지, 에러가 있는지 신경쓰지 않는다**
	- 장비 : <mark style="background: #FFF3A3A6;">통신 케이블</mark>(전송통로), 리피터(신호증폭기), 허브(리피터+여러장비 연결)

___
# 2계층 - 데이터 링크계층(DataLink Layer)

- 물리계층을 통해 송수신되는 정부의 오류와 흐름을 관리 -> <font color="#ff0000">안전한 정보의 전달을 수행</font>하도록 한다.
	- 오류를 발견하고, 재전송하는 기능
	- 데이터 충돌과 같은 흐름제어
	- 이를 위해서 여러가지 <u>데이터 전송 프로토콜</u>을 활용함
- **MAC 주소를 가지고 통신한다.**
	- 통신 단위 : **프레임**
	- 장비 : <mark style="background: #FFF3A3A6;">브리지, 스위치</mark>(연결된 장비들의 MAC주소를 파악하고, 주소에 맞는 포트에만 데이터를 전달)

___
# 3계층 - 네트워크 계층(Network Layer)

- 가장 중요한 기능은 <font color="#ff0000">데이터를 목적지(포트)까지 안전하고 빠르게 전달하는 기능(라우팅)</font>이다.
	- 다양한 프로토콜 & 라우팅 기술이 존재함
- **데이터의 경로를 선택하고 주소를 정하고, 경로에 따라서 패킷을 전달한다.**
	- <font color="#ff0000">경로를 위해서 IP주소를 사용함</font>
	- 통신 단위 : **패킷**
	- 장비 : <mark style="background: #FFF3A3A6;">라우터, 스위치</mark>(Layer3 스위치는 라우팅 기능이 있음)
- 통신을 위해서 여러 노드를 거칠때 마다 경로를 찾아주는 역할 
	- 다양한 데이터를 네트워크를 통해 전달
	- 전송 계층(4계층)이 요구하는 서비스 품질(Qos)을 제공하기 위한 기능적, 절차적 수단을 제공
- 라우팅, 흐름제어, 세그멘테이션, 오류제어, 인터네트워킹을 수행
- **라우팅에 필요한 새로운 헤더를 추가한다.**

데이터를 가야하는 곳에 전송될 수 있도록 한다 -> 인터넷이 가능하게 해줌 ☺️
___
# 4계층 - 전송 계층(Transport Layer)

- <font color="#ff0000">TCP 프로토톨을 이용해, 포트를 열어서 application들이 전송을 할 수 있게 해줌</font>
	- 데이터가 왔다면, 전송계층에서 하나로 합쳐서 5계층으로 전송
- 양 끝단(수신자, 송신자)가 신뢰성 있는 데이터를 주고 받을 수 있게 함
	- 오류제어 : 시퀀스 넘버 기반
- **연결기반**
	- <u>전송 계층이 패킷들의 전송이 유효한지 확인하고, 전송 실패한 패킷들을 다시 전송한다.</u>

ex) TCP
1. 신뢰성(패킷 손실, 중복, 순서바뀜 없음을 보장)
2. 연결지향적(3-way handshake) SYN -> SYN+ACK -> ACK

___
# 5계층 - 세션 계층(Session Layer)

- <font color="#ff0000">데이터타 통신하기 위한 논리적인 연결</font>
- 양 끝단의 응용 프로세스가 통신을 관리하기 위한 방법
	- 동시 송수신 방식(duplex)
	- 반이중 방식(half-duplex)
	- 전이중 방식(full-duplex)
- TCP/IP 세션을 만들고 없에는 책임을 진다.

___
# 6계층 - 표현계층(Presentation Layer)

- 데이터 표현이 상이한 응용프로세스의 독립성을 제공, 암호화
- 코드 간의 번역을 담당, 데이터 형식상 차이를 다루는 책임을 응용계층(7계층)으로부터 덜어줌
ex) EBCDIC -> ASCII
ex2) 해당 파일이 TEXT, MPEG, GIF, JPG... 인지 구분

___
# 7계층 - 응용계층(Application Layer)

- HTTP, FTP, SMTP, POP3...etc 과 같은 프로토콜들이 있다.
	- 통신패킷들은 프로토콜에 의해서 몯 처리된다.
- 일반적인 <font color="#ff0000">응용서비스를 수행</font>
	- 네트워크 소프트웨어 UI부분, 사용자 입출력 부분




### HTTP 프로토콜
- 웹 상에서 웹 서버 및 웹브라우저 상호 간의 데이터 전송을 위한 응용계층 프로토콜
- 텍스트, 이미지, 비디오, 음성 거의 모든 형식의 데이터 전송 가능

1. 요청 / 응답 구조
2. 메세지 교환 형태의 프로토콜
3. 트랜잭션 중심의 비연결성 프로토콜
4. 전송계층 프로토콜 및 사용 포트 번호
	- TCP
	- port:8080

___

L2 장비 : 단순하게, 모든 데이터를 N개의 장비에 보냄
L3 장비 : 들어온 데이터의 IP주소를 파악해서 해당 장비에만 보냄 -> 네트워크 부하가 줄어듦
L4 장비 : 와서는 안되는 데이터는 삭제, 라우팅
