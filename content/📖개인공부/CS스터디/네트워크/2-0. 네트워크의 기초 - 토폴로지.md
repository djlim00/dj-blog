# 네트워크
___
: <span style="background:#fff88f">노드(node)</span>와 <span style="background:#fff88f">링크(link)</span>가 서로 연결되어 있고, 리소스를 공유하는 집합
- 노드 : 서버, 라우터, 스위치 등등 네트워크 장치
- 링크(엣지) : 유선 또는 무선과 같은 연결매체 (wifi나 LAN)
![[Pasted image 20250207131848.png|600]]


# 트래픽
___
: <mark style="background: #FFF3A3A6;">특정 시점에서 링크 내에 "흐르는" 데이터의 양.</mark>
ex) 서버에 저장된 파일을 클라이언트가 다운로드 시 발생되는 데이터의 누적량
- **트래픽** : 흐르는 데이터의 양
	- ex) 10명에서 10MB의 파일을 요청함 -> 트래픽 100MB
	- 단위 : bps(bit per second)
- **처리량** : 링크 내에서 성공적으로 전달된 데이터의 양, 보통 처리되는 트래픽의 양
	- 많은 트래픽을 처리한다 == 많은 처리량을 가진다.
	- 단위 : bps
	- ex) 트래픽이 너무 많아지면, 처리량이 줄어든다.


# 대역폭
___

- 단위 : bps
- ex) 2차선, 4차선

# 네트워크 토폴로지(구성요소)
___
: 노드와 링크가 어떻게 구성되어 있는지를 말합니다.


# 🚍 버스 토폴로지
___

![[Pasted image 20250207133934.png]]
#### 특징
> 1. <mark style="background: #FFF3A3A6;">하나의 회선에 여러개의 노드</mark>
> 2. 노드 추가, 삭제 쉬움
> 3. 설치비용 낮음
#### 장점
> 1. 소규모 네트워크 구축 쉬움
> 2. 한 노드에 장애가 발생해도 다른 노드에 영향 X
#### 단점
> 1. 메인 링크에 많은 트래픽이 생기면, 정체현상 발생가능성 높음(패킷 손실율 높음)
> 2. 메인 링크에 장애가 생기면 큰일!





# 🌟 스타 토폴로지
___
![[Pasted image 20250207134051.png]]
#### 특징 -> 버스토폴과 동일(하나의 회선은 아님)

#### 장점
> 1. 중앙노드가 아닌 노드의 장애는 영향X
> 2. <mark style="background: #FFF3A3A6;">안정성이 높음</mark> -> 중앙이 아닌 노드는 침해가 되어도, 확장되기 어려움(중앙으로 몰리는데, 중앙에는 방화벽이 깐깐하게 해놓음)
> 3. 한 링크에 문제가 생겨도 나머지는 정상적

#### 단점 -> 중앙 망하면 다 망함 🙉



# 🌲 트리 토폴로지
___
![[Pasted image 20250211171832.png]]
### 계층적 구조 : <mark style="background: #FFF3A3A6;">버스 + 스타의 하이브리드 형태</mark>


### 장점
1. 노드 추가, 삭제가 용이(**리프 노드만**)
	- 이외 노드는 어려움
2. 리프 노드에서의 에러는 별로 영향이 없음

### 단점
1. 특정 노드 트래픽 집중시 -> 하위노드도 영향을 받음
2. 루트 노드(제일 상단)가 망하면 망함

⚠ 백본케이블 : 여러 작은 소형 네트워크를 묶어, 대규모 파이프라인을 통해서, 극도로 높은 대역폭으로 다른 네트워크들의 집합과 연결되는 네트워크(대규모 패킷통신망)
=> 이걸 기반으로 트리 토폴로지를 만듦





# 💍 링형 토폴로지
___
![[Pasted image 20250211172424.png]]

### 노드 추가 삭제가 용이함(링크드 리스트처럼)

### 장점
1. 노드 수가 많아져도 데이터 손실이 없음 -> 토큰 권한이 있는지만 확인하면서 지나감 -> 권한이 없으면 데이터 자체를 받지 않음

### 단점
1. <mark style="background: #FFF3A3A6;">하나만 망가져도 망함</mark>
2. 토큰이 없는 노드는 통신에서 제외되서 데이터 공유가 안됨.
