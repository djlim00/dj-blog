
##**What is 암호화?**
![[Pasted image 20250217161054.png]]
: 암호화는 승인된 당사자만 데이터를 이해할 수 있도록
: 데이터를 <mark style="background: #FFF3A3A6;">**"스크램블"**</mark> 하는 방법
: 이를 복호화하려면 서로 동의한 <mark style="background: #FFF3A3A6;">**"키"**</mark> 가 필요함!

### 스크램블
___
- 패턴없이 무작위로 데이터 비트를 섞음
![[Pasted image 20250217161509.png]]
- AES(걍 암호화 방법)로 암호화하면 비트가 10번 스크램블 됨 -> 해독하는데 오래걸림,,!(좋다는 뜻)


### 대칭 암호화
___
##### <span style="background:rgba(205, 244, 105, 0.55)">키를 하나만 사용하는 암호화 방식</span>

ex) hello를 암호화/복호화 - 둘다 하나의 키인 **2jjd8932kd8**을 사용
![[Pasted image 20250217161609.png]]
ex) DES, AES

### 비대칭 암호화(공개키 암호화)
___
#### <span style="background:rgba(205, 244, 105, 0.55)">두 개의 다른 키(공개키, 개인키)로 데이터를 암호화 하거나 서명하고, 공개키는 누구나 사용할 수 있도록 하는 방법</span>

- 공개키로 암호화된 데이터는 개인키로만 복호화가능!
- HTTPS를 가능하게 하는 <mark style="background: #FFF3A3A6;">TLS 프로토콜은 부분적 비대칭 암호화</mark>
	- 비대칭 암호화로 인증, 대칭 암호화로 보안적 통신 시작
ex) RSA, DH

공개키 : 상자
개인키 : 상자를 여는 열쇠
- 공개키는 탈취가 되도 상관없음~, 개인키가 없으면 열 수 없음
##### TLS 핸드셰이크 과정에서 처음 인증할 때는 비대칭 암호화(세션키 생성됨), 그 이후 클라이언트와 서버는" 세션키"라는 키를 기반으로 대칭 암호화 기반 암호화 통신을 함


### 암호화의 필요성
___
- 의도된 수신자나 송신자를 제외하고는 통신을 하이재킹하여
- 읽을 수 없도록 한다
- 민감한 데이터의 유출을 방지, 데이터 무결성 보장
