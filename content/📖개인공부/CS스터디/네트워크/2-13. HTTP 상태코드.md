
# 1xx(정보)
___
서버가 요청을 잘 받았으며 프로세스를 이어가며 처리하는 것
- 100 : 계속하겠음!



# 2xx(성공)
___
서버가 요청을 잘 받았고, 이를 기반으로 클라이언트에게 성공적으로 데이터를 보낸 것을 의미
- 200 OK : 요청이 성공적으로 되었음~
- 201 Created : 요청이 성공적이었으며, 그 결과로 새로운 리소스가 생성됨


# 3xx(리다이렉션)
___
서버가 클라이언트의 요청에 대해 완료를 위해 추가 작업 조치가 필요함
- 301 Moved Permanently :  리소스의 URI가 변경되었음을 의미. 변경된 새로운 URI를 301 상태코드와 함께 줘야함

->자동으로 리다이렉션 된다.


# 4xx(클라이언트 오류)
___
클라이언트가 요청한 페이지를 제공할 수 없거나, <mark style="background: #FFF3A3A6;">클라이언트의 요청이 잘못되어</mark> 결과적으로 요청을 처리할 수 없음
- 401 Bad Request : 서버가 클라이언트의 요청을 이해못함~
- 401 Unauthorized : 클라이언트의 인증이 되지 않음
- 404 Not Found : 요청 받은 컨텐츠를 찾을 수 없음
ex) stackoverflow의 예시

# 😡 5xx(서버 오류)
___
<font color="#ff0000">서버가 클라이언트의 요청을 처리하지 못하는 상태</font>

- 500 Internal Server Error : 서버에 오류가 있음을 의미
- 502 Bad Gateway : 게이트웨이 혹은 프록시서버가 오류가 생김
- 504 Gateway Timeout : 게이트웨이 혹은 프록시 서버가 정해진 Timeout시간 동안 클라이언트의 요청을 처리하지 못함
