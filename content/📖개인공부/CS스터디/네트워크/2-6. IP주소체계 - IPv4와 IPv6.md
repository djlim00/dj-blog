# 👯‍♂️ IPv4와 IPv6
___

### IPv4
___
: 32비트 주소체계 -> 2^32개의 주소를 표현 가능
: 8비트 단위로 4개로 쪼개서 표현(8비트씩 10진수로 표현)
=><mark style="background: #FFF3A3A6;"> 보통 이걸로 부족해서 NAT, 서브네팅 등 여러가지 부수적인 기술이 생김</mark>

![[Pasted image 20250213175006.png]]
그냥 8비트씩 원래는 0010101010이런건데, 끊어서 십진수로 나타낸다고


### IPv6
___
: 128비트 주소체계라서 2^128 개의 주소를 표현
: 주소가 많기 때문에, NAT, 서브네팅 등의 기술이 필요없음
: <mark style="background: #FFF3A3A6;">16비트씩 8개로 표현</mark>

![[Pasted image 20250217093626.png]]
- 앞의 64비트는 <mark style="background: #FFF3A3A6;">네트워크 주소</mark>
- 뒤의 64비트는 <mark style="background: #FFF3A3A6;">인터페이스 주소</mark>
![[Pasted image 20250217093706.png]]

#### 1. IPSec이 내장됨
: 데이터 패킷을 암호화하는 <u>보안 네트워크 프로토콜</u> 제품군이 내장

#### 2. 단순해진 헤더 포멧
: IPv4의 불필요한 헤더는 제거
![[Pasted image 20250217093859.png]]
##### 2-1. IPv4는 체크섬이 있지만, IPv6은 체크섬이 없음
![[Pasted image 20250217094157.png|500]]
- TCP, UDP 계층에서 이미 체크섬이 처리가 되기 때문에
- IP에서 체크할 필요가 없음 -> 그렇기 때문에 IPv6에서는 뺏음
- <font color="#ff0000">UDP는 체크섬이 선택사항</font> -> <mark style="background: #FFF3A3A6;">반드시 IPv6에서 체크섬을 사용한다고 체크해야함</mark>

##### 2-2. IPv4는 헤더 길이가 가변적, IPv6는 헤더길이가 40비트 고정
- 그렇기 때문에 헤더 길이를 알려주는 비트도 필요가 없음

##### 2-3. TTL(Time to Leave)는 HOP liimit으로 변경
=> 무한히 순환하는 패킷을 방지하기 위한 변수
=> 패킷이 라우터를 지날 떄 마다 TTL-1
=> 0이 되면 패킷이 폐기
![[Pasted image 20250217094438.png]]


### IPv4 vs IPv6
___
- IPv6는 IPv4보다 많은 주소를 표현할 수 있고
- <font color="#ff0000">불필요한 헤더가 삭제되어 빠르고, 기본적으로 IPSec이라는 보안 제품군</font>이 포함되어서
- <mark style="background: #FFF3A3A6;">보안적 측면도 좋음</mark>
- 보통은 IPv6를 사용하는 것이 IPv4보다 속도가 빠르지만
- <mark style="background: #FFF3A3A6;">일부 사례에서는 속도가 느린 경우도 있음</mark>
