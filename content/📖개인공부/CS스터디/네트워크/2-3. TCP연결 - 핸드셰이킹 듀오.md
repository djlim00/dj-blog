# 🤝 3-웨이 핸드셰이크 - 연결
___
1. <mark style="background: #FFF3A3A6;">SYN</mark> : 클라이언트는 서버에 클라이언트의 ISN을 담아서 SYN보냄
2. <mark style="background: #FFF3A3A6;">SYN + ACK</mark> : 서버는 클라이언트의 SYN을 수신하고, 서버의 ISN을 보내며 승인번호로 클라이언트의 ISN+1을 보냄
3. <mark style="background: #FFF3A3A6;">ACK</mark> : 클라이언트는 서버의 ISN+1한 값인 승인번호를 담아 ACK를 서버에 보냄

![[Pasted image 20250213132627.png]]
[참고] ISN : TCP(Transmission Control Protocol) 기반 데이터 통신에서 각각의 <mark style="background: #FFF3A3A6;">새 연결에 할당된 고유한 32비트 시퀀스 번호</mark>를 나타냅니다. TCP 연결을 통해 전송되는 다른 데이터 바이트와 충돌하지 않는 시퀀스번호를 할당하는 데 도움이 됩니다.
ex) 와이어샤크로 보는 TCP 연결
- **즉 ACK를 보낼때는 받은 ISN+1을 해서 보냄**

# 👐 4-웨이 핸드셰이크 - 해제
___
1. 먼저 클라이언트가 연결을 닫으려고 할 때 FIN으로 설정된 세그먼트를 보냅니다.그리고 클라이언트는 FIN_WAIT_1 상태로 들어가고 서버의 응답을 기다림.

2. 서버는 클라이언트로 ACK라는 승인 세그먼트를 보내고 CLOSE_WAIT 상태에들어갑니다. 클라이언트가 세그먼트를 받으면 FINWAIT__2 상태에 들어감.

3. 서버는 LAST_ACK상태가 되며 일정 시간 이후에 클라이언트에 FIN이라는세그먼트를 보냄

4. 클라이언트는 TIME_WAIT 상태가 되고 다시 서버로 ACK를 보내서 서버는 CLOSED상태가 되며 이후 클라이언트는 어느 정도 시간(TIME_WAIT으로 설정된 시간)을 대기한 후 연결이 닫힘.

![[Pasted image 20250213133656.png]]

클라 FIN_WAIT_1 FIN -> 서버 CLOSE_WAIT ACK-> 클라 FIN_WAIT_2
서버 LAST_ACK FIN -> 클라 TIME_WAIT ACK -> 서버 CLOSER -> 클라 CLOSED 

### TIME_WAIT
: <mark style="background: #FFF3A3A6;">지연 패킷 등이 발생했을 때 데이터 무결성을 해결하기 위해 패킷을 기다리는 시간</mark>
- 2 * MSL(최대 패킷수명)동안 기다림 ->  사실 OS마다 다름
- 또한 CLOSED가 아니라 LAST_ACK로 되어 있으면 그 다음 연결 때 오류가 발생하기 때문에, 일정 시간 기다려야함
