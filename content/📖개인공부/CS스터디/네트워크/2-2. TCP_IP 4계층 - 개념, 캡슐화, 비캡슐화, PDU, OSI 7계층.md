# 4️⃣ TCP/IP 4계층 개념
___

### 인터넷 상에서 데이터를 주고받을 때 쓰는 독립적인 프로토콜의 집합

- TCP랑 IP를 통해서 주로 주고 받으니까 이런 말을 쓰는거다~
![[Pasted image 20250211184918.png|300]]
- 요렇게 구성이 되어 있어요~ 서로 각자의 <u>역할과 책임</u>이 있어요~

### 1. 애플리케이션 계층
: HTTP를 통해서 <mark style="background: #FFF3A3A6;">서비스</mark>를 실질적으로 제공하는 계층(이메일, 게임..etc)

### 2. 전송계층
: HTTP에서 받은 메시지를 기반으로
: <mark style="background: #FFF3A3A6;">세그먼트 또는 데이터그램</mark>으로 데이터를 쪼개고
: 데이터가 손실없이 도착할 수 있도록 해주는 계층
: **<u>오류도 본다</u>**

### 3. 인터넷 계층
: IP와 같이
: 한 노드에서 다른 노드로 전송계층에서 받은
: 세그먼트 혹은 데이터그램을
: <mark style="background: #FFF3A3A6;">패킷화</mark> 해서 목적지로 전송하는 역할

### 4. 링크(네트워크 엑세스) 계층
: 전선, 광섬유, 무선
: <mark style="background: #FFF3A3A6;">데이터가 물리적</mark>으로 
: 전송되는 방식을 정의
-> 진짜 물리적으로 연결

### 계층간은 독립적이므로, 서로에게 영향을 주지 않는다.
ex) HTTP버전이 바뀌었다고, 뭐 크롬(애플리케이션)을 다시 깔고 그러지 않는다
ex) 전선 바꾸었다고, 뭐 컴터 설정할 것 없다.



# 💊 캡슐화 & 비캡슐화

### 캡슐화
: 송신자가 수신자에게
: 데이터를 보낼 때

## 😡 PDU (프로토콜 데이터 유닛) - <font color="#ff0000">계층마다 데이터 단위 다름</font>
___

>**1. 애플리케이션 : 메시지**
>**2. 전송 : 세그먼트(TCP에서), 데이터그램(UDP에서)**
>**3. 인터넷 : 패킷**
>**4. 링크 : 프레임(데이터 링크), 비트(물리)**

- <mark style="background: #FFF3A3A6;">세그먼트 == 데이터그램</mark>
	: 적절하게 데이터를 쪼갠 것
- 패킷 : 세그먼트에 SP와 DP가 포함된 IP헤더가 추가
- 프레임 : MAC주소 헤더와 CRC/체크섬 트레일러가 붙은 조각

#### IP헤더 - 송신자, 수신자 정보 -> <span style="background:rgba(205, 244, 105, 0.55)">인터넷 계층에서 추가되는 헤더</span>
![[Pasted image 20250211191247.png]]
- Source IP Address : 송신자 주소
- Destination IP Address : 수신자 주소

#### 패킷 & 프레임
![[Pasted image 20250211191330.png|500]]
- **SP, DP** : 그냥 IP주소 - IPv4 기준
- 프레임 : MAC주소 헤더랑, CRC, 체크섬

### CRC/체크섬 트레일러
___
: <mark style="background: #FFF3A3A6;">데이터 오류를 감지하기 위해 수학적 함수가 적용된 값</mark>
: 링크의 오류(과도한 트래픽)으로 인한 데이터 손상 감지

# MTU와 MSS와 PMTUD
___

## 1. MTU (패킷 전송크기)
![[Pasted image 20250211203904.png]]
#### 패킷으로 쪼갤 때, MTU(Maximun Transmission Unit)을 기반으로 쪼갠다!
=> 최대 적재량, 터널 높이 같은 느낌

- <mark style="background: #FFF3A3A6;">두 장치만이 아니라 중간의 모든 라우터, 스위치, 서버와 같은 장비의 MTU도 신경을 써야함</mark>
	- 가장 MTU가 낮은 장비를 기준으로 보내야한다.
	- 제일 낮은 터널을 지날 수 있게 차를 만들어야한다.

#### 🥸 패킷이 분할되지 않는 경우
: 패킷을 분할할 수 없어서, 네트워크 경로 상의 장비의 MTU를 초과하면, 중간에 분할하는 것이 아니라 아예 전달을 하지 않을 수도 있다.
- IPv4는 분할 가능
![[Pasted image 20250211203843.png|300]]
- IPv6는 분할 불가능


## 2. MSS (MTU - IP헤더 - TCP헤더)
![[Pasted image 20250211204003.png]]
- <span style="background:rgba(205, 244, 105, 0.55)">MTU에서 위의 헤더를 뺀 데이터 크기</span>
- 일반적으로 MTU는 1500바이트이며 MSS는 1460바이트
	- <font color="#ff0000">따라서, 데이터는 보통 MSS기준으로 1460이하로 보낸다</font>


## 3. PMTUD

: 패킷을 분할할 수가 없어서, 보내지 못할 때
: <mark style="background: #FFF3A3A6;">MTU를 찾아 가면서, MTU에 맞게 다시 쪼개서 보내는 것</mark>

# 인터넷 계층
___
- IP, ICMP, ARP가 대표적이며 한 노드에서 다른 노드로 전송 계층에서 받은 세그먼트 또는 데이터그램을 <mark style="background: #FFF3A3A6;">패킷화 하여 전송</mark>합니다.

## ICMP - 통신 테스팅 프로토콜
![[Pasted image 20250213131940.png]]
- ICMP ( Internet Control Message Protocol ) 는 노드와 노드 사이에서 <mark style="background: #FFF3A3A6;">통신이 잘되나를 확인할 때 쓰는 프로토콜</mark>입니다.
- 이는 <mark style="background: #FFF3A3A6;">데이터를 교환하는데 사용되지 않는 프로토콜</mark>입니다.

ex) www.google.com 에 ping 날려볼 때 ICMP로 테스팅 된다.
