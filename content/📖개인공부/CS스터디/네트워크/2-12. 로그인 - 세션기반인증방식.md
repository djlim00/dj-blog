## 세션기반 로그인 프로세스
___
-  <mark style="background: #FFF3A3A6;">세션</mark> : 서버와 클라이언트의 연결이 활성화된 상태
-  <mark style="background: #FFF3A3A6;">세션ID</mark> : 웹 서버 또는 DB에 저장되는 클라이언트에 대한 유니크한 ID

### 과정
![[Pasted image 20250219222551.png]]
1. 처음 로그인 >> 세션ID 생성 >> 서버에서 <mark style="background: #FFF3A3A6;">세션ID(SHA256으로 암호화된 문자)</mark>를 쿠키로 설정해서 클라이언트에 전달
2. 클라이언트가 서버에 요청을 보낼 때 해당 <mark style="background: #FFF3A3A6;">세션ID를 쿠키에 담아서</mark> 전에 로그인했던 아이디인지를 확인
3. 서버는 쿠키에 담긴 세션ID를 기반으로 유효한 값인지를 WAS or DB를 조회해서 확인
4. 로그인을 유지

#### DB에 저장되는 경우
: <font color="#ff0000">직렬화와 역직렬화에 대한 Cost</font>

#### WAS에 저장하는 경우
: 사용자가 많아지면 서버 과부화(메모리 소모)
