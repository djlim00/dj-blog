# 멱등성을 가지는 메서드
___
## 1. Get
: 서버에서 정보를 가져오는데 사용
-> 여러개의 정보를 가져와도 결국 가져오는 정보는 같음(멱등)
- <span style="background:#fff88f">안전한 메서드</span>


## 2. HEAD - GET요청 확인용 메서드
: GET 요청이 생성하는 응답의 헤더를 검색.
- 보통 리소스 전체를 다운로드하는 오버헤드를 피하고
- 어떤 유형의 데이터가 반환되는지, 해당 리소스가 있는지, 수정날짜 등을 확인하기 위해 쓰인다
- <span style="background:#fff88f">안전한 메서드</span>
![[Pasted image 20250224224826.png]]


## 3. OPTIONS - 서버 통신 옵션 확인
: 대상 리소스의 통신 옵션(서버에서 지원하는 HTTP메서드 등)을 확인
-> 그냥 확인용이니까 <span style="background:#fff88f">"안전한 메서드"</span>


## 4. PUT
: 리소스를 완전히 다른 버전으로 대체
: 계속해서 리소스가 대체됨 -> 첫 번째 요청의 대체가 그냥 반복됨(GET과 다를바 없음)


## 5. DELETE
: 리소스 삭제
: 아무리 삭제해도 그냥 삭제 상태임


# 멱등성을 가지지 않는 메서드
___
## 1. POST
: 새로운 리소스를 생성함
: <span style="background:#fff88f">여러개의 리소스가 생성됨(멱등X)</span>


## 2. PATCH
: 리소스를 부분적으로 수정(업데이트)
: cnt++를 하는 로직이라면 -> 계속해서 증가함 (멱등 X)
- 단순히 이름을 업데이트한다면 (멱등 O)
