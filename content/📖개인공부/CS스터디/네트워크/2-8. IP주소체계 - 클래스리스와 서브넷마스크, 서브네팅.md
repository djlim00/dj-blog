# 클래스리스
___
#### 서브넷마스크를 중심으로 어디까지가 네트워크 주소고, 어디까지가 호스트주소인지를 나눈다.
- **서브네팅** : 네트워크를 나눈다
- **서브넷** : 쪼개긴 네트워크(서브네트워크)
- **서브넷마스크** : <mark style="background: #FFF3A3A6;">서브네트워크를 위한 비트 마스크</mark>
=> <span style="background:rgba(205, 244, 105, 0.55)">현재에는 클래스리스를 사용함</span>

### 서브넷 마스크
![[Pasted image 20250217111304.png]]
- 네트워크 부분만 모두 1
- 호스트 부분은 모두 0
ex) 192.168.50.112 실제 주소 / 255.255.255.0 서브넷 마스크
이걸 AND(&) 연산을 하면 -> 192.168.50.<font color="#ff0000">0</font> 이 뽑힘
클래스 A : 255.0.0.0
클래스 B : 255.255.0.0
클래스 C : 255.255.255.0

![[Pasted image 20250217112415.png|500]]
#### ex) **255.255.255.192**
- 십진수: 255.255.255.192
- 2진수: `11111111.11111111.11111111.11000000`
- 192 = 11000000 = 2^7 + 2^6 + 0+0+0+0.. = 192
- `1`의 개수 = **26** → CIDR **/26**

### 🥸 10개의 사무실이 있으며 사무실당 12개의 장치를 연결해야 한다면?
- <span style="background:rgba(205, 244, 105, 0.55)">하나의 네트워크당 123.12.12.12 /28 -> 32- 28 =2 즉 2^4 =16개의 네트워크를 확보</span>
- 이를 통해서 16-2 = 14 개의 장치를 확보할 수 있음
- <font color="#ff0000">세분화가 가능함!!</font>
