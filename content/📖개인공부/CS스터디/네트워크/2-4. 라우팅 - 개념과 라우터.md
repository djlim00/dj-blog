# 라우팅 테이블
___
: IP 주소를 기반으로
: 라우터의 위치를 저장한
: 테이블 또는 데이터베이스
: 다양한 네트워크에 대한 정보와 연결하는 방법이 포함

![[Pasted image 20250213170505.png]]
- <mark style="background: #FFF3A3A6;">라우팅 테이블에는 Destination 101.25.67.0에 대한 정보가 있음</mark>

### 라우팅 테이블의 구성요소
![[Pasted image 20250213171151.png]]
1. <mark style="background: #FFF3A3A6;">Network Destination</mark> : 목적지 네트워크의 IP 주소
2. <mark style="background: #FFF3A3A6;">Netmask</mark> : 대상 주소를 설명할 때 쓰이는 값
3. <mark style="background: #FFF3A3A6;">Gateway</mark> : 이 장치와 연겨려된 홉, 즉 패킷이 전달되는 다음 IP주소(외부 네트워크와 연결된 장치). 만약 목적지가 로컬 네트워크라면 "연결됨(connected)"라고 표기되며, 다른 네트워크면 해당 네트워크의 게이트웨이
4. <mark style="background: #FFF3A3A6;">Interface</mark> : 게이트웨이로 가기 위해 거치는 장치
5. <mark style="background: #FFF3A3A6;">Metric</mark> : 우선순위, 패킷 전송을 위해 최적의 경로가 선택되도록 참고되는 값. 동일한 라우팅 테이블 요소가 2개 있을 때, 이 값이 낮은 요소가 선택. 일반적으로 hop count가 들어가며, 지연시간, 처리량이 들어갈 수도 있음
	
### <span style="background:rgba(205, 244, 105, 0.55)">Gateway</span>
: 프로토콜 변환기라고도 하며, 네트워크와 네트워크를 잇는 장치 -> 라우터와 비슷
![[Pasted image 20250213171415.png]]

### <span style="background:rgba(205, 244, 105, 0.55)">홉(hop)</span>
: 네트워크 출발지 -> 목적지 사이에 위차한 장치(라우터, 스위치 등등)의 개수
