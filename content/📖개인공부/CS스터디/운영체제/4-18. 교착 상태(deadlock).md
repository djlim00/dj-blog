- <span style="background:rgba(3, 135, 102, 0.2)">두 개 이상의 프로세스들이 서로가 가진 자원을 기다리며 중단된 상태.</span>
- 각 프로세스는 서로가 원하는 자원을 유지한채 다른 프로세스의 자원을 얻기를 기다리는 것이다.(이러지도 저러지도 못함)

## 교착 상태의 원인
___

#### 1. 상호 배제(mutual exclusion)
: 주어진 시간 내에 하나의 프로세스만 자원을 독점할 수 있음. 즉 다른 프로세스는 접근이 불가능함.

#### 2. 점유 대기
: 특정 프로세스가 점유한 자원을 다른 프로세스가 요청하며 대기하는 상태.

#### 3. 비선점
: 다른 프로세스의 자원을 강제적으로 가져올 수 없음.

#### 4. 환형 대기 
: 프로세스 A는 프로세스 B의 자원을 요구하고, 프로세스 B는 프로세스 A의 자원을 요구하는 등, 서로가 서로의 자원을 요구하는 상황.


😡 **4가지 상황이 모두 충족되야지 Deadlock -> 하나만 해소되도 일어나지 않음!**



## 교착 상태 해결방법
___
1. 애초에 4가지 조건이 성립되지 않도록 설계
2. <mark style="background: #FFF3A3A6;">교착 상태 가능성이 없을 때만 자원 할당</mark>되며, 프로세스당 요청할 자원들의 최대치를 통해서, 자원 할당가능 여부를 파악하는 **'은행원 알고리즘'**을 쓴다.
3. Deadlock이 발생하면, <mark style="background: #FFF3A3A6;">사이클이 있는지 찾아보고</mark> 이에 관련한 프로세스를 한 개씩 지운다.
4. <span style="background:rgba(3, 135, 102, 0.2)">Deadlock은 매우 드물게 발생하기 때문에, 이를 처리하는 비용이 더 커서 교착 상태가 발생하며 사용자가 작업을 종료함.</span>
	- 현대 OS는 이 방법을 채택 -> '응답없음'
