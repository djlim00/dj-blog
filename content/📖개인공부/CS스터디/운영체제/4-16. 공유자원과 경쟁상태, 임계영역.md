### 공유자원
___
공유자원(Shared Resource)란 시스템 안에서 각 프로세스, 스레드가 함께 접근할 수 있는 모니터, 프린터, 메모리, 파일, 데이터 등의 자원이나 변수를 의미

### 경쟁상태 (race condition)
___
경쟁상태(Race Condition)는 <mark style="background: #FFF3A3A6;">공유자원을 둘 이상의 프로세스 또는 쓰레드가 동시에 읽거나 쓰는 상황을 말함.</mark>
<mark style="background: #FFF3A3A6;">동시에 접근을 시도할 때의 타이밍이 예상되는 결과 값에 영향을 줄 수 있는 상태</mark>

### 임계영역 (critical section)
____
임계영역은 둘 이상의 프로세스 또는 쓰레드가 공유자원에 접근할 때 순서 등의 이유로 달라지는 코드영역을 말합니다. 
<mark style="background: #FFF3A3A6;">이 영역은 한 번에 둘 이상의 프로세스나 스레드가 들어갈 수 없게 설계</mark>

![[Pasted image 20250324101155.png]]

```java title:"Java Race Condition Example"
public class Counter { 
	private int count = 0; 
	
	public void increment() {
		count++; // 여기에 lock이 없음 
	} 
	
	public int getCount() { return count; } 
	
	public static void main(String[] args) throws InterruptedException {
		Counter counter = new Counter();
		
		Thread t1 = new Thread(() -> { 
			for (int i = 0; i < 1000; i++) {
				counter.increment(); 
			} 
		}); 
		
		Thread t2 = new Thread(() -> { 
			for (int i = 0; i < 1000; i++) {
				counter.increment(); 
			} 
		}); 
		
		t1.start();
		t2.start(); 
		
		t1.join(); 
		t2.join(); 
		
		System.out.println("Final count: " + counter.getCount()); 
		// 예상 결과는 2000이지만, 경쟁 조건으로 인해 다른 값이 출력될 수 있음
	} 
}
```
![[Pasted image 20250324102356.png]]
- 1000번씩 count를 증가시키니까 2000이 되야하는데 쓰레드들끼리 race condition이 발생하면서 1906이 출력된 모습


## 경쟁 상태 관리의 중요성
____
경쟁 상태를 잘 해결하지 못하면 데이터 정합성, 데이터 무결성을 지키지 못할 수 있음.

#### 1. 데이터 정합성
: <mark style="background: #FFF3A3A6;">예상되는 데이터의 값과 다른 것</mark>
ex) 읽을 때 0원이어야하는데 1000원임


#### 2. 데이터 무결성
: <mark style="background: #FFF3A3A6;">데이터의 어떠한 규칙이 위반하면 안되는 것.</mark>
ex) 은행에서 0원인데 출금

![[Pasted image 20250324102622.png]]

경쟁상태를 해결하는 대표적인 방법임!
- 상호배제
- 한정대기
- 진행의 융통성
위의 조건을 만족시키며 경쟁상태를 해결

1. <mark style="background: #FFF3A3A6;">상호배제(mutual exclusion)</mark> : 한 프로세스가 critical section에 들어갔을 때, 다른 프로세스는 <font color="#ff0000">진입 불가능</font>
2. <mark style="background: #FFF3A3A6;">한정 대기(bouded waiting)</mark> : 특정 프로세스가 critical section에 진입을 요청한 후 해당 요청이 승인되기 전까지 <font color="#ff0000">다른 프로세스가 임계영역에 진입하는 횟수를 제한</font>.
   이를 통해 특정 프로세스가 영원히 임계영역에 들어가지 못하는 것을 방지
3. 진행의 융통성(progress) : 만약 어떠한 프로세스도 critical section을 사용하지 않는다면, critical section 외부의 어떠한 프로세스도 진입가능하며. 이때 프로세스끼리 서로 방해하지 않음.


## 뮤텍스
___
- 공유자원을 lock()을 통해 잠금설정하고, 사용한 후에 unlock()을 통해 잠금해제가 되는 객체 lock을 기반으로 경쟁상태를 해결
- 잠금이 설정되면 다른 프로세스나 쓰레드는 잠긴 critical section에 진입할 수 없고, 해제는 그와 반대
- 한번에 하나의 프로세스만 critical section에 있을 수 있음
ex) 한칸 뿐인 탈의실
