IPC는 프로세스끼리 데이터를 주고 받고 공유하는 데이터를 관리하는 메커니즘.
IPD의 종류로는
1. 공유 메모리
2. 파일
3. 소켓
4. 파이프
5. 메세지 큐
ex) 브라우저를 띄워서 네이버 서버와 HTTP 통신해서 html 등의 파일을 가져오는 것도 IPC의 일종


### 1. 공유 메모리
____
- 여러 프로세스가 서로 통신할 수 있도록 메모리를 공유하는 것.
- 어떤 매개체도 없고 메모리 그 자체를 공유 -> <mark style="background: #FFF3A3A6;">데이터 복사 오버헤드 X</mark>
	- 나머지 방법은 복사를 해서 전달
- 여러 프로세스가 같은 메모리 영역을 공유 -> <mark style="background: #FFF3A3A6;">동기화(Synchronization)이 필요</mark>

<font color="#ff0000">"가장 빠른 통신방법"</font>



### 2. 파일
____
- 디스크에 저장된 데이터를 기반으로 통신
- <font color="#ff0000">요즘은 안씀</font>


### 3. 소켓
____
- 네트워크 인터페이스(TCP, UDP, HTTP 등)을 기반으로 통신


### 3. 파이프
___
- 익명 파이브와, 명명 파이프 두개가 있음
#### 3-1. 익명 파이프
: <span style="background:rgba(3, 135, 102, 0.2)">프로세스 사이에 FIFO 기반의 통신 채널을 만들어서 통신하는 것.</span>
![[Pasted image 20250320132946.png]]
-> pipe  하나당 단반향 통신이기 때문에 <span style="background:rgba(3, 135, 102, 0.2)">2개를 둬야지 양방향 통신이 가능</span>함.

![[Pasted image 20250320133010.png]]
- 부모, 자식 프로세스 간에만 사용할 수 있으며 <mark style="background: #FFF3A3A6;">다른 네트워크 상에서는 사용이 불가</mark>능
- 파이프의 <mark style="background: #FFF3A3A6;">데이터 용량은 제한</mark>되어 있음
- 쓰기 프로세스가 읽기 프로세스보다 더 빠르게 데이터를 쓸 수 없음.
	- 파이프로 들어오는 데이터의 양과 나오는 데이터의 양은 같아야함(아니면 손실되어서 줄어들던가)

#### 3-2. 명명 파이프
: 익명 파이프의 확장. <span style="background:rgba(3, 135, 102, 0.2)">다른 네트워크 상에서도 통신이 가능함.</span>
: 보통 서버, 클라이언트용 파이프를 구분해서 동작

### 4. 메시지 큐
____
- 메시지를 <span style="background:rgba(3, 135, 102, 0.2)">Queue 자료구조 형태로 관리하는 버퍼를 만들어서 통신</span>

![[Pasted image 20250320133753.png]]
1. 프로세스가 메시지를 보내거나 받기 전에 큐를 초기화
2. 보내는 프로세스(sender)의 메시지는 큐에 <u>복사</u>되어 바는 프로세스(receiver)에 전달.
