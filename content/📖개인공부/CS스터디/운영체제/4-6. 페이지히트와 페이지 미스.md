
### 페이지 히트(Page Hit)
- 프로세스가 요청한 해당 페이지가 이미 메모리에 적재되어 있어 바로 접근할 수 있는 상황

### 페이지 미스
- 프로세스가 요청한 해당 페이지가 메모리에 없는 상황

### 페이지미스와 페이지 폴트
-> <mark style="background: #FFF3A3A6;">페이지 미스는 페이지 폴트를 초래할 수 있음</mark>
- **페이지 폴트** : 요청된 페이지가 메모리에 없어 운영체제가 보조 저장 장치에서 해당 페이지를 메<mark style="background: #FFF3A3A6;">모리로 로드해야하는 상황</mark>



### 😎 모든 페이지폴트는 페이지 미스로 일어나지만, 모든 페이지미스가 페이지폴트를 일으키진 않는다.

1. **접근권한 오류**
	- 해당 페이지에 대한 접근 권한이 없거나(쓰기 금지에 쓰기 시도) 잘못된 접근이 시도되었을 경우 오류로 처리
	- 페이지미스가 발생하지만 메모리에 로드하지 않고, 접근권한위반 오류만 발생시킴

2. **성능최적화를 위해 미리 로드 할 때의 페이지 미스**
	- 어떤 시스템 또는 운영체제에서 성능최적화를 위해 <mark style="background: #FFF3A3A6;">페이지를 실제로 사용하기 전에 미리 요청</mark>(프리패칭)하는 최적화 기법을 사용하기도 함.
	- 이때 <mark style="background: #FFF3A3A6;">실제 데이터 사용전에 다른 데이터로 덮어쓰게 되면 페이지 미스는 발생하지만, 사용되지 않아 페이지 폴트로 발전되지는 않음.</mark>
		- 미리 깔아놓았는데, 다른 데이터를 쓰게 되면 지금 메모리에는 내가 요청한 다른 데이터는 없으니까 폴트가 뜨진 않음
