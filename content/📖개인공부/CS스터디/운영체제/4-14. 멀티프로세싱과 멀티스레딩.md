
## 멀티프로세싱
____
- 여러개의 '프로세스', 즉 멀티프로세스를 통해 동시에 두 가지 이상의 일을 수행하는 것.
- 프로세스는 격리성이 있기 떄문에, 일부 프로세스에 문제가 생겨도 다른 프로세스에 영향을 끼치지 않음

### 장점
1. <span style="background:rgba(3, 135, 102, 0.2)">신뢰성과 격리성이 높음</span>
2. <span style="background:rgba(3, 135, 102, 0.2)">프로그램 병렬성</span> -> 여러개의 CPU코어를 활용해서 다중 CPU 시스템에서 프로세를 병렬적으로 실행 -> <font color="#ff0000">이건 멀티 스레드도 마찬가지</font>
![[Pasted image 20250320114642.png|600]]
3. <span style="background:rgba(3, 135, 102, 0.2)">시스템 확장성</span> : 각 프로세스가 독립적이라서 새로운 모듈을 추가해도 영향 X, 시스템 규모확장에 유리함
	- 하나의 컴퓨터에 여러개의 서버 프로세스를 띄워서 요청을 분산 -> 로드밸런싱


### 단점
1. 컨텍스트 스위칭 오버헤드가 발생함
2. 자원 공유의 비효율성


## 멀티스레딩
____
- **프로세스 내** 작업을 멀티스레드로 처리하는 기법.

### 장점
1. 스레드끼리 <span style="background:rgba(3, 135, 102, 0.2)">서로 자원을 공유하고 프로세스보다 가볍기 떄문에 효율성이 높음</span>
	![[Pasted image 20250320131023.png|500]]
2. 컨텍스트 스위칭 비용 감소
	- 스레드에도 오버헤드가 존재하지만, 프로세스 컨텍스트 스위칭 오버헤드보다 비용이 훨씬 낮음
3. <mark style="background: #FFF3A3A6;">응답 시간 단축</mark>


### 단점
1.  <mark style="background: #FFF3A3A6;">안정성</mark> : <u>하나의 스레드에 문제가 생기면 다른 스레드에도 영향을 끼쳐 프로세스에 영향을 줄 수 있음</u>
2. 데드락(Deadlock) : 스레드들 끼리 자원을 공유하기 때문에 교착상태에 빠질 수 있음
	 ![[Pasted image 20250320131233.png]]
3. 디버깅이 어려움 : 디버깅 하는 도중에 다른 스레드가 실행되서 예상치 못한 결과가 발생할 수 있고 여러 개의 스레드가 동시에 실행되기 때문에 추적하기가 어려움
4. 운영체제의 지원 필요 - 과거의 OS들은 멀티 스레딩을 지원하지 않았음.
