# 시스템 콜
___
시스템 콜이란 운영체제가 <mark style="background: #FFF3A3A6;">커널에 접근하기 위한 인터페이스</mark>이며
<mark style="background: #FFF3A3A6;">유저 프로그램과 운영체제의 서비스를 받기 위해 커널 함수를 호출할 때 이를 거쳐서 호출</mark>되도록 설계되어 있음
- 프로세스 관리, 생성, 삭제
- 파일관리
- 디바이스 관리
- 시간 날짜 관리
- 프로세스간 통신

ex) kill : 프로세스 종료하는 시스템 콜 -> 커널 함수가 호출되어서 프로세스가 종료


### 시스템 콜의 과정

**ex) 유저 프로그램이 IO 요청을 위한 트랩(인터럽트)를 발동**
1. 유저 프로그램이 IO 요청 트랩발동!
2. 올바른 IO 요청인지 확인
3. <mark style="background: #FFF3A3A6;">유저 모드가 시스템콜을 통해 커널 모드로 변환</mark>(권한이 생김)
4. 커널 함수 실행
![[Pasted image 20250317133607.png]]
- modebit=1이면 유저 모드기 떄문에, 권한이 없다가 
- 시스템콜에 의해서 modebit=0으로 바꾸면 커널 모드가 되면서 -> 컴퓨터의 디스크에 접근해서
- 파일을 읽어올 수 있음!
- 다시 유저모드로 변환(시스템콜에 의해서)

#### 커널함수
: 커널 안에 있는 여러 함수

#### 커널
: 운영체제 핵심 -> 시스템콜을 제공하여(보안, 메모리, 프로세스, 파일 시스템, IO디바이스, IO요청 관리 등) 운영체제의 중추적인 역할

#### 유저모드
: 유저가 접근할 수 있는 영역을 제한 -> 컴퓨터 자원 침범 방지

#### 커널 모드
: 컴퓨터의 모든 자원에 접근 가능

#### modebit
: 유저모드와 커널 모드를 구별하는 비트 -> 1은 유저, 0은 커널


## 시스템 콜의 장점

- 유저 모드와 커널 모드가 분리
- 유저 프로그램은 복잡한 파일 시스템과 프로세스 생성과 같은 내부동작을 신경쓸 필요가 없음
- 운영체제 관리하에 프로그램이 운영 -> 시스템 안정성과 보안 강화
-> <mark style="background: #FFF3A3A6;">시스템으로 들어올 수 있는 단 하나의 통로만을 만듦</mark>
