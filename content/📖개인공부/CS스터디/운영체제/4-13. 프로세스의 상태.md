
CREATE -> READY -> running -> exit or block-> ready or blocked suspend-> ready suspend -> ready

- ready suspend와 blocked suspend를 묶어서 waiting이라고도 함.


### 생성상태
____
생성상태(create or new)는 프로세스가 생성된 상태를 의미 fork() 또는 exec()함수를 통해 생성
이때 PCB가 할당

- **fork()** : <mark style="background: #FFF3A3A6;">부모 프로세스의 주소 공간을 그대로 복사하며, 새로운 자식 프로세스를 생성하는 함수</mark>
	- 주소 공간만 복사하지, 부모 프로세스의 비동기 작업 등을 상속하지는 않음
- **exex()** : <mark style="background: #FFF3A3A6;">새롭게 프로세스를 생성</mark>하는 함수


### 대기 상태
_____
ready상태는 처음 프로세스가 생성(create)된 이후 메모리에 공간이 충분하면 메모리를 할당받고, 아니면 아닌 상태로 준<mark style="background: #FFF3A3A6;">비큐(준비 순서열)에 들어가서 대기중인 상태</mark>를 말함.
CPU로 부터 CPU 소유권이 넘어오기를 기다림


### 대기 중단 상태
___
ready suspend 상태는 준비큐가 꽉찬 상태 -> <mark style="background: #FFF3A3A6;">즉 메모리가 부족으로 일시 중단</mark>


### 실행 상태
___
running상태는 CPU 소유권과 메모리를 할당받고, <mark style="background: #FFF3A3A6;">명령을 수행중인 상태</mark> == CPU burst 가 일어났다.


### 중단 상태
____
blocked 상태는 <mark style="background: #FFF3A3A6;">어떤 이벤트가 발생한 이후 기다리며 프로세스가 차단된 상태</mark>.
ex) 예를 들어 프린트 인쇄 버튼을 눌렀을 때 실행되고 있던 프로세스가 잠깐 멈춘 듯 할 떄가 있음
	이는 프린트 인쇄에 대한 IO요청으로 인해 인터럽트가 발생되어 현재 실행하고 있던 프로세스가 blocked 상태로 잠시 변경된 것.


### 일시 중단 상태
____
blocked suspended는 대기 중단과 유사 -> 중단된 상태에서 프로세스가 실행되려 했지만 <mark style="background: #FFF3A3A6;">메모리 부족으로 일시 중단</mark>


### 종료 상태
____
terminated(exit) 상태는 프로세스 실행이 완료되어 해당 프로세스에 대한 <mark style="background: #FFF3A3A6;">자원을 반납하며, PCB가 삭제</mark>되는 상태. 
자연스럽게 종료되는 것도 있지만, 부모 프로세스가 자식 프로세스를 강제적으로 종료시켜 <u>비자발적 종료(abort)</u>도 있음
	자식 프로세스에 할당된 자원의 한계치를 넘거나
	부모 프로세스가 종료 되거나
	사용자가 process.kill 명령어로 종료
	시킬 때 abort 발생
