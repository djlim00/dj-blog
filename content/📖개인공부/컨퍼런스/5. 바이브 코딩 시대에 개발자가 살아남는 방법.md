# 바이브 코딩이란?

- 직관과 느낌에 의존하여 기존의 엄밀한 논리나 설계없이 AI의 도움을 받아 빠르게 개발하는 코딩 방식
- <span style="background:#fff88f">바이브 코딩의 단점을 베이직 코딩의 장점으로 보완하자! -> 둘다 하자.</span>

___
# 누가 대체될까?

> 1. 사용못하는 사람
>2. 활용 못하는 사람
>3. 자동화 못하는 사람


활용을 못하는 사람 -> MCP로 연결해서 사용하기
자동화 못하는 사람 -> make, n8n, zapier, Agentica ...etc

# 개발자의 본질은 무엇인가?

1. 문제 해결능력
2. 설계력
-> 코더가 아닌 개발자가 되자..!

> - 비즈니스 기획에 공감하고
> - 진짜 문제를 정의하며
> - 최적의 솔루션을 설계하는 사람

->  <mark style="background: #BBFABBA6;">항상 "왜"를 생각하자</mark>
->  <mark style="background: #BBFABBA6;">빠른 정보습득력을 갖추자</mark> (컨퍼런스, 뉴스레터 구독, 오픈채팅방)
->  AI로 바로 빠르게 직접 해보자
->  <mark style="background: #BBFABBA6;">명확한 지시 능력을 가지자</mark>
	- <u>컴퓨팅적 사고 능력을 가지자</u>
	- <u>목표에 대한 명확한 이해도를 가지자</u>

___

### 1. <span style="background:#fff88f">나만의 색</span>을 확실히 굳히자
### 2. <span style="background:#fff88f">잘하는</span> 개발자가 되자
- 시뮬레션을 잘하는 개발자 -> 단순히 동작 뿐만 아니라 예외상황 같은 경우도
### 3. 많은 경험을 <span style="background:#fff88f">내것으로</span> 만들자
- 단순히 경험 하는 것과 "내것으로 만드는 것"은 다르다.
### 4. <span style="background:#fff88f">쉽게 말하기</span>와 <span style="background:#fff88f">잘 듣는 능력</span>을 기르자
