## 2. Database 설계

<기본 흐름>
: 클라이언트 -> API -> 데이터베이스

-> 이런 단계는 모두 비용 -> 이걸 해결하기 위해서 SDK를 쓰자는거다

### 정규화
: 중복데이터를 저장하지 않는다(이론적으로는 맞는 얘기)

### 비정규화
: 중복 데이터 허용 -> <mark style="background: #BBFABBA6;">이걸 적절히 해야지 성능적으로 JOIN 횟수를 줄여서 성능이 좋아질 수 있음</mark>
: 얼마나 자주 조회되냐에 따라서 {정규화 vs 비정규화}의 이점을 가져가는 것임!

### 인덱스
: 데이터를 빠르게 조회할 수 있게 해주는 자료구조

항상 풀스캔 -> 성능 에바마..
있으면, 컬럼을 정해서 하면 된다.

#### 인덱스 공식
1. 유니크 컬럼은 자동으로 생성된다.
2. 외래키 컬럼은 인덱스를 명시적으로 생성하자.
3. status 컬럼은 인덱스를 추가하자.
4. 유니크 하지 않은 외래키와 Status는 복합 인덱스를 추가하자.
5. NOT 쿼리 쓰지 않기(인덱스를 안탈 가능성이 높음)

<mark style="background: #BBFABBA6;">supabase에서 막 통계를 제공해주는 기능이 있음..!</mark>

### Scale up(사양) / Scale out(개수)
- 자원 증가를 시키자! 걍 더 좋은 컴터를 쓰자
- 순산적으로 트래픽이 늘었을 때
	- <mark style="background: #BBFABBA6;">보통 사양을 바꾸는거보다는 개수를 늘리는게 빠르다!</mark>
	- 현장에서는 개수를 늘리는 경우가 많음
- 평균 트래픽에 맞춰서 적절한 사양을 선택하는 것이 중요함
- vercel에서도 둘 다 지원..
