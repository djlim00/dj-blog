- 시간복잡도 : O(ElogV)

- 최단 거리 배열에서 현재 값이 가장 작은 노드를 고른다.
	- 처음에는 시작 노드가 제일 작다(0이므로)
![[Pasted image 20250122171801.png]]

### <font color="#ffff00">4. 최단 거리 배열 업데이트하기 ⭐️</font>

- 선택한 노드에 연결된 에지의 값을 바탕으로(인접 리스트), 다른 노드의 값을 업데이트한다.
- 현재 선택한 노드의 엣지들을 탐색하고 -> 최단거리 리스트 업데이트
- 연결 노드의 최단 거리는 아래의 두 값 중 더 작은 값으로 업데이트
	- <font color="#ffff00">Math.min(선택 노드의 최단 거리 배열 값 + 엣지 가중치, 연결 노드의 최단거리 배열 값)</font>
![[Pasted image 20250122171957.png|700]]
-> 처음에는 1번 노드와 연결된 2번 노드에서 8과 무한을 비교 -> 8
-> 3번 노드에서 3과 무한을 비교 -> 3


### 5. 과정 3~4를 반복해서 최단거리 배열 완성하기! - 방문배열 처리

- 한번 선택된 노드는 다시 방문하기 않도록 visited[]를 만들어서 방문처리를 한다.





완성된 최단거리 배열 -> 시작노드에서 1번 노드까지 가는 최단거리, 시작노드에서 2번 노드까지 가는 최단거리

즉 위의 예시에서는 시작노드1에서 다른 노드로 가는 최단거리를 뜻한다!
