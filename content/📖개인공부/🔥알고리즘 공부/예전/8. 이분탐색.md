- 결국 이분탐색은 **정렬이 된** 값에서 내가 원하는 값을 찾는 것이다.
	- 이때 mid의 값을 기준으로 더 작은 부분을 버리거나, 더 큰 부분을 버리는 것이다. 
	- 이때 여기에는 두가지 방법이 있단 Upper Bound(상한선), Lower Bound(하한선)
	- Upper Bound 방법은 말 그대로 "상한선" 즉 우리가 찾는 값을 만족하는 최대의 인덱스를 구할 수 있다.
		- Upper Bound 방식에서 나온 index에 -1을 하면, index는 **초과**하는 첫번째 인덱스기 때문이다.
	- Lower Bound 방법은 "하한선" 즉 우리가 찾는 값을 만족하는 가장 작은 인덱스를 구한다.

- 두 방식은 **"중복값 이 존재하는 경우"**, 가장 큰 인덱를 원하는 경우 Upper Bound, 가장 작은 인덱스를 원하는 경우에는 Lower Bound 방식을 선택한다. 

### 1. Upper Bound 방식

- 우리가 반환받는 값은 초과하는 첫번쨰 값이기 때문에 right의 최댓값은 arr.length-1이 아니라 arr.length여야 한다.(초과하는 값이기 때문에 맨 마지막 값이 우리가 찾는 것일 수도 있음)

```java 
public static int upperBound(int[] arr, int value){ 
	
	int right = arr.length;
	int left = 0; 
	
	while(left<right){ 
		int mid = (left+right)/2;
		
		if(value<arr[mid]){ 
			right = mid; 
		}else{ 
			left = mid+1; 
		} 
	} 
	
	return left-1; 
}
```
- 이때 우리가 찾는 값보다 arr[mid] 의 값이 더 크다면 우리는 오른쪽 부분을 버린다. 
	- 이때 right=mid가 된다(mid도 초과하는 값에 해당되기 때문이다).
- 우리가 찾는 값보다 arr[mid] 의 값이 더 **작거나 같으면** 우리는 왼쪽 부분을 버린다.
	- 이때 left = mid+1된다(mid는 결고 우리가 찾는 범위에 속하지 않는다).

### 2. Lower Bound 방식
- 우리가 반환받는 값은 값 이상인 첫번째 값이 때문에 right=arr.length로 시작한다. 

```java
public static int lowerBound(int[] arr, int value){
	int right = arr.length;
	int left = 0;
	
	while(left<right){
		int mid = (left+right)/2;
		
		if(value > arr[mid]){
			left = mid+1; 
		}else{
			right = mid; 
		} 
	} 
	
	return left; 
}
```

- 이때 value가 arr[mid] 보다 큰 경우에는 우리는 왼쪽 값을 버린다. 
	- 이때 left=mid+1 왜냐하면, mid는 절대 우리가 찾는 범위에 해당하지 않기 때문이다.
- value가 arr[mid] 보다 **작거나 같은 경우**에는 오른쪽 값을 버린다.
	- 이때 right=mid(해당될 수 있기 때문이다.)
