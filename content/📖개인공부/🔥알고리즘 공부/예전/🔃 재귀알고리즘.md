# 재귀란?
___
: 자기 자신을 정의할 때 자기자신을 재참조 하는 것을 재귀적이라고 한다.
ex) 팩토리얼, 피보나치 수열 구하기, 순열, 조합, DFS...etc

팩토리얼(!)은 n * (n-1)!이다. -> !을 정의하기 위해서 !이 필요하다.

```java title:"팩토리얼 구하기 재귀함수 예제", hl:12,14
public class 팩토리얼 {  
    public static void main(String[] args) {  
        int N = 5;  
  
        int 팩토리얼값 = 팩토리얼(N);  
  
        System.out.println(팩토리얼값);  
    }  
  
    private static int 팩토리얼(int n) {  
        if (n > 0) { //유도(파생) 파트 
            return n * 팩토리얼(n - 1);  
        } else {  // 기저(재귀 탈출) 파트
            return 1;  
        }  
    }  
}
```

- <mark style="background-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color-color: #FFF3A3A6;">재귀함수는 탈출조건이 필요하다.</mark>
- <mark style="background: #FFF3A3A6;">결과를 만드는 유도(파생)파트가 필요하다</mark>

## 장점
1. 변수를 여러개 만들 필요가 없다
2. while문이나 for문같은 반복문을 사용하지 안하도 되기에, 코드가 짧아진다.

## 단점
1. 지속적으로 함수를 호출하면서, 지역변수, 매개변수, 반환값을 모두 process stack에 저장해야한다. 그리고 이런 과정은 선언한 변수의 값만 사용하면 되는 반복문에 비해 메모리를 더 많이 사용하고 속도저하가 일어난다.
2. 함수호출 -> 복귀를 위한 컨텍스트 스위칭 비용이 많이 든다.

## 😎 재귀 함수를 사용하는 이유
___

1. **알고리즘 자체가 재귀함수 표현이 자연스러운 경우가 있다.**
	- 피보나치 수열 -> 이걸 반복문으로 표현할려고 하면 코드가 길어지고, 점화식이 잘 표현이 안된다.
	- 팩토리얼
2. **변수의 개수를 줄여준다**
3. **가독성이 향상된다**



## ex) 유클리드 호제법 - 최대공약수
___

**💡 호제법이란?****- 두 수가 서로 상대방 수를 나누어서 결국 원하는 수를 얻는 알고리즘을 의미합니다.


>1. 최대 공약수를 구하려는 두 수를 각가 한변으로 하는 직사각형은 만든다
>2. 짧은 변을 기준으로 하는 정사각형으로 직사각형을 나눈다
>3. 남는 직사각형을 계속해서 짧을 변을 기준으로 나눈다
>4. 정사각형으로만 구성되었을 떄 변의 길이가 최대공약수다.

- 23, 9 -> 23 / 9 = 2 ...<mark style="background: #FFF3A3A6;">5</mark> -> 9 / <mark style="background: #FFF3A3A6;">5</mark>  = 1...<mark style="background: #FFF3A3A6;">4</mark> -> <mark style="background: #FFF3A3A6;">5</mark> / <mark style="background: #FFF3A3A6;">4</mark> = 1...1 => 영원히 나눠지지 않기 때문에 GCD는 1임
- 22, 8 -> 22 / 8 = 2...6 -> 8 / 6 = 1...2 -> 6 / <mark style="background: #FFF3A3A6;">2</mark> = 3...0

```java title:"유클리드 호제법을 이용한 최대공약수", hl:11-14  
public class 유클리드호제법 {  
    public static void main(String[] args) {  
        int A = 120;  
        int B = 25;  
  
        int gcd = 유클리드호제법GCD(A, B);  
  
        System.out.println(gcd);  
    }  
  
    private static int 유클리드호제법GCD(int a, int b) {\  
        if (b == 0) return a;  
        return 유클리드호제법GCD(b, a % b);  
    }  
}
```


## ex) 피보나치 수열
___
```java title:"재귀를 이용한 피보나치 수열구하는 코드"
package A0210;  
  
public class 재귀함수분석1_피보나치수열 {  
    private static int 피보나치(int n) {  
  
        if (n <= 1) return n;  
        return 피보나치(n - 1) + 피보나치(n - 2);  
    }  
  
    public static void main(String[] args) {  
        int N = 3;  
  
        int pibo = 피보나치(N);  
        System.out.println(pibo);  
    }  
}
```


## ex) 꼬리재귀 - 재귀하고 또 재귀하고 ƺ
___
```java title:"재귀가 끝나고나서 따라오면서 수행되는 경우", hl:5,7
public class 재귀함수분석2_재귀의재귀 {  
  
    private static void Recur(int n) {  
        if (n > 0) {  
            Recur(n - 1);  
            System.out.println(n);  
            Recur(n - 2);  
        }  
    }  
  
    public static void main(String[] args) {  
        int n = 4;  
  
        Recur(4);  
    }  
}
```
- 재귀로 들어갔다가, 나오면서 다시 재귀를 타는 경우이다.
- 이런걸 뒤따라 온다는 의미에서 꼬리라고 부른다.
